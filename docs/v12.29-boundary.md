# Agent Space v12.29 Boundary

状态：complete
目标：Make package proof verification reject malformed packaged file lists before accepting package proof metadata.

## v12.29 新增

- `asp-verify.mjs package-proof <manifest.json>` requires `files` to be a non-empty array of unique safe relative paths.
- File entries with absolute paths, backslashes, empty path segments, `.` segments, or `..` segments are rejected with `package proof files invalid`.
- `test/package-contract.test.mjs` covers self-consistent package proofs whose `files` field is not an array or escapes the package root.

## 为什么这是 v12.29

v12.28 bound the displayed filename to the tarball path. v12.29 closes the next external-consumer metadata hole: a manifest can no longer present malformed packaged file lists while keeping every digest and size field self-consistent.
