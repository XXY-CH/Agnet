# Agent Space v5.7 Boundary

状态：complete
目标：把 MCP initialize session metadata 绑定进 sandbox evidence。

## v5.7 新增

- MCP sandbox evidence records `mcp_session`。
- `mcp_session` includes:
  - `protocol_version`
  - `server_info`
- `sandbox_proof` signs this metadata because it signs sandbox evidence。
- Remote audit proof returns the metadata through the signed receipt。

## 为什么这是 v5.7

Tool command digest says which local command was launched。

MCP session metadata says which MCP protocol/server identity answered initialize:

```text
MCP initialize
  -> protocol/server metadata
  -> sandbox evidence
  -> signed sandbox_proof
  -> receipt / remote audit proof
```

This improves tool trust without implementing long-running MCP sessions。

## v5.7 不新增

- 不做 long-running MCP session reuse。
- 不做 session resume。
- 不做 resources/prompts/list。
- 不做 MCP server registry。
- 不做 server signature verification。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 *.test.mjs
```

The Go integration test must fail if MCP session metadata is absent from sandbox evidence or not covered by the signed sandbox proof。
