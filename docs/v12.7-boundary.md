# Agent Space v12.7 Boundary

状态：complete
目标：Make proof bundle verification validate every proof-file path before opening any proof files.

## v12.7 新增

- `asp-verify.mjs proof-bundle <bundle.json>` resolves and validates `receipt_frame`, `trusted_zones`, `swarm_close_frame`, and `swarm_close_trusted_zones` before reading any of them.
- Unsafe later path fields fail with their own field-specific error even if an earlier path is safe but missing.
- `test/public-node-proof.test.mjs` proves an escaping `swarm_close_frame` is rejected before a missing `receipt_frame` is opened.

## 为什么这是 v12.7

v12.6 made the bundle manifest itself fail closed. The next smallest path-boundary fix is preflight ordering: all proof-file references should be validated as bundle inputs before the verifier opens any proof file.
