# Agent Space v0.9 Boundary

状态：v0.9 candidate  
目标：为 v1 Federation Gateway 增加 trusted Zone store。

## v0.9 新增

- `verifyZoneDescriptor(zoneDescriptor)`。
- `writeTrustedZones(file, zones)`。
- `loadTrustedZones(file)`。
- Trusted Zone store 文件格式：

```json
{
  "zones": [
    {
      "name": "zone://remote-b",
      "zid": "zid:ed25519:...",
      "public_key_spki": "...",
      "zone_signature": "..."
    }
  ]
}
```

- 新增 trusted Zone store 验证测试。
- 新增 tampered Zone descriptor 拒绝测试。

## 为什么这是 v0.9

v1 要做两个 Zone 之间的 Federation Gateway。

Federation 不能从“连上了就信”开始。Zone A 必须先明确自己信任 Zone B 的哪个 `zid` 和公钥。

v0.9 定义最小 trust anchor：

```text
remote Zone descriptor
  -> recompute zid from public key
  -> verify Zone descriptor self-signature
  -> store by zid
```

## v0.9 不新增

- 不做 Federation Gateway。
- 不做 cross-zone task frame。
- 不做 trust policy language。
- 不做 remote registry fetch。
- 不做 certificate chain。
- 不做 trust-on-first-use。

这些进入 v1。

## v1 入口条件

v1 可以从这里开始：

```text
Zone A trusted-zones.json contains Zone B zid
Zone A sends FED_TASK_OPEN to Zone B gateway
Zone B verifies requester/task
Zone B runs local worker
Zone B returns events and receipt
Zone A verifies remote receipt
```
