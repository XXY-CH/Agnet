# Agent Space v12.31 Boundary

状态：complete
目标：Make package proof verification reject manifests whose package `name` and `version` metadata do not match the local npm tarball filename.

## v12.31 新增

- `asp-verify.mjs package-proof <manifest.json>` requires `filename` to equal `<name>-<version>.tgz` for the current local npm package proof.
- `test/package-contract.test.mjs` covers a self-consistent package proof whose package `name` no longer matches the tarball filename.

## 为什么这是 v12.31

v12.30 bound the package proof's `manifest` field to the verifier input filename. v12.31 closes the next metadata split: a manifest can no longer claim one package identity while proving a tarball filename for another local npm package identity.
