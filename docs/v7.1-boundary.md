# Agent Space v7.1 Boundary

状态：complete
目标：显式 drain 一个 queued task，并复用现有 Go execution path。

## v7.1 新增

- Go gateway accepts `FED_QUEUE_DRAIN` with `task_id`。
- Queue item stores enough signed task context for later drain:
  - origin zone descriptor
  - requester descriptor
  - signed task
- Drain verifies the stored requester/task again through the existing `verifyTaskOpen` path。
- Drain changes queue state:
  - `queued`
  - `running`
  - `completed`
- Completed queue item records `receipt_digest`。
- Drain emits the existing task event, receipt, and close frames from `executeTask`。

## 为什么这是 v7.1

v7.0 proved tasks can exist durably before execution。

v7.1 proves that queued tasks can be explicitly executed without inventing a second executor:

```text
durable queue item
  -> FED_QUEUE_DRAIN
  -> verify stored signed task
  -> existing executeTask path
  -> completed queue state
```

This turns the queue from storage into the first scheduler control surface。

## v7.1 不新增

- 不做 automatic drain loop。
- 不做 leases。
- 不做 retry/backoff。
- 不做 priority。
- 不做 multi-node scheduler。
- 不做 Human Gateway drain button。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 test/go-fed-discovery.test.mjs
```

The Go integration test must fail if `FED_QUEUE_DRAIN` cannot execute a queued task or if the queue item does not end as `completed` with a receipt digest。
