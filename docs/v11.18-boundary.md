# Agent Space v11.18 Boundary

状态：complete
目标：Make the Node `FED_SWARM_CLOSE` verifier reject missing signing Zones with a protocol error.

## v11.18 新增

- Node `verifySwarmClose` rejects missing or non-object `zone` descriptors before calling Zone descriptor verification.
- The Node conformance test proves a trusted `FED_SWARM_CLOSE` body without `zone` fails with `swarm close zone missing`, not a low-level descriptor field `TypeError`.
- The portable Swarm close verifier keeps its boundary structural: signing Zone object, close proof object, close signature, signed Swarm id, frame/body Swarm id match, and step receipt shape.

## 为什么这是 v11.18

`FED_SWARM_CLOSE` is a verifier-facing proof frame. The signing Zone descriptor is the trust anchor for the close signature, so missing Zone evidence should fail as protocol validation before descriptor internals are read. v11.18 keeps the fix to one verifier guard and one red test.

## v11.18 不新增

- 不实现 generic Zone schema validation。
- 不实现 Node audit-backed Swarm completeness verification。
- 不验证 step receipts against an audit log。
- 不实现 dynamic Swarm decomposition。
- 不实现 scheduler-owned routing。
- 不实现 parallel or cross-Zone Swarm execution。
- 不实现 public reachability proof。
- 不实现 A2A/ARD compatibility。
