# Agent Space v12.30 Boundary

状态：complete
目标：Make package proof verification reject manifests whose `manifest` field does not match the verifier input filename.

## v12.30 新增

- `asp-verify.mjs package-proof <manifest.json>` requires `manifest` to equal the final path segment of the verifier input path.
- `package-contract.test.mjs` covers a self-consistent package proof whose `manifest` field names a different manifest file.

## 为什么这是 v12.30

v12.29 validated the packaged file-list shape. v12.30 closes the next metadata split: a package proof file can no longer present one manifest filename while being verified through another file path.
