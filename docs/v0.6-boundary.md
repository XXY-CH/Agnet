# Agent Space v0.6 Boundary

状态：v0.6 candidate  
目标：定义 Agent key rotation 的最小连续性证明。

## v0.6 新增

- `rotationProof(previousAgent, nextAgent)`。
- `verifyRotationProof(proof, previousDescriptor, nextDescriptor)`。
- Rotation body：

```json
{
  "previous_aid": "aid:ed25519:...",
  "next_aid": "aid:ed25519:..."
}
```

- `previous_signature`：旧 Agent key 对 body 签名。
- `next_signature`：新 Agent key 对 body 签名。
- 新增 rotation proof 测试。

## 为什么这是 v0.6

v0.5 让身份跨重启稳定。

v0.6 处理下一件真实事情：key 终有一天要换。

`aid` 不应该假装能在换 key 后保持不变。`aid` 是公钥派生身份，换 key 就换 `aid`。

v0.6 给出的是连续性证明：

```text
old aid controls old key
new aid controls new key
old key signs transition
new key signs transition
```

这样以后 Zone 可以把同一个 `agent://` alias 从旧 `aid` 重新绑定到新 `aid`，并附上 rotation proof。

后续 Zone alias rebinding proof 边界见 `docs/v0.7-boundary.md`。

## v0.6 不新增

- 不做自动轮换命令。
- 不做 revocation list。
- 不做 Zone alias rebinding policy。
- 不做 compromised-key recovery。
- 不做 lineage graph。

这些需要 Zone policy 和 federation 规则。
