# Agent Space v12.40 Boundary

状态：complete
目标：Make trusted package signer files reject missing signer lists without leaking JavaScript type errors.

## v12.40 新增

- `asp-verify.mjs package-proof <manifest.json> <trusted-signers.json>` rejects `null` trusted signer files with `trusted package signer list missing`.
- The trusted signer loader now checks the parsed trust file is an object or array before reading `signers`.
- `test/package-contract.test.mjs` covers null trusted signer list rejection.

## 为什么这是 v12.40

v12.39 added a caller-supplied trusted signer file as the local package proof trust root. That file is a trust-boundary input, so malformed shapes must fail closed with protocol errors rather than JavaScript property-read errors.

## v12.40 不新增

- 不实现 JSON Schema。
- 不实现 npm registry signing。
- 不实现 release transparency。
- 不实现 SBOM。
- 不实现 package publish。
- 不实现 hosted public node。
- 不实现 real outside-host reachability。
- 不实现 scheduler-owned routing。
- 不实现 A2A/ARD compatibility。

This is trusted signer list shape validation for the local package proof verifier, not a full package release trust system.
