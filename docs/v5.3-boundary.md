# Agent Space v5.3 Boundary

状态：complete
目标：让 retry 成为 signed, auditable execution link。

## v5.3 新增

- Go gateway accepts `FED_TASK_RETRY`。
- Retry request includes `retry_of`。
- Retry still carries a normal signed task。
- Gateway rejects retry without `retry_of`。
- Retried task executes through the normal task path。
- Retried receipt records `retry_of`。
- Remote audit query can fetch the retried receipt by new task id。

## 为什么这是 v5.3

Ultimate needs failure recovery, but the current gateway has no scheduler。

v5.3 adds only the evidence link:

```text
failed or cancelled task id
  -> signed retry task
  -> normal execution
  -> receipt.retry_of
  -> remote audit proof
```

This proves lineage without pretending retry is automatic。

## v5.3 不新增

- 不做 automatic retry。
- 不做 backoff。
- 不做 retry queue。
- 不做 scheduler。
- 不验证 `retry_of` 对应的旧 task 是否存在。
- 不做 failure classification。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 *.test.mjs
```

The Go integration test must fail if `FED_TASK_RETRY` is unsupported, accepts missing `retry_of`, or the retried receipt does not preserve retry lineage。
