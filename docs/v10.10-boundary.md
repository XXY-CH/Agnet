# Agent Space v10.10 Boundary

状态：complete
目标：Add a minimal Docker proof demo contract around the existing local proof demo.

## v10.10 新增

- `Dockerfile` runs `bash scripts/proof-demo.sh` in a Node image.
- `.dockerignore` keeps local state, artifacts, logs, and `.git` out of the image build context.
- `scripts/docker-proof-demo.sh` builds `agnet-proof-demo` and runs it.
- `test/docker-demo.test.mjs` guards that the Docker demo delegates to the local proof script.

## 为什么这是 v10.10

v10.9 made the proof layer reproducible from one local command. v10.10 adds the smallest Docker contract around that command, without adding services, Compose, public node hosting, or new runtime claims.

## v10.10 不新增

- 不做 Docker Compose。
- 不做 public node。
- 不做 HTTP verifier service。
- 不做 long-running daemon。
- 不做 package release。
- 不做 remote artifact fetch。
- 不做 receipt store/search。
- 不声称本机已经运行过 Docker image；当前验证环境 Docker daemon 不可用。

## 验收

```bash
node --test --test-concurrency=1 test/docker-demo.test.mjs
node --test --test-concurrency=1 test/proof-demo.test.mjs
go test ./...
node --test --test-concurrency=1 test/*.test.mjs
git diff --check
```

The Docker contract test must fail before `Dockerfile` and `scripts/docker-proof-demo.sh` exist.
