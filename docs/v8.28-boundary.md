# Agent Space v8.28 Boundary

状态：complete
目标：Audit verification checks configured filesystem artifact mirrors against signed manifests.

## v8.28 新增

- `--verify-audit --artifact-store <dir>` verifies mirrored artifact bytes at `<dir>/by-sha256/<sha256>`.
- `--verify-audit --artifact-store <dir>` verifies mirrored manifest sidecars at `<dir>/by-sha256/<sha256>.manifest.json`.
- The verifier rejects mirrored sidecars that differ from signed receipt artifact manifests.
- The verifier rejects mirrored bytes whose size or SHA-256 differs from signed receipt artifact manifests.
- The Go integration test tampers with mirror sidecar and mirror bytes and expects audit verification failure.

## 为什么这是 v8.28

v8.27 made artifact store mirrors writable.
v8.28 makes that mirror auditable:

```text
signed receipt manifest
  -> local artifact verification
  -> local digest sidecar verification
  -> optional --artifact-store mirror verification
```

This keeps the storage track honest: a mirror is useful only if later audit can detect drift or tampering.

## v8.28 不新增

- 不做 S3/MinIO/object-store API。
- 不做 remote artifact fetch/download。
- 不做 artifact GC。
- 不做 artifact browser。
- 不做 auth model for mirrored artifacts。
- 不做 container sandbox。
- 不做 public deployment。
- 不做 A2A/ARD compatibility。

## 验收
