# Agent Space v5.13 Boundary

状态：complete
目标：在 MCP `tools/call` 前拒绝缺少 selected tool schema `required` 字段的 arguments。

## v5.13 新增

- Selected MCP tool `inputSchema.required` is checked before `tools/call`。
- Missing required argument returns `FED_TASK_ERROR`。
- Existing `mcp_tool_arguments_digest` remains digest-only evidence。

## 为什么这是 v5.13

v5.12 records argument payload identity。

v5.13 adds the first argument policy edge without owning a full JSON Schema validator:

```text
selected tool inputSchema.required
  -> arguments presence check
  -> reject missing required field before tools/call
```

## v5.13 不新增

- 不做完整 JSON Schema validation。
- 不做类型校验。
- 不做格式校验。
- 不保存 arguments 明文。
- 不做 policy engine。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 *.test.mjs
```

The Go integration test must fail if a selected MCP tool requires an argument that is absent and the runtime still calls the tool。
