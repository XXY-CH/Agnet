# Agent Space v8.20 Boundary

状态：complete
目标：Local artifacts are also written under a content-addressed SHA-256 path.

## v8.20 新增

- `writeArtifact` keeps writing the named `artifact://local/...` path.
- The same bytes are also written to `artifacts/by-sha256/<sha256>`.
- The digest-addressed copy gets the same `.manifest.json` sidecar as the named artifact.
- Human Gateway can read the digest-addressed artifact through the existing `/artifacts/by-sha256/<sha256>` static route.

## 为什么这是 v8.20

Ultimate artifact evidence needs a stable content-addressed retrieval path, not only a task-shaped filename.

v8.20 keeps the smallest local version:

```text
artifact bytes
  -> sha256
  -> artifacts/by-sha256/<sha256>
  -> existing Human Gateway static artifact route
```

## v8.20 不新增

- 不改 artifact manifest schema。
- 不做 dedupe index。
- 不做 garbage collection。
- 不做 remote artifact download。
- 不做 artifact browser。
- 不做 object storage / MinIO / S3。
- 不做 full transcript artifact storage。
- 不做 container sandbox。
- 不做 public deployment。
- 不做 A2A/ARD compatibility。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 go-fed-discovery.test.mjs
node --test --test-concurrency=1 *.test.mjs
```

The Go integration test must fail if the digest-addressed artifact is missing, differs from the named artifact, lacks the manifest sidecar, or cannot be read through Human Gateway.
