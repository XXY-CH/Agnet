# Agent Space v6.5 Boundary

状态：complete
目标：`FED_TASK_RESUME` 必须引用 audit 中真实存在的 checkpoint。

## v6.5 新增

- Resume request still includes `checkpoint_id`。
- Gateway scans the audit log for a matching checkpoint before executing the resumed task。
- Unknown checkpoint ids are rejected with `resume checkpoint not found`。
- Existing resumed receipts still record:
  - `resumed_from`
  - new checkpoint refs
  - signed checkpoint object

## 为什么这是 v6.5

v5.1 proved resume can link a new task to a parent checkpoint id。

v6.5 makes that link evidence-backed:

```text
FED_TASK_RESUME checkpoint_id
  -> audit checkpoint lookup
  -> resumed task execution
  -> new receipt links parent checkpoint
```

This is still not process restore. It closes the dishonest gap where a resume could point at a checkpoint id that never existed。

## v6.5 不新增

- 不做 process snapshot restore。
- 不做 long-running MCP session resume。
- 不做 scheduler queue。
- 不做 checkpoint storage service。
- 不做 distributed checkpoint lookup。
