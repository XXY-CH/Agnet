# Agent Space v0.7 Boundary

状态：v0.7 candidate  
目标：把 Agent key rotation 接入 Zone-controlled `agent://` alias 生命周期。

## v0.7 新增

- `aliasRebindingProof(zone, previousDescriptor, nextDescriptor, agentRotationProof)`。
- `verifyAliasRebindingProof(proof, zoneDescriptor, previousDescriptor, nextDescriptor)`。
- Rebinding body：

```json
{
  "zone": "zid:ed25519:...",
  "alias": "agent://local/summarizer",
  "previous_aid": "aid:ed25519:...",
  "next_aid": "aid:ed25519:..."
}
```

- Zone key 对 rebinding body 签名。
- Agent rotation proof 嵌入 rebinding proof。
- 新增 alias rebinding 测试。

## 为什么这是 v0.7

v0.6 证明旧 Agent key 和新 Agent key 有连续性。

v0.7 证明 Zone 接受同一个 `agent://` alias 从旧 `aid` 迁到新 `aid`：

```text
Agent proves old aid -> new aid
Zone proves alias follows old aid -> new aid
```

后续 Zone-local revocation 边界见 `docs/v0.8-boundary.md`。

这保持了三层边界：

```text
aid       key-derived Agent identity
zid       key-derived Zone identity
agent://  Zone-controlled route name
```

## v0.7 不新增

- 不做 automatic key rotation command。
- 不做 registry history。
- 不做 revocation list。
- 不做 compromised-key recovery。
- 不做 federation trust propagation。

这些需要更完整的 Zone policy。
