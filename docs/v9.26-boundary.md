# Agent Space v9.26 Boundary

状态：complete
目标：Go federation gateway accepts one explicit two-step Swarm DAG and binds the downstream receipt to the upstream artifact manifest.

## v9.26 新增

- Go accepts `FED_SWARM_OPEN` after the normal authenticated federation session handshake.
- A Swarm frame carries a `swarm_id` and ordered `steps`.
- Each step reuses the existing signed task verifier, worker lookup, policy check, event stream, artifact write, audit append, and signed receipt path.
- A step can declare `after: [...]`; every dependency must already be completed in the same Swarm frame.
- Each signed receipt now can carry a signed `swarm` proof object when executed inside a Swarm:
  - `swarm_id`
  - `step_id`
  - `after`
  - `input_artifacts`
- For dependent steps, `input_artifacts` binds the dependency step id to the upstream artifact manifest fields: `uri`, `sha256`, and `manifest_hash`.
- The integration test proves two different Go workers execute one Swarm DAG: `summary` then `translation`.

## 为什么这是 v9.26

Fable 的反馈指出当前最大结构性缺口不是 A2A/ARD compatibility，也不是经济层，而是 Swarm / multi-agent collaboration。
v9.26 接受这个方向，但只落最小可验证内核：caller-supplied DAG + two workers + artifact dependency proof。

## v9.26 不新增

- 不做动态任务拆解。
- 不做候选 Agent 召回。
- 不做 scheduler。
- 不做自动 drain。
- 不做并行执行。
- 不做冲突解决。
- 不做 Swarm UI。
- 不做跨 Zone Swarm。
- 不做 container namespace sandbox。
- 不做 Node Swarm server。
- 不做 A2A/ARD compatibility。
