# Agent Space v9.3 Boundary

状态：complete
目标：Go audit readers accept larger valid JSONL audit entries.

## v9.3 新增

- `readAuditEntries` raises the scanner max token size to 1 MiB.
- Go package test writes a valid audit line larger than the default 64 KiB scanner limit.
- The test verifies the shared audit reader accepts that line.

## 为什么这是 v9.3

The review report identified Go audit scanning as a reliability bug: `bufio.Scanner` defaults to 64 KiB per token, while audit entries can contain large sandbox, artifact, or MCP evidence.

v9.3 fixes the shared reader without changing the audit format:

```text
audit JSONL line
  -> readAuditEntries
  -> scanner buffer up to 1 MiB
```

## v9.3 不新增

- 不做 streaming JSON decoder。
- 不做 audit compaction。
- 不做 audit pagination。
- 不做 state-file atomic writes。
- 不做 Go approval locking。
- 不做 queue grant replay index。
- 不做 A2A/ARD compatibility。

## 验收

```bash
go test ./cmd/go-fed-discovery
go test ./...
node --test --test-concurrency=1 test/*.test.mjs
```

The Go package test must fail if `readAuditEntries` falls back to the default 64 KiB scanner token limit.
