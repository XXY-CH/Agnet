# Agent Space v1.4 Boundary

状态：v1.4 candidate  
目标：把 exact capability discovery 接到 signed task handoff。

## v1.4 新增

- `federation-gateway.mjs request-capability`
- Capability query result -> select first verified match -> send `FED_TASK_OPEN`
- Gateway rejects task whose `task.to` does not match local worker alias。
- Capability handoff integration test。
- No-match failure integration test。

## 为什么这是 v1.4

v1.3 只能回答：

```text
谁有 summarize.text？
```

v1.4 完成：

```text
谁有 summarize.text？
  -> verified remote match
  -> signed task to that alias
  -> remote execution
  -> signed receipt
```

这让 capability discovery 从“查询”变成“可执行任务路由”的最小闭环。

## v1.4 不新增

- 不做多候选选择策略。
- 不做排序。
- 不做 reputation。
- 不做 vector routing。
- 不做 fallback。
- 不做 scheduler。

当前选择策略是：第一个 verified exact match。

后续 signed capability credential 边界见 `docs/v1.5-boundary.md`。
