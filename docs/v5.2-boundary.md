# Agent Space v5.2 Boundary

状态：complete
目标：让取消意图成为 signed, auditable protocol evidence。

## v5.2 新增

- Go gateway accepts `FED_TASK_CANCEL`。
- Cancel request includes a signed `cancel` object。
- Cancel object binds:
  - `task_id`
  - `from`
  - `to`
  - `reason`
- Gateway verifies requester descriptor and cancel signature。
- Gateway emits `task.cancelled` event。
- Target worker signs a cancellation receipt。
- Remote audit query can fetch the cancellation receipt by task id。

## 为什么这是 v5.2

Ultimate requires human/organization revocation power。

v5.2 adds the protocol evidence for that power:

```text
signed cancel request
  -> task.cancelled event
  -> worker-signed cancellation receipt
  -> hash-chained audit
  -> remote audit proof
```

This is intentionally smaller than a real scheduler. The current gateway executes tasks synchronously, so v5.2 proves cancellation evidence, not live interruption。

## v5.2 不新增

- 不中断正在运行的 external/MCP process。
- 不做 task scheduler。
- 不做 task state table。
- 不做 retry queue。
- 不做 authorization UI。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 test/*.test.mjs
```

The Go integration test must fail if `FED_TASK_CANCEL` is unsupported, cancel signature is ignored, or the cancellation receipt is missing from remote audit query。
