# Agent Space v9.17 Boundary

状态：complete
目标：Go audit append serializes same-host writers and refreshes the hash-chain head under lock.

## v9.17 新增

- `AuditLog.Append` takes an exclusive `<audit>.lock` file lock before writing.
- The lock is process-wide on the same host, not just an in-process mutex.
- `Append` re-reads and verifies the existing audit log under the lock before choosing `prev_hash`.
- Stale `AuditLog` instances append after the current shared head instead of forking the chain.
- Corrupt shared audit logs are rejected before appending.

## 为什么这是 v9.17

Ultimate needs receipts and audit chains to stay authoritative under multiple local gateway processes.
v9.17 closes the Go same-host append race without introducing a distributed log protocol.

## v9.17 不新增

- 不做 Node cross-process audit locking。
- 不做 distributed audit log sync。
- 不做 remote audit consensus。
- 不做 multi-host locking。
- 不做 database-backed audit store。
- 不做 scheduler or automatic drain。
- 不做 A2A/ARD compatibility。
