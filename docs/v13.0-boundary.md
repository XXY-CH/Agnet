# Agent Space v13.0 Boundary

状态：complete
目标：Open v13 as the Ultimate-facing protocol milestone without claiming new runtime capability.

## v13.0 新增

- V13 is now active at `v13.0-protocol`.
- The active roadmap is organized around five large evidence gates: real hosted/public reachability, release trust/SBOM, strong sandbox/remote attestation, semantic discovery/reputation ranking, and dynamic Swarm scheduling.
- README, implementation status, ASP Core draft, and docs contract point to the v13 opening state.

## 为什么这是 v13.0

v12 closed the externally consumable proof surface. The next Ultimate gaps are no longer another local-only verifier field; they require evidence that crosses bigger trust boundaries. v13.0 records those boundaries before implementation so future slices do not drift into compatibility work, upper-layer demo orchestration, or unverified capability claims.

## v13.0 不新增

- 不实现新的 protocol frame。
- 不实现新的 verifier command。
- 不实现 hosted public node。
- 不实现 real outside-host reachability。
- 不实现 semantic discovery/reputation ranking。
- 不实现 dynamic Swarm scheduling。
- 不实现 strong sandbox/remote attestation。
- 不实现 release trust/SBOM。
- 不实现 upper-layer demo/master-agent orchestration。
- 不实现 A2A/ARD compatibility。

## 硬边界

- no fake public reachability without external-host evidence.
- `external-host` reachability must be signed, trusted, and bound to the verified transport proof and receipt digest.
- semantic discovery/reputation ranking must remain inspectable and evidence-based, not opaque vector-only routing.
- dynamic Swarm scheduling must preserve complete close proof accountability.
- strong sandbox/remote attestation must fail closed when the required runtime or attestation evidence is unavailable.
- release trust/SBOM must bind to the produced package artifact and signer evidence.
