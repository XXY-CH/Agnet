# Agent Space v9.18 Boundary

状态：complete
目标：Go federation TCP listener can run over TLS with operator-provided certificate and key files.

## v9.18 新增

- `--tls-cert <path>` and `--tls-key <path>` wrap the main Go federation TCP listener with TLS.
- The listener reports `transport: "fed+tls"` when TLS is enabled and `transport: "fed+tcp"` otherwise.
- Startup rejects partial TLS config: both cert and key are required.
- Go package coverage proves a TLS client can complete a handshake with the federation listener.

## 为什么这是 v9.18

Ultimate needs federation to leave loopback-only assumptions.
v9.18 is the smallest transport hardening step: TLS on the existing frame protocol without changing task, receipt, or Human Gateway behavior.

## v9.18 不新增

- 不做 mTLS client certificate verification。
- 不做 public bind address。
- 不做 WebSocket TLS。
- 不做 Human Gateway HTTPS。
- 不做 QUIC。
- 不做 DHT / relay / edge gateway。
- 不做 A2A/ARD compatibility。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 test/go-fed-discovery.test.mjs
node --test --test-concurrency=1 test/*.test.mjs
git diff --check
```

The Go package test must fail if the federation listener ignores TLS cert/key paths or cannot complete a TLS handshake.
