# Agent Space v3.7 Boundary

状态：v3.7 complete
目标：让 Go worker execution 支持最小 MCP stdio tool call。

## v3.7 新增

- Worker profile can set:
  - `tool: "mcp.stdio"`
  - `tool_name`
  - `tool_command`
- Go runs an MCP stdio server as a short-lived process。
- Go sends newline-delimited JSON-RPC:
  - `initialize`
  - `notifications/initialized`
  - `tools/call`
- Go reads `result.content[].text` and stores it as the task artifact。
- Receipt records `tool: "mcp.stdio"`。
- Node integration test starts a temporary MCP stdio server and verifies:
  - selected worker invokes `tools/call`
  - artifact is MCP-produced
  - receipt records the MCP tool adapter
  - Human Gateway serves the MCP-produced artifact

## 为什么这是 v3.7

v3.6 proved external stdio tool execution。

v3.7 upgrades the external tool envelope to the MCP shape:

```text
task envelope
  -> MCP initialize
  -> MCP tools/call
  -> content text
  -> artifact
  -> receipt
```

This proves MCP compatibility at the smallest useful boundary。

## v3.7 不新增

- 不做 MCP resources。
- 不做 MCP prompts。
- 不做 streaming progress。
- 不做 long-running MCP sessions。
- 不做 Streamable HTTP transport。
- 不做 container sandbox。
- 不做 tool approval UI。

## 验收

```bash
node --test --test-concurrency=1 *.test.mjs
go test ./...
```

The Go integration test must fail if the artifact falls back to `external.stdio` or built-in tool output。
