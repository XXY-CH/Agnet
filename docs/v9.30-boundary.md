# Agent Space v9.30 Boundary

状态：complete
目标：Swarm audit verification rejects duplicate completed receipts for the same `swarm_id` + `step_id`.

## v9.30 新增

- `--verify-audit` now rejects a second completed Swarm receipt with the same `swarm_id` and `step_id`.
- The verifier no longer lets a later duplicate step overwrite the first completed step's artifact/receipt evidence.
- Go package coverage proves a signed duplicate step receipt is rejected even when downstream dependencies match the duplicate receipt's artifact manifest and receipt digest.
- This aligns audit verification with `FED_SWARM_OPEN` execution, which already rejects duplicate step ids in one Swarm frame.

## 为什么这是 v9.30

v9.29 made declared Swarm edges match the input artifact step ids.
v9.30 closes the matching uniqueness hole: a Swarm step id names one completed step, not whichever same-id receipt appeared last in the audit scan.

## v9.30 不新增

- 不做 duplicate step merge 语义。
- 不做 Swarm retry step semantics。
- 不做 cross-audit dependency lookup。
- 不做 remote audit sync。
- 不做 scheduler。
- 不做 dynamic decomposition。
- 不做 parallel DAG execution。
- 不做 conflict resolution。
- 不做 Swarm UI。
- 不做 Node Swarm server。
- 不做 container namespace sandbox。
- 不做 A2A/ARD compatibility。
