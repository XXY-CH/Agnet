# Agent Space v12.26 Boundary

状态：complete
目标：Make package proof verification reject npm digest metadata that does not match the tarball bytes.

## v12.26 新增

- `asp-verify.mjs package-proof <manifest.json>` verifies `shasum` as SHA-1 over the manifest-relative tarball bytes.
- `asp-verify.mjs package-proof <manifest.json>` verifies `integrity` as npm-style `sha512-<base64>` over the manifest-relative tarball bytes.
- `test/package-contract.test.mjs` covers mismatched `shasum` and `integrity` fields while keeping `proof_digest` self-consistent.

## 为什么这是 v12.26

v12.25 made the package proof directory locally copyable. v12.26 makes the verifier check all digest metadata emitted by `npm pack`, not only the ASP-style SHA-256 and proof digest.

## v12.26 不新增

- 不实现 package signing。
- 不实现 SBOM。
- 不发布 npm package。
- 不改变 `package.json` exports/bin/files。
- 不改变 tarball SHA-256 calculation。
- 不实现 external public reachability proof。
- 不实现 hosted public node。
- 不增加 DNS, TLS, QUIC, NAT traversal, or remote probe infrastructure。
- 不改变 normal `fed-receipt` verification。
- 不改变 `proof-bundle` verifier JSON output。
- 不实现 transport negotiation。
- 不实现 batch verifier。
- 不实现 JSON Schema。
- 不实现 generic proof bundle schema。
- 不实现 A2A/ARD compatibility。
- 不实现 scheduler-owned routing。

This verifies npm-owned digest metadata, not package signatures or SBOM.
