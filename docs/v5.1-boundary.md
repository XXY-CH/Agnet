# Agent Space v5.1 Boundary

状态：planned
目标：把 signed checkpoint evidence 推进到最小 resume flow。

## v5.1 新增

- Go gateway accepts `FED_TASK_RESUME`。
- Resume request includes `checkpoint_id`。
- Gateway rejects resume without a checkpoint id。
- The resumed task still goes through normal task signature, policy, approval, tool, artifact, receipt, and audit checks。
- The resumed checkpoint records `parent_checkpoint` as the requested checkpoint id。
- The resumed receipt records:
  - `resumed_from`
  - the new checkpoint ref
  - the signed checkpoint object

## 为什么这是 v5.1

v4 created protocol-native checkpoint evidence。

v5.1 is the smallest runtime step after evidence:

```text
signed checkpoint
  -> FED_TASK_RESUME binds to checkpoint_id
  -> new checkpoint links parent_checkpoint
  -> receipt records resumed_from
```

This moves toward long-task pause/resume in `agent-space-ultimate-vision.md` without pretending to implement a durable scheduler。

## v5.1 不新增

- 不恢复 model KV/cache。
- 不做 durable scheduler。
- 不做 retry queue。
- 不做 checkpoint storage service。
- 不做 long-running MCP session resume。
- 不做 artifact merge。

## 验收


The Go integration test must fail if `FED_TASK_RESUME` is unsupported, does not bind to a checkpoint id, or the resumed receipt does not reference the parent checkpoint。
