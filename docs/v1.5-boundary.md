# Agent Space v1.5 Boundary

状态：v1.5 candidate  
目标：增加 CA-like signed capability credential。

## v1.5 新增

- `capabilityCredential(authorityZone, subjectDescriptor, capability, claims)`。
- `verifyCapabilityCredential(credential, authorityDescriptor, subjectDescriptor)`。
- `FED_QUERY_RESULT.matches[].credentials`。
- Client-side credential verification。
- Capability credential primitive test。
- Go verification vector for signed capability credential。

Credential body：

```json
{
  "issuer": "zid:ed25519:...",
  "subject": "aid:ed25519:...",
  "capability": "summarize.text",
  "claims": {
    "level": "L1",
    "evidence": ["zone-b-local-worker"]
  }
}
```

## 为什么这是 v1.5

v1.4 能找到并调用有某个 capability 的 Agent。

v1.5 开始回答：

```text
谁证明它有这个 capability？
证明内容是什么？
证明是否被签名？
证明是否绑定到这个 aid？
```

这类似 CA 证书链，但不是全局信用分。

## v1.5 不新增

- 不做全局 reputation score。
- 不做实时 gossip。
- 不做 credential status feed。
- 不做 revocation propagation。
- 不做多级 authority chain。
- 不做 ranking。

这是 scoped capability credential，不是 reputation system。

后续 Go trust verifier preflight 边界见 `docs/v1.6-boundary.md`。
