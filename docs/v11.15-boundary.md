# Agent Space v11.15 Boundary

状态：complete
目标：Make the Node `FED_SWARM_CLOSE` verifier reject unsafe close step task ids.

## v11.15 新增

- Node `verifySwarmClose` applies the implemented `task_id` token boundary to each close step `task_id`.
- The Node conformance test proves a trusted signed close proof cannot carry a path-like close step task id such as `../bad/task`.
- The `FED_SWARM_CLOSE` portable verifier boundary now aligns step task identity with `FED_TASK_OPEN` task id validation.

## 为什么这是 v11.15

`FED_SWARM_CLOSE` is portable accountability evidence. Even without a Node audit-backed Swarm verifier, its signed step receipts should not be able to name task identities that `FED_TASK_OPEN` would reject before execution. v11.15 closes that structural verifier asymmetry without adding scheduler or audit-log semantics to the Node close verifier.

## v11.15 不新增

- 不实现 Node audit-backed Swarm completeness verification。
- 不验证 step receipts against an audit log。
- 不实现 dynamic Swarm decomposition。
- 不实现 scheduler-owned routing。
- 不实现 parallel or cross-Zone Swarm execution。
- 不实现 public reachability proof。
- 不实现 A2A/ARD compatibility。
