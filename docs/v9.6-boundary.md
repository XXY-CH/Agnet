# Agent Space v9.6 Boundary

状态：complete
目标：Go local JSON state files are replaced atomically instead of overwritten in place.

## v9.6 新增

- `writeJSONStateFile` serializes JSON state and writes through a same-directory temp file.
- The temp file is synced, closed, and renamed over the final state file.
- The parent directory is best-effort synced after rename.
- Task state files now use the shared atomic state writer.
- Approval state files now use the shared atomic state writer.
- Queue state files now use the shared atomic state writer.
- Human Gateway requester registry and requester rebinding history JSON files now use the shared atomic state writer.
- A Go package test verifies repeated state replacement leaves a complete JSON file and no temp file.

## 为什么这是 v9.6

Ultimate needs durable task/approval/queue state before scheduler, restore, and broader runtime coordination are credible.
This slice keeps the current file-per-record shape but removes the most fragile overwrite path:

```text
marshal JSON
  -> same-directory temp file
  -> fsync temp
  -> rename over state file
  -> best-effort fsync directory
```

## v9.6 不新增

- 不做 cross-process locking。
- 不做 database。
- 不做 lockfile protocol。
- 不做 audit append atomic rewrite。
- 不做 artifact byte atomic write。
- 不做 artifact-store `objects.ndjson` compaction or atomic rewrite。
- 不做 queue scheduler or automatic drain。
- 不做 container namespace sandbox。
- 不做 A2A/ARD compatibility。
