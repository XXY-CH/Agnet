# Agent Space v8.19 Boundary

状态：complete
目标：Local artifacts persist their manifest sidecar and expose it through a read-only Human Gateway API.

## v8.19 新增

- `writeArtifact` writes `artifact://local/...` payloads as before and also writes a sibling `.manifest.json` sidecar.
- The sidecar stores the same manifest carried by the `artifact.created` event and signed receipt.
- Human Gateway exposes `GET /api/artifacts/manifest?uri=artifact://local/...` for read-only manifest retrieval.
- Artifact URI path handling rejects unsupported schemes, empty paths, absolute paths, and `..` traversal before reading sidecars.

## 为什么这是 v8.19

Ultimate artifact evidence needs a durable manifest surface that is independent of the transient event stream and receipt JSON.

v8.19 keeps the smallest local version:

```text
artifact bytes
  -> manifest with content digest
  -> .manifest.json sidecar
  -> read-only Human Gateway manifest API
```

This makes artifact verification easier without claiming a remote object store.

## v8.19 不新增

- 不做 content-addressed artifact store。
- 不做 remote artifact download。
- 不做 artifact browser。
- 不做 auth model for read-only artifact manifests。
- 不做 multi-artifact graph。
- 不做 object storage / MinIO / S3。
- 不做 full transcript artifact storage。
- 不做 container sandbox。
- 不做 public deployment。
- 不做 A2A/ARD compatibility。

## 验收

