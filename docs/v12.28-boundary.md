# Agent Space v12.28 Boundary

状态：complete
目标：Make package proof verification bind the displayed filename to the tarball path being verified.

## v12.28 新增

- `asp-verify.mjs package-proof <manifest.json>` requires `filename` to equal the final path segment of `tarball`.
- `package-contract.test.mjs` covers a manifest whose `proof_digest` is self-consistent but whose `filename` does not match the tarball path basename.

## 为什么这是 v12.28

v12.27 made package proof verification return verified metadata. v12.28 closes the next metadata split: a manifest can no longer present one filename while proving bytes from a different tarball path.

## v12.28 不新增

- 不实现 package signing。
- 不实现 SBOM。
- 不发布 npm package。
- 不改变 `package.json` exports/bin/files。
- 不改变 tarball SHA-256 calculation。
- 不改变 package proof verification inputs。
- 不实现 external public reachability proof。
- 不实现 hosted public node。
- 不增加 DNS, TLS, QUIC, NAT traversal, or remote probe infrastructure。
- 不改变 normal `fed-receipt` verification。
- 不改变 `proof-bundle` verifier JSON output。
- 不实现 transport negotiation。
- 不实现 batch verifier。
- 不实现 JSON Schema。
- 不实现 generic proof bundle schema。
- 不实现 A2A/ARD compatibility。
- 不实现 scheduler-owned routing。

This binds existing manifest metadata; it is not package signing or SBOM.
