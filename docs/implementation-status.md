## 一句话

v14.0 opens the next local-first milestone above the v13 proof core: deeper Agent Overlay Network, stronger Agent Swarm Layer, and Multi-signal routing. v14 stays local-first: no P2P DHT, no token economy, no public marketplace, and no production global Agent Net claim.

v14.1 Swarm micro-contract implementation is active in Node and Go: before each Swarm step executes, the selected worker emits a signed `micro_contract: "ok"` commitment with declared token/seconds cost estimate, capability proof, policy digest, contract digest, and worker signature; `FED_SWARM_CLOSE.close.micro_contracts` carries one contract per step and Node `verifySwarmClose` rejects tampered micro-contract signatures.
v14.2 Multi-signal FED_QUERY routing is complete in Node and Go: `discovery_evidence.routing` now exposes `cost_score`, `latency_score`, `availability_score`, and `signals_used`; `agent_score.total` includes those routing scores while missing descriptor or audit evidence stays neutral instead of inventing advantage.
v14.3 Cross-zone trust chains are complete in the local-first proof surface: `zoneTrustDelegation` creates a Zone-signed delegation record, `verifyZoneTrustDelegation` checks it against the trusted local authority descriptor, and FED_QUERY discovery evidence has a `zone_trust_chain` provenance slot where direct membership is represented as `zone_trust_chain: []`.
v14.4 Task failure migration is complete in the local-first Swarm surface: Node `FED_SWARM_OPEN` and Go `FED_SWARM_OPEN` / `FED_SWARM_SCHEDULE` retry a failed step once on a different same-capability worker, and `FED_SWARM_CLOSE.close.migration_log` records `original_worker_aid`, `reason`, `migrated_to_worker_aid`, and `migration_at` inside the signed close body.
v14.5 Intent Decomposition is complete in the local-first Swarm surface: `swarmPlan` emits a signed `FED_SWARM_PLAN` over the original `intent`, ordered step specs, `policy_digest`, and `plan_digest`; `verifySwarmPlan` validates Zone trust, non-empty step shape, NUL-free `step_id`, digest, and signature; `FED_SWARM_CLOSE.close.plan_digest` can link a close proof back to the verified plan digest.
v14.6 Knowledge Gateway Proto is complete in the local-first Node proof surface: `knowledgeQuery` emits signed `FED_KNOWLEDGE_QUERY` frames over `intent`, `sources`, `policy_digest`, `query_id`, and `query_digest`; `knowledgeResponse` emits signed `FED_KNOWLEDGE_RESPONSE` frames with source-cited, freshness-tagged, license-noted results; `verifyKnowledgeResponse` binds responses back to the verified query digest. This is not a web crawler, semantic cache, vector store, or RAG pipeline.
v14.7 Policy and Risk Routing Signals are complete in the local-first Node and Go FED_QUERY surfaces: `discovery_evidence.routing` now exposes `policy_match` and `risk_match` alongside cost/latency/availability, `agent_score.total` includes both new verifier-owned signals with neutral fallback, and `ranking.reasons` can name `policy_match` / `risk_match` when they positively contribute. This is not a new trust oracle, policy language, global risk score, marketplace ranker, or opaque ML router.
v14.8 Swarm Conflict Resolution is complete in the local-first Node and Go Swarm surfaces: duplicate artifact refs with different SHA-256 digests produce signed `conflict_resolutions` close evidence, the deterministic resolver chooses the higher agent_score reputation worker with alias ascending `alias_tiebreak`, and Node `verifySwarmClose` validates candidate-step membership, chosen-worker binding, resolution digest, and Zone signature. This is not voting/quorum, not automatic merge of conflicting content, not payment/settlement, or a remote arbitration layer.
v14.9 Cross-Netns Reachability Evidence is complete in the local-first proof-bundle verifier: trusted `vantage: "cross-netns"` evidence reports `reachability_scope: "cross-netns"` only for a literal private inter-namespace IP, rejects loopback and globally routable IPs, and records the Apple `container` separate-VM experiment at `192.168.64.6/24`. This is not public reachability and does not complete the still-pending v13.1 hosted external-host run.
v14.10 Node FED_SWARM_SCHEDULE ready-DAG parity is complete: Node validates the whole signed Swarm graph before execution, topologically orders ready steps with original input order as the deterministic tie rule, executes serially, and emits signed close `scheduler: { mode: "ready-dag", step_order: [...] }`. Missing dependencies, duplicate step IDs, self-dependencies, and a cycle or unresolvable graph fail before receipts or close. Existing Swarm proof invariants remain intact. This is no parallel execution, no resource scheduling, no economic ranking, no LLM scheduling, and no new trust inputs.
v14.11 Apple Container private-workspace isolation proof is complete on Darwin: one agent-owned private `0700` workspace is mounted read-write at `/work`; staged input remains immutable, result publication is bounded and postflight-verified, and workspace device/inode/owner/mode plus exact entry shape fail closed. The real smoke proves input/root write denial, no default route/DNS, bounded transcript/result handling, overflow non-promotion, and cleanup; postflight rejects symlinks, hardlinks, devices, and unexpected paths. This is not full Docker isolation parity: it makes no Apple tmpfs quota or PID parity claim beyond `nproc=64:64`.
Phase C U19-U30 durable local Swarm is complete for the Go runtime: its same-host filesystem journal under OS process locks is authoritative and materializes replayable views; worker execution is at-least-once, while the fenced signed receipt commitment is exactly-once. It records deterministic parallel ready waves, a byte-stable close, output verification, and an irreversible signed disband. Observed crash/concurrency gates cover journal/view replacement and close/disband append faults, receipt synchronization before response, stale-lease rejection after reclaim, concurrent-coordinator exclusion, and ready-wave barriers. Node is a pure verifier of fixed offline U29 vectors for this durable format. Live public proof excludes durable Swarm completion; Phase C claims no real container smoke, cross-host operation, remote artifact handling, or exactly-once worker execution.
v13.1 reachability evidence gates are active: verifier-owned scope classes and observer evidence binding landed with tests; real hosted external-host evidence is still pending as the remaining v13.1 exit criterion. `external-host` requires a globally routable literal-IP listen host; hostname listen hosts are out of scope for this slice.
`asp-release-trust/v1` remains a scoped release trust proof: not CycloneDX, not SPDX, not SLSA provenance, not npm registry signing, not package publish, not release transparency, and not a generic supply-chain platform.
