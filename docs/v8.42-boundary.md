# Agent Space v8.42 Boundary

状态：complete
目标：Receipt-scoped artifact read responses expose the hash-chained audit entry for the selected signed receipt.

## v8.42 新增

- `GET /api/artifacts/read?task_id=<id>&uri=<artifact-uri>` returns `X-Agent-Space-Audit-Hash`.
- `HEAD /api/artifacts/read?task_id=<id>&uri=<artifact-uri>` returns the same header.
- The hash comes from the existing audit entry found by task-scoped receipt proof lookup.
- Existing receipt artifact verification still runs before returning headers or bytes.
- The Go integration test checks the header on both GET and HEAD responses.

## 为什么这是 v8.42

v8.41 bound artifact read responses to the signed receipt digest.
v8.42 binds those same responses to the audit hash-chain entry that contains that receipt:

```text
receipt task_id + artifact uri
  -> hash-chained audit entry
  -> signed receipt
  -> receipt digest header
  -> verified artifact headers/bytes
```

This is the smallest step from receipt-addressed artifact inspection toward audit-addressed proof without adding a broader audit product.
