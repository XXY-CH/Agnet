# Agent Space v8.1 Boundary

状态：complete
目标：queued drain pauses on Human Gateway approval before tool execution.

## v8.1 新增

- `FED_QUEUE_DRAIN` writes pending approval state when the queued worker policy requires tool approval.
- TCP queued drain waits for explicit `POST /api/approvals/actions` approval before running the tool.
- Human Gateway queue `drain` action also waits for the same explicit approval before returning.
- Approved queued drains emit the existing signed `approval.granted` event and receipt evidence.
- Failed queued drains still require approval before tool/schema execution can fail.

## 为什么这是 v8.1

v8.0 made direct task execution wait for Human Gateway approval.

v8.1 closes the same product-control gap for durable queued execution:

```text
FED_QUEUE_DRAIN / Human Gateway drain action
  -> approval.required
  -> pending approval state
  -> POST /api/approvals/actions approve
  -> signed approval.granted
  -> tool execution
```

This keeps the queue explicit; it does not add automatic scheduling.

## v8.1 不新增

- 不做 approval denial。
- 不做 approval expiry。
- 不做 login/session identity。
- 不做 scheduler auto-drain。
- 不做 browser-side private-key storage。
- 不做 public transport/deployment hardening。

## 验收
