# Agent Space v6.3 Boundary

状态：complete
目标：任务进入运行态后，先持久化最小 running task state。

## v6.3 新增

- Go task execution writes `status: "running"` after `task.started`。
- The running state records:
  - `task_id`
  - `status`
  - `worker`
- Completed/cancelled/failed state overwrites the same task state file。

## 为什么这是 v6.3

v6.0 and v6.1 made terminal states durable。

v6.3 adds the missing non-terminal state:

```text
task.started
  -> running task state
  -> external/MCP execution
  -> terminal task state
```

This gives the future Human Gateway task table a real status source without introducing a scheduler yet。

## v6.3 不新增

- 不做 task state index。
- 不做 Human Gateway task table。
- 不做 scheduler queue。
- 不做 persisted running process handles。
- 不做 distributed cancellation。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 *.test.mjs
```

The Go integration test must fail if a slow running task has no durable `running` state before cancellation。
