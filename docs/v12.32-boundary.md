# Agent Space v12.32 Boundary

状态：complete
目标：Make Go protocol signing, signature verification, and digest paths match Node canonical JSON for signed and digested values containing `<`, `>`, and `&`.

## v12.32 新增

- `agnet/verifier.VerifyFederatedReceipt` verifies a receipt and supplied signed task evidence containing `<>&` when signed/digested with Node canonical bytes.
- `cmd/go-fed-discovery` protocol `signBodyWithKey`, `verifyMapSignature`, and `digestHex` use `json.Encoder.SetEscapeHTML(false)` and trim the encoder newline before signing, verifying, or digesting.
- Focused Go tests cover reusable verifier acceptance and Go runtime signing/digest no-HTML-escape behavior.

## 为什么这是 v12.32

Node canonical JSON does not HTML-escape `<`, `>`, or `&`. Go `json.Marshal` does. That made Go reject otherwise valid Node-signed receipts or supplied signed task evidence containing those characters. v12.32 closes the cross-implementation verification bug before continuing higher-level proof work.

## v12.32 不新增

- 不改变 JSON key ordering beyond Go's existing deterministic encoding.
- 不实现 full RFC/JCS canonicalization。
- 不改变 verifier CLI command shapes。
- 不实现 external public reachability proof。
- 不实现 hosted public node。
- 不实现 package signing。
- 不实现 SBOM。
- 不实现 scheduler-owned routing。
- 不实现 A2A/ARD compatibility。

This is a protocol canonicalization parity fix, not a new proof type.
