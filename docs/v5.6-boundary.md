# Agent Space v5.6 Boundary

状态：complete
目标：把 tool output digest 显式绑定到 receipt 和 artifact manifest。

## v5.6 新增

- Execution receipt records `tool_output_digest`。
- `tool_output_digest` equals the produced artifact manifest `sha256`。
- Audit verifier checks digest alignment when `tool_output_digest` is present。
- Remote audit proof returns the digest because it is part of the signed receipt。

## 为什么这是 v5.6

Artifact manifests already contain hashes, so v5.6 does not add a second artifact system。

It adds the missing link name:

```text
tool output text
  -> tool_output_digest
  -> artifact manifest sha256
  -> signed receipt
  -> remote audit proof
```

This makes the tool-output-to-artifact edge explicit without storing extra content。

## v5.6 不新增

- 不做 multi-artifact output graph。
- 不做 streamed output chunks。
- 不做 content-addressed artifact store。
- 不做 separate tool transcript storage。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 *.test.mjs
```

The Go integration test must fail if the signed receipt omits `tool_output_digest` or if the digest does not match the artifact manifest sha256。
