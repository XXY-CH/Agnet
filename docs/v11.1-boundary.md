# Agent Space v11.1 Boundary

状态：complete
目标：Require requester Zone binding on federated task entry.

## v11.1 新增

- `FED_TASK_OPEN` includes `requester_zone_binding`.
- Node `verifyFederatedTaskOpen` verifies that binding against `origin_zone` and the requester descriptor.
- Go `verifyTaskOpen` verifies the same binding when a federation frame carries `origin_zone`.
- Node and Go federation clients emit requester bindings.
- Go queue state carries requester bindings through claim and drain verification.
- The shared `FED_TASK_OPEN` vector includes a requester Zone binding.

## 为什么这是 v11.1

v11.0 made receipts reject unknown origin Zones. The next smaller provenance gap is task entry: before this slice, a trusted origin Zone could forward any requester descriptor without signing that the requester belongs to that Zone. v11.1 closes that edge by reusing the existing Zone binding primitive.

## v11.1 不新增

- 不实现 DID document/resolver。
- 不实现 remote trust-store sync。
- 不改变 receipt frame requester proof。
- 不实现 public reachability proof。
- 不实现 scheduler-owned routing。
- 不实现 dynamic Swarm decomposition。
- 不实现 A2A/ARD compatibility。
