# Agent Space v12.24 Boundary

状态：complete
目标：Make the package proof verifier reject unsafe package proof tarball paths before reading tarball bytes.

## v12.24 新增

- `asp-verify.mjs package-proof <manifest.json>` rejects unsafe tarball paths with `package proof tarball path invalid`.
- Invalid tarball paths include absolute paths, backslashes, empty paths, `.` segments, and `..` segments.
- `package-contract.test.mjs` covers absolute and parent-directory tarball paths.

## 为什么这是 v12.24

v12.22 added a package proof verifier command and v12.23 made non-object manifests fail closed. v12.24 closes the next byte-read boundary: the verifier checks the package proof tarball path shape before reading package bytes.
