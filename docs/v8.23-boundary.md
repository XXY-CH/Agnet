# Agent Space v8.23 Boundary

状态：complete
目标：Audit verification checks digest-addressed artifact manifest sidecars against signed receipt manifests.

## v8.23 新增

- `--verify-audit` reads each `artifacts/by-sha256/<sha256>.manifest.json` sidecar.
- The verifier rejects digest-addressed sidecars that differ from the receipt's signed artifact manifest.
- The Go integration test tampers with the digest-addressed sidecar after a successful audit verify and expects `--verify-audit` to fail.

## 为什么这是 v8.23

v8.20 wrote content-addressed artifact copies with matching sidecars.
v8.22 verified only the named artifact sidecar.

v8.23 closes the remaining local sidecar verification gap:

```text
receipt artifact manifest
  -> artifacts/by-sha256/<sha256>.manifest.json
  -> digest sidecar equality check
  -> audit verification result
```

## v8.23 不新增

- 不做 remote artifact fetch。
- 不校验 remote object store。
- 不做 artifact GC。
- 不做 artifact browser。
- 不做 full transcript artifact storage。
- 不做 container sandbox。
- 不做 public deployment。
- 不做 A2A/ARD compatibility。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 go-fed-discovery.test.mjs
node --test --test-concurrency=1 *.test.mjs
```

The Go integration test must fail if `--verify-audit` accepts a receipt whose digest-addressed local artifact sidecar no longer matches the signed manifest.
