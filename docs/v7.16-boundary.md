# Agent Space v7.16 Boundary

状态：complete
目标：Human Gateway can draft, locally sign, and enqueue queued work through the existing queue action path.

## v7.16 新增

- `POST /api/queue/drafts` accepts `task_id`, `to`, `intent`, `scope`, and `budget`.
- The Human Gateway creates a local requester descriptor for the draft.
- The drafted task is signed by local gateway-held proof material.
- Draft enqueue reuses the existing `enqueue` queue action grant, actor binding, actor policy, task verification, queue write, and `go_queue_action` audit path.
- `/api/queue` exposes the drafted queued task.

## 为什么这是 v7.16

v7.6 proved the Human Gateway can enqueue an already signed task.

v7.16 closes the missing local creation edge without adding a scheduler or browser key custody:

```text
POST /api/queue/drafts
  -> local requester descriptor
  -> signed task
  -> existing enqueue queue action
  -> durable queued item
```

This is local proof signing, not user account identity or browser-held requester keys.

## v7.16 不新增

- 不做 browser-side private-key storage。
- 不做 login/session identity。
- 不做 requester key management UI。
- 不做 task form builder。
- 不做 automatic scheduler loop。
- 不做 public transport/deployment hardening。
