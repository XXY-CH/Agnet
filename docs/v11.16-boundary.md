# Agent Space v11.16 Boundary

状态：complete
目标：Make the Node `FED_SWARM_CLOSE` verifier reject missing close signatures with a protocol error.

## v11.16 新增

- Node `verifySwarmClose` rejects missing or empty `close_signature` before calling the crypto verifier.
- The Node conformance test proves a structurally valid close body without `close_signature` fails with `swarm close signature missing`, not a low-level runtime type error.
- The portable Swarm close verifier keeps its boundary structural: signing Zone, close signature presence and verification, signed Swarm id, frame/body Swarm id match, and step receipt shape.

## 为什么这是 v11.16

`FED_SWARM_CLOSE` is an externally portable proof frame. A missing required signature field should fail as a protocol-level verifier error, not leak a runtime `Buffer` type error. v11.16 keeps the fix to one verifier guard and one red test.

## v11.16 不新增

- 不实现 Node audit-backed Swarm completeness verification。
- 不验证 step receipts against an audit log。
- 不实现 dynamic Swarm decomposition。
- 不实现 scheduler-owned routing。
- 不实现 parallel or cross-Zone Swarm execution。
- 不实现 public reachability proof。
- 不实现 A2A/ARD compatibility。
