# Agent Space v8.17 Boundary

状态：complete
目标：External/MCP sandbox evidence records the executable binary digest, not only the command vector digest.

## v8.17 新增

- External/MCP sandbox evidence records `tool_binary_digest`.
- The digest is SHA-256 over the executable bytes resolved from `tool_command[0]`.
- `sandbox_proof` signs the same sandbox evidence, so the binary digest is covered by the Zone authority signature.
- Existing `tool_command_digest` remains unchanged.

## 为什么这是 v8.17

v5.5 recorded the command vector, which proves what command was requested.

v8.17 adds the smallest stronger provenance hook:

```text
tool_command[0]
  -> executable bytes
  -> tool_binary_digest
  -> sandbox evidence
  -> signed sandbox_proof
```

This improves verifiable execution provenance without pretending to solve package supply chain trust.

## v8.17 不新增

- 不做 package signature verification。
- 不做 SBOM。
- 不做 dependency provenance。
- 不做 container sandbox。
- 不做 sandbox broker service。
- 不做 public deployment。
- 不做 A2A/ARD compatibility。

## 验收

```bash
go test ./...
```

The Go integration test must fail if External/MCP sandbox evidence omits `tool_binary_digest`, or if the signed sandbox proof does not carry the same digest.
