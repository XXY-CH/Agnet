# Agent Space v3.6 Boundary

状态：v3.6 complete
目标：让 Go worker execution 支持 external stdio tool。

## v3.6 新增

- Worker profile can set:
  - `tool: "external.stdio"`
  - `tool_command`
- Go runs the external tool through a process sandbox envelope:
  - JSON stdin
  - JSON stdout
  - 2 second timeout
  - temporary working directory
  - minimal environment
  - no direct file/network handles passed by Agent Space
- External tool output becomes the task artifact。
- Receipt records `tool: "external.stdio"`。
- Node integration test writes a temporary stdio tool and verifies:
  - selected worker invokes it
  - artifact is external-tool produced
  - receipt records the external tool
  - Human Gateway serves the produced artifact

## 为什么这是 v3.6

v3.5 proved built-in tool adapter routing。

v3.6 proves the next execution boundary: a worker can call an external tool without giving the tool the full Agent Space runtime:

```text
task envelope
  -> constrained JSON stdin
  -> external tool process
  -> JSON stdout
  -> artifact
  -> receipt
```

This is a process sandbox envelope, not an OS/container security boundary。

## v3.6 不新增

- 不做 MCP wire protocol。
- 不做 container sandbox。
- 不做 network isolation enforcement。
- 不做 file capability mounts。
- 不做 long-running tool sessions。
- 不做 approval-gated tools。

## 验收

```bash
node --test --test-concurrency=1 *.test.mjs
go test ./...
```

The Go integration test must fail if the artifact falls back to the built-in tool output。
