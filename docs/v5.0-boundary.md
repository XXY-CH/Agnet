# Agent Space v5.0 Boundary

状态：v5.0 complete
目标：把本地 sandbox evidence 变成可验证、可远程取回的 signed sandbox proof。

## v5.0 新增

- Receipt records `sandbox_proof`。
- `sandbox_proof` includes:
  - `proof_type: "local.sandbox.v1"`
  - `authority`
  - `task_id`
  - `worker`
  - `policy_digest`
  - `sandbox`
  - `sandbox_signature`
- Go gateway signs sandbox proof with the Zone authority key。
- Go audit verifier checks:
  - proof exists
  - proof task/worker/policy matches receipt
  - proof sandbox evidence matches receipt sandbox
  - `sandbox_signature` verifies against Zone authority。
- Node integration verifies sandbox proof signature and confirms remote audit query returns the proof。

## 为什么这是 v5.0

v4.5 made remote audit proof queryable。

v5.0 starts the stronger runtime proof layer:

```text
tool execution
  -> sandbox evidence
  -> Zone-signed sandbox proof
  -> receipt
  -> remote audit query can retrieve proof
```

This moves toward the Trust & Verification layer in `agent-space-ultimate-vision.md` without pretending local temp-dir execution is a real container boundary。

## v5.0 不新增

- 不做 Docker/gVisor/Firecracker。
- 不做 OS-level network blocking。
- 不做 hardware attestation。
- 不做 sandbox broker。
- 不做 long-running MCP session isolation。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 test/*.test.mjs
```

The Go integration test must fail if receipt sandbox proof is missing, unsigned, mismatched, or absent from remote audit query results.
