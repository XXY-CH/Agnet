# Agent Space v13.1 Boundary

状态：active
目标：Land verifier-owned reachability evidence scope derivation while honestly keeping the real hosted external-host run pending.

## v13.1 新增

- `proof-bundle` now owns three reachability scopes: `local-interface`, `container-observer`, and `external-host`.
- Observer evidence binds `vantage`, `observed_host`, `observed_port`, `observed_at`, the same `transport_proof`, the same `receipt_digest`, `reached: true`, observer Zone trust, and the observer signature.
- Valid container observer evidence returns `reachability_scope: "container-observer"` and `reachability_observer_zid`.
- Valid external-host evidence returns `reachability_scope: "external-host"` and `reachability_observer_zid` only when the signed receipt `listen_host` is a globally routable literal IP.
- Negative gates for stale evidence, future evidence, invalid vantage, wrong observed host, wrong observed port, wrong receipt digest, untrusted observer, unsigned evidence, bundle-owned reachability scope, and external-host over non-routable listen hosts have landed with tests.

## 为什么这是 v13.1

v13.0 opened the Ultimate-facing hosted/public reachability gate without claiming new runtime capability. v13.1 lands the verifier-owned mechanism needed to separate local interface evidence, container-observer evidence, and real external-host evidence. It is active, not complete: the mechanism and negative gates are in place, but the hosted external-host observer run has not happened yet and remains the remaining v13.1 exit criterion.

## v13.1 不新增

- 不完成 real hosted external-host observer run; that remains the v13.1 exit criterion.
- 不把 same-host, LAN-only, or container-only observer evidence labeled as hosted public reachability.
- 不把 `container-observer` collapsed into `external-host`; it is a distinct verifier-owned scope.
- 不支持 hostname listen hosts for `external-host`; this slice requires a globally routable literal-IP listen host.
- 不改 protocol tag; the active tag stays `v13.0-protocol` until the slice completes.
- 不做 NAT traversal, relay network, QUIC, DHT, or production deployment automation.

## 硬边界

- v13.1 reachability evidence gates are active: verifier-owned scope classes and observer evidence binding landed with tests; real hosted external-host evidence is still pending.
- `external-host` requires trusted signed observer evidence with `vantage: "external-host"`, matching observed endpoint fields, fresh `observed_at`, the same receipt digest, the same transport proof, and a globally routable literal-IP `listen_host`.
- Hostname listen hosts are out of scope for this slice and are not globally routable external-host evidence.
- `container-observer` proves a containerized observer boundary only; it is not hosted external-host reachability.
- `reachability_observer_zid` is the verifier output field for trusted observer evidence.
