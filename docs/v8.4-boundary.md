# Agent Space v8.4 Boundary

状态：complete
目标：Human Gateway write actions can require a local bearer token before mutation.

## v8.4 新增

- Go gateway accepts `--human-token`.
- When `--human-token` is set, Human Gateway write endpoints require `Authorization: Bearer <token>`.
- The token gate covers:
  - `POST /api/approvals/actions`
  - `POST /api/queue/actions`
  - `POST /api/queue/drafts`
- Read endpoints remain available on the localhost-bound gateway.
- Rejected unauthenticated writes return `401` before mutating queue or approval state.

## 为什么这是 v8.4

v8.0-v8.3 made the Human Gateway a real control surface.

v8.4 adds the smallest deployable-gateway guard before any non-local hardening:

```text
localhost Human Gateway
  -> read state
  -> write requires bearer token when configured
```

This is still not a public deployment story. It is a local write gate that prevents ambient localhost writes when the gateway is exposed through a developer tunnel, proxy, or browser context.

## v8.4 不新增

- 不做 login/session identity。
- 不做 role model。
- 不做 CSRF/session cookies。
- 不做 TLS/QUIC/public transport。
- 不做 token storage or rotation。
- 不开放非 localhost bind。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 go-fed-discovery.test.mjs
node --test --test-concurrency=1 *.test.mjs
```

The Go integration test must fail if `--human-token` is missing, if Human Gateway writes succeed without the bearer token, or if token-protected writes stop using the existing queue/approval verification paths.
