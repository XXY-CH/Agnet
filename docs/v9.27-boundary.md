# Agent Space v9.27 Boundary

状态：complete
目标：Go audit verification rejects signed Swarm receipts whose declared input artifact does not match the upstream Swarm step receipt.

## v9.27 新增

- `--verify-audit` now tracks completed Swarm steps while scanning the same audit log.
- For each receipt with `receipt.swarm`, verifier records the step's first artifact manifest under `swarm_id + step_id`.
- For dependent steps, verifier checks every `input_artifacts` entry against the already completed dependency step:
  - `step_id`
  - `uri`
  - `sha256`
  - `manifest_hash`
- Go package coverage proves a worker-signed downstream receipt with a false upstream artifact digest is rejected.
- The same coverage proves the clean dependency graph still verifies.

## 为什么这是 v9.27

v9.26 signed Swarm dependency evidence into receipts.
v9.27 makes that evidence auditable instead of only signed bytes.

## v9.27 不新增

- 不做跨 audit Swarm proof lookup。
- 不做 remote audit sync。
- 不做 scheduler。
- 不做 dynamic decomposition。
- 不做 parallel DAG execution。
- 不做 conflict resolution。
- 不做 Swarm UI。
- 不做 Node Swarm server。
- 不做 A2A/ARD compatibility。
