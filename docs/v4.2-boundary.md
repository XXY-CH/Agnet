# Agent Space v4.2 Boundary

状态：v4.2 complete
目标：把当前 Go policy subset 固化为 canonical policy scope evidence，并给 policy deny 增加稳定 reason code。

## v4.2 新增

- Receipt records `policy_scope`。
- Receipt records `policy_digest`。
- Checkpoint `policy_digest` matches receipt `policy_digest`。
- Canonical policy scope includes:
  - `network`
  - `write`
  - `tools`
  - `data_domains`
  - `approval_required`
  - `expires_at`
- Policy deny frames include stable `code`。
- Go audit verifier checks policy scope shape and digest。
- Node integration test verifies policy scope, policy digest, checkpoint digest alignment, and network deny code。

## 为什么这是 v4.2

v4.1 proved artifact manifest digest evidence。

v4.2 makes scoped policy part of the evidence chain:

```text
task scope + worker policy
  -> canonical policy_scope
  -> policy_digest
  -> checkpoint + receipt evidence
  -> audit verifier checks digest
```

This strengthens the narrow waist without adding a policy service。

## v4.2 不新增

- 不做 policy admin UI。
- 不做 dynamic policy service。
- 不做 role / tenant model。
- 不做 policy editor。
- 不做 policy negotiation。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 *.test.mjs
```

The Go integration test must fail if policy scope evidence is missing from receipt/checkpoint or policy deny lacks a stable code。
