# Agent Space v6.1 Boundary

状态：complete
目标：任务执行失败且还没有 receipt 时，也持久化最小 task state。

## v6.1 新增

- Failed Go task execution writes one JSON state file per task。
- The state file records:
  - `task_id`
  - `status: "failed"`
  - `worker`
  - `error`

## 为什么这是 v6.1

v6.0 made completed/cancelled tasks durable through receipt-linked state。

v6.1 closes the first failure hole:

```text
task execution error before receipt
  -> failed task state
  -> durable error evidence
```

This gives later retry, cancellation, and Human Gateway status views a failure surface to build on。

## v6.1 不新增

- 不做 failed receipt。
- 不做 retry scheduler。
- 不做 live external/MCP process interruption。
- 不做 checkpoint restore。
- 不做 database。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 *.test.mjs
```

The Go integration test must fail if an execution error returns `FED_TASK_ERROR` without writing failed task state。
