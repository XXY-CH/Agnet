# Agent Space v12.27 Boundary

状态：complete
目标：Make successful package proof verification return the verified npm digest and size metadata.

## v12.27 新增

- `asp-verify.mjs package-proof <manifest.json>` returns verified `size`, `shasum`, and `integrity` fields in its JSON output.
- `test/package-contract.test.mjs` covers those fields in the successful package proof verifier result.

## 为什么这是 v12.27

v12.26 made the verifier check npm digest metadata. v12.27 makes the verifier output carry the same verified metadata so downstream proof consumers do not need to trust or reread the manifest for those fields.

## v12.27 不新增

- 不实现 package signing。
- 不实现 SBOM。
- 不发布 npm package。
- 不改变 `package.json` exports/bin/files。
- 不改变 tarball SHA-256 calculation。
- 不改变 package proof verification inputs。
- 不实现 external public reachability proof。
- 不实现 hosted public node。
- 不增加 DNS, TLS, QUIC, NAT traversal, or remote probe infrastructure。
- 不改变 normal `fed-receipt` verification。
- 不改变 `proof-bundle` verifier JSON output。
- 不实现 transport negotiation。
- 不实现 batch verifier。
- 不实现 JSON Schema。
- 不实现 generic proof bundle schema。
- 不实现 A2A/ARD compatibility。
- 不实现 scheduler-owned routing。

This exposes already-verified metadata; it is not package signing or SBOM.
