# Agent Space v7.11 Boundary

状态：complete
目标：queue action grants cannot be replayed after successful use.

## v7.11 新增

- Successful Human Gateway queue actions consume the signed grant digest.
- A later queue action using the same signed grant is rejected before applying the action.
- Replay failures are still recorded as `go_queue_action` audit records.

## 为什么这是 v7.11

v7.10 bounded queue action grants by action scope and expiry.

v7.11 makes a successful grant one-time within the local audit history:

```text
signed grant
  -> signature / scope / expiry pass
  -> grant digest has no prior successful queue action
  -> apply queue action
  -> audit result with grant_digest
```

## v7.11 不新增

- 不做 user/session identity。
- 不做 grant revocation list。
- 不做 durable nonce index。
- 不做 concurrent replay race hardening。
- 不做 browser-side signing UX。
- 不开放非 localhost。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 go-fed-discovery.test.mjs
node --test --test-concurrency=1 *.test.mjs
```

The Go integration test must fail if a signed queue action grant can be reused after one successful queue action.
