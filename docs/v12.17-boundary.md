# Agent Space v12.17 Boundary

状态：complete
目标：Make public proof bundle verification keep reachability scope verifier-owned.

## v12.17 新增

- `asp-verify.mjs proof-bundle <bundle.json>` rejects bundle manifests that include `reachability_scope`.
- `test/public-node-proof.test.mjs` covers a manifest trying to self-claim `reachability_scope: "external-host"`.
- `docs/asp-core-draft.md` documents `reachability_scope` as verifier-owned output.

## 为什么这是 v12.17

v12.16 labeled the current public proof bundle as `reachability_scope: "local-interface"`, but a bundle manifest could still include a conflicting scope claim that consumers might mistake for evidence. v12.17 rejects that field at the verifier boundary so future `external-host` reachability can only come from verifier-owned outside-host evidence.

## v12.17 不新增

- 不实现 external public reachability proof。
- 不实现 hosted public node。
- 不增加 DNS, TLS, QUIC, NAT traversal, or remote probe infrastructure。
- 不改变 normal `fed-receipt` verification。
- 不改变 successful `proof-bundle` verifier JSON output。
- 不实现 transport negotiation。
- 不实现 batch verifier。
- 不实现 JSON Schema。
- 不实现 generic proof bundle schema。
- 不实现 package signing or SBOM。
- 不实现 A2A/ARD compatibility。
- 不实现 scheduler-owned routing。

It does not prove reachability from another host.
