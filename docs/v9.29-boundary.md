# Agent Space v9.29 Boundary

状态：complete
目标：Swarm dependency evidence binds each signed input artifact to the declared `after` dependency step.

## v9.29 新增

- `--verify-audit` now checks each `swarm.input_artifacts[i].step_id` equals `swarm.after[i]`.
- A downstream Swarm receipt cannot claim it depends on one step while supplying another completed step's artifact and receipt digest.
- Go package coverage proves that a signed step mismatch is rejected even when the substituted step's artifact manifest and receipt digest are otherwise valid.
- Existing clean Swarm dependency coverage still verifies.

## 为什么这是 v9.29

v9.27 checked Swarm dependency artifact manifests.
v9.28 bound those dependencies to upstream receipt digests.
v9.29 closes the remaining local DAG semantic gap: the signed dependency list and the signed input artifact list must name the same upstream steps in the same order.

## v9.29 不新增

- 不做 cross-audit dependency lookup。
- 不做 remote audit sync。
- 不做 scheduler。
- 不做 dynamic decomposition。
- 不做 parallel DAG execution。
