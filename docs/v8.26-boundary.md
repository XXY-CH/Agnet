# Agent Space v8.26 Boundary

状态：complete
目标：Human Gateway direct approval actors pass a configurable local allowlist.

## v8.26 新增

- `--human-actor-policy` also accepts an `approval_actions` map.
- `/api/approvals/actions` rejects approval actors not allowed for `approve` or `deny`.
- Default behavior remains local-only when no approval policy is configured.
- The Go integration test denies `human://guest` approval and allows `human://operator` approval through the configured policy.

Example:

```json
{
  "approval_actions": {
    "human://local": ["approve", "deny"],
    "human://operator": ["approve"]
  }
}
```

## 为什么这是 v8.26

v8.25 preserved named local human actors in signed approval evidence.
v8.26 adds the smallest governance gate above that identity:

```text
human://operator
  -> approval_actions allowlist
  -> approve allowed
  -> Zone-signed approval grant
```

## v8.26 不新增

- 不做 login/session store。
- 不做 roles/admin model。
- 不做 dynamic policy service。
- 不做 hot reload。
- 不做 token storage / rotation。
- 不做 public deployment。
- 不做 A2A/ARD compatibility。
