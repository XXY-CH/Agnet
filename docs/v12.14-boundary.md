# Agent Space v12.14 Boundary

状态：complete
目标：Make `proof-bundle` reject loopback public transport proofs.

## v12.14 新增

- `asp-verify.mjs proof-bundle <bundle.json>` now rejects signed receipt transport proofs whose `listen_host` is `localhost`, `::1`, or an IPv4 `127.*` loopback address.
- A bundle whose signed receipt transport proof uses `listen_host: "127.0.0.1"` while keeping `transport: "fed+tcp"` and `public_transport: true` is rejected with `bundle transport_proof invalid`.
- `test/public-node-proof.test.mjs` proves this by re-signing a receipt with loopback transport evidence and matching the bundle fields.

## 为什么这是 v12.14

v12.13 required the current federation TCP transport label, but a loopback listener can still be signed as `public_transport: true`. This blocks that proof shape before future external reachability work.

## v12.14 不新增

- 不实现 external public reachability proof。
- 不实现 hosted public node。
- 不增加 DNS, TLS, QUIC, NAT traversal, or remote probe infrastructure。
- 不改变 Go receipt transport proof field shape。
- 不改变 normal `fed-receipt` verification。
- 不改变 verifier JSON output。
- 不实现 transport negotiation。
- 不实现 batch verifier。
- 不实现 JSON Schema。
- 不实现 generic proof bundle schema。
- 不实现 package signing or SBOM。
- 不实现 A2A/ARD compatibility。
- 不实现 scheduler-owned routing。

It does not prove reachability from another host.
