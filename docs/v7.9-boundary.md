# Agent Space v7.9 Boundary

状态：complete
目标：Human/local queue actions require a signed action grant.

## v7.9 新增

- Human Gateway `POST /api/queue/actions` rejects actions without `action_grant`.
- Action grant includes:
  - `action`
  - `task_id`
  - `task_digest`
  - `authority`
  - `authority_descriptor`
  - `grant_signature`
- Grant authority descriptor must be self-signed.
- Grant signature must verify with the authority descriptor key.
- Grant must match the requested action and task.
- Enqueue grants also bind the signed task digest.
- `go_queue_action` audit records include `grant_digest` for successful grants.

## 为什么这是 v7.9

v7.8 made local queue actions auditable.

v7.9 adds the smallest authorization proof before any future non-local exposure:

```text
action grant
  -> verify authority descriptor
  -> verify grant signature
  -> apply queue action
  -> audit grant digest
```

## v7.9 不新增

- 不做 login/session identity。
- 不做 browser-side signing UX。
- 不做 persistent user accounts。
- 不要求 remote federation frames use action grants。
- 不开放非 localhost。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 test/go-fed-discovery.test.mjs
node --test --test-concurrency=1 test/*.test.mjs
```

The Go integration test must fail if Human Gateway queue actions work without a grant, if a grant can be reused for a mismatched task/action, or if successful action audit records omit the grant digest.
