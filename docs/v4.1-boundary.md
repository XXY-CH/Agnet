# Agent Space v4.1 Boundary

状态：v4.1 complete
目标：把 task-scoped Git worktree 的输出固化为 signed merge proposal 和 checkpoint evidence。

## v4.1 新增

- Go finalizes changed Git worktree context into a detached commit。
- Go emits `checkpoint.created` when a worktree commit is produced。
- Receipt records:
  - `checkpoints`
  - `merge.mode: "git-merge-proposal"`
  - `merge.base_head`
  - `merge.commit`
  - `merge.diff_sha256`
  - `merge.changed_files`
- Worker signs checkpoint evidence with `checkpoint_signature`。
- Worker signs merge proposal with `merge_signature`。
- Go audit verifier checks checkpoint and merge proposal signatures。
- Human Gateway receipt table shows merge mode。
- Node integration test verifies an MCP tool writes a file, produces a detached commit, and returns signed checkpoint/merge evidence。

## 为什么这是 v4.1

v4 proved task-scoped Git worktree context。

v4.1 makes the worktree output promotable without mutating main:

```text
tool runs in task worktree
  -> file changes are detected
  -> worktree changes become detached commit
  -> checkpoint.created event carries signed commit evidence
  -> receipt carries signed merge proposal
  -> audit verifier checks signatures
```

This is the smallest useful step toward Swarm-safe output promotion。

## v4.1 不新增

- 不自动 merge 到 main。
- 不做 branch queue。
- 不做 conflict resolution。
- 不做 review UI。
- 不做 realtime context sync。
- 不做 long-running checkpoint resume。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 *.test.mjs
```

The Go integration test must fail if a worktree file change does not produce signed checkpoint and merge proposal evidence。
