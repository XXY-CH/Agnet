# Agent Space v11.2 Boundary

状态：complete
目标：Make the Node `FED_SWARM_CLOSE` verifier fail closed on structurally empty close proofs.

## v11.2 新增

- Node `verifySwarmClose` rejects signed close proofs with missing or empty `step_receipts`.
- Node `verifySwarmClose` rejects close steps with missing `step_id`, missing `task_id`, or non-hex 64-byte `receipt_digest`.
- The Node conformance test proves a trusted Zone signature is not enough when the close proof has no step receipts.

## 为什么这是 v11.2

v11.1 tightened task-entry provenance. The next smallest external verifier gap was Swarm close structure: a trusted Zone could sign an empty close body and the Node CLI would return a digest. v11.2 keeps the verifier narrow, but makes signed close proofs carry at least one concrete step receipt.

## v11.2 不新增

- 不实现 Node audit-backed Swarm completeness verification。
- 不验证 step receipts against an audit log。
- 不实现 dynamic Swarm decomposition。
- 不实现 scheduler-owned routing。
- 不实现 parallel or cross-Zone Swarm execution。
- 不实现 public reachability proof。
- 不实现 A2A/ARD compatibility。
