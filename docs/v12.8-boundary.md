# Agent Space v12.8 Boundary

状态：complete
目标：Make the proof bundle verifier reject extra positional CLI arguments.

## v12.8 新增

- `asp-verify.mjs proof-bundle <bundle.json>` now accepts exactly one bundle path argument.
- Extra positional arguments fall through to the existing usage error instead of being ignored.
- `public-node-proof.test.mjs` proves `proof-bundle <bundle.json> extra.json` is rejected.

## 为什么这是 v12.8

v12.7 made proof-file path handling fail closed. The next smallest external-consumption boundary is the CLI contract itself: a verifier command should not silently ignore extra user-supplied proof inputs.

## v12.8 不新增

- 不改变其他 verifier CLI commands。
- 不实现 JSON Schema。
- 不实现 generic proof bundle schema。
- 不实现 artifact byte relocation。
- 不实现 hosted public node。
- 不实现 external public reachability proof。
- 不实现 package publish/signing。
- 不实现 SBOM。
- 不实现 batch verification。
- 不实现 A2A/ARD compatibility。
- 不实现 scheduler-owned routing。

It does not make `proof-bundle` a batch command.
