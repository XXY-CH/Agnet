# Agent Space v12.43 Boundary

状态：complete
目标：Reject malformed package byte metadata before tarball reads.

## v12.43 新增

- `asp-verify.mjs package-proof <manifest.json>` rejects empty or non-string `shasum`, `integrity`, and `sha256` values with `package proof byte metadata invalid`.
- It also rejects missing, negative, or non-integer `size` values with the same error.
- Byte metadata shape validation runs before reading tarball bytes, so missing files cannot mask malformed signed metadata.
- `package-contract.test.mjs` covers malformed byte metadata rejection before tarball reads.

## 为什么这是 v12.43

The package proof manifest signs npm byte metadata. That metadata must be scalar before the verifier reads package bytes and compares digest values; otherwise filesystem errors can mask malformed signed proof claims.

## v12.43 不新增

- 不实现 JSON Schema。
- 不实现 digest regex preflight。
- 不实现 tarball member proof。
- 不实现 npm registry signing。
- 不实现 release transparency。
- 不实现 SBOM。
- 不实现 package publish。
- 不实现 hosted public node。
- 不实现 real outside-host reachability。
- 不实现 scheduler-owned routing。
- 不实现 A2A/ARD compatibility。

This is package byte metadata shape validation for the local package proof verifier, not a generic package release trust system.
