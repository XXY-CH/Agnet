# Agent Space v11.64 Boundary

状态：complete
目标：Make Go audit-backed Swarm close proof verification reject malformed step receipt list entries instead of silently filtering them.

## v11.64 新增

- Go `verifySwarmCloseProof` rejects non-object entries in signed close `step_receipts` lists with `swarm close step receipt invalid`.
- Existing close step identity, duplicate, count, order, task id, receipt digest, duplicate close, unknown Swarm, and NUL identity checks stay unchanged.
- The fix stays on the signed Swarm close proof surface: close step lists are validated before count, order, task, and digest checks, but this slice does not add generic Swarm close schema validation.

## 为什么这是 v11.64

v11.63 made signed Swarm dependency lists fail closed before dependency proof matching. The audit-backed close proof path still used the broad `mapsFromAny` helper for signed `step_receipts`, so a close proof could include malformed step entries that were silently filtered before the verifier counted and matched the completed audit steps.

## v11.64 不新增

- 不实现 dynamic Swarm decomposition。
- 不实现 scheduler-owned routing。
- 不实现 parallel Swarm execution。
- 不实现 cross-Zone Swarm execution。
- 不实现 Node audit-backed Swarm completeness verification。
- 不实现 generic Swarm close schema validation。
- 不实现 A2A/ARD compatibility。
