# Agent Space v4.5 Boundary

状态：v4.5 complete
目标：让 federation requester 可以按 task id 拉取远端最小 audit proof。

## v4.5 新增

- `FED_AUDIT_QUERY` / `FED_AUDIT_RESULT` / `FED_AUDIT_CLOSE` frames。
- Node CLI:
  - `node federation-gateway.mjs audit <port> <trusted-zones.json> <task_id>`
- Go gateway handles authenticated `FED_AUDIT_QUERY`。
- Go gateway scans local audit JSONL for matching `go_fed_receipt`。
- `FED_AUDIT_RESULT` reuses `FED_RECEIPT` proof shape:
  - `zone`
  - `worker`
  - `zone_binding`
  - signed `receipt`
- The receipt proof includes checkpoint refs/checkpoints and artifact manifests from prior v4 evidence work。
- Node verifier checks remote Zone trust, worker binding, receipt task id, and receipt signature。
- Integration test covers CLI audit query and raw TCP frame query。

## 为什么这是 v4.5

v4.4 proved authenticated session transport binding。

v4.5 makes execution evidence remotely retrievable:

```text
task execution
  -> signed receipt in hash-chained audit log
  -> FED_AUDIT_QUERY(task_id)
  -> trusted receipt proof
```

This stays on the `agent-space-ultimate-vision.md` bottom layer: audit receipt, artifact reference, checkpoint evidence, and federation。

## v4.5 不新增

- 不做完整 audit log 同步。
- 不做跨 Zone 搜索。
- 不做 product audit viewer。
- 不做 pagination/indexing。
- 不做 public audit service。
