# Agent Space v9.5 Boundary

状态：complete
目标：artifact-store can apply the GC plan to delete orphaned mirror objects.

## v9.5 新增

- `--artifact-store-gc-apply --artifact-store <dir> --audit <file>` applies the verified GC plan.
- The command deletes orphaned `by-sha256/<sha256>` mirror bytes.
- The command deletes orphaned `by-sha256/<sha256>.manifest.json` sidecars.
- The Go integration test creates an orphan mirror object, applies GC, and verifies referenced artifacts remain.

## 为什么这是 v9.5

v9.1 made orphan detection read-only.
v9.5 adds the smallest destructive lifecycle step behind an explicit apply flag:

```text
verified GC plan
  -> orphan sha256
  -> delete mirror bytes + sidecar
```

## v9.5 不新增

- 不 compact or rewrite `objects.ndjson`。
- 不做 retention policy。
- 不删除 local `artifacts/` named outputs。
- 不做 remote object-store API。
- 不做 S3/MinIO backend。
- 不做 artifact browser。
- 不做 auth model for mirrored artifacts。
- 不做 container namespace sandbox。
- 不做 A2A/ARD compatibility。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 go-fed-discovery.test.mjs
node --test --test-concurrency=1 *.test.mjs
```

The Go integration test must fail if GC apply does not delete orphan mirror bytes and sidecars or if it deletes referenced mirror artifacts.
