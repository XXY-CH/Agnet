# Agent Space v12.15 Boundary

状态：complete
目标：Make `proof-bundle` and Go public transport status reject unspecified listener hosts.

## v12.15 新增

- Go `isPublicListenHost` now rejects `0.0.0.0` and `::`.
- `scripts/public-node-proof.mjs` chooses the first non-loopback IPv4 address for the proof gateway instead of binding to `0.0.0.0`.
- `asp-verify.mjs proof-bundle <bundle.json>` rejects signed receipt transport proofs whose `listen_host` is `0.0.0.0` or `::`.
- `test/public-node-proof.test.mjs` proves this by re-signing a receipt with `listen_host: "0.0.0.0"` and matching the bundle fields.

## 为什么这是 v12.15

v12.14 rejected loopback listener evidence, but an unspecified bind address is still not an externally meaningful endpoint. This blocks that overclaim while keeping real external reachability as a future evidence slice.

## v12.15 不新增

- 不实现 external public reachability proof。
- 不实现 hosted public node。
- 不增加 DNS, TLS, QUIC, NAT traversal, or remote probe infrastructure。
- 不改变 normal `fed-receipt` verification。
- 不改变 verifier JSON output。
- 不实现 transport negotiation。
- 不实现 batch verifier。
- 不实现 JSON Schema。
- 不实现 generic proof bundle schema。
- 不实现 package signing or SBOM。
- 不实现 A2A/ARD compatibility。
- 不实现 scheduler-owned routing。

It does not prove reachability from another host.
