# Agent Space v11.5 Boundary

状态：complete
目标：Let external verifiers check a receipt's `task_digest` against supplied signed task evidence.

## v11.5 新增

- Node `verifyFederatedReceipt` accepts optional signed task evidence and rejects `task_digest` mismatches.
- `asp-verify.mjs fed-receipt` accepts an optional task JSON file after the trusted-Zones file.
- Go `agnet/verifier.VerifyFederatedReceipt` accepts optional signed task evidence and rejects `task_digest` mismatches.
- Node and Go tests prove mismatched supplied task evidence fails closed.

## 为什么这是 v11.5

v11.4 made `task_digest` mandatory, but still left external verifiers unable to check that digest when they already had the signed task body. v11.5 adds that optional check at the verifier boundary without introducing storage, search, or a new protocol object.

## v11.5 不新增

- 不实现 task store/search。
- 不要求 receipt permanently embed raw task bodies。
- 不改变 existing receipt files without task evidence。
- 不实现 batch verifier。
- 不实现 scheduler-owned routing。
- 不实现 dynamic Swarm decomposition。
- 不实现 A2A/ARD compatibility。
