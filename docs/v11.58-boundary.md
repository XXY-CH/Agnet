# Agent Space v11.58 Boundary

状态：complete
目标：Make Go filesystem artifact mirror index readers reject present media types that are not strings.

## v11.58 新增

- Go `readArtifactStoreIndex` rejects present `media_type` fields that are not strings with `artifact mirror index invalid`.
- Focused Go test proves an index row with `media_type: {"type":"text/plain"}` no longer reads as a valid orphan/index row.
- The fix stays on the mirror-index proof input surface: media type is validated when present, but this slice does not make it required and does not add full mirror index schema validation.

## 为什么这是 v11.58

Receipt and audit artifact manifest verification already reject non-string `media_type` values before accepting signed metadata. Mirror index rows can also carry media type proof metadata into artifact-store verification and GC outputs, so present index metadata should follow the same shape rule at the reader boundary.

## v11.58 不新增

- 不改变 artifact manifest metadata schema。
- 不实现 generic JSON Schema validation。
- 不改变 receipt artifact manifest comparison semantics。
- 不实现 remote artifact fetch。
- 不实现 object-store artifact backend。
- 不实现 artifact retention policy。
- 不实现 scheduler-owned routing。
- 不实现 dynamic Swarm decomposition。
- 不实现 A2A/ARD compatibility。
