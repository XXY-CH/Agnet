# Agent Space v12.6 Boundary

状态：complete
目标：Make proof bundle verification reject non-object bundle manifests before reading proof fields.

## v12.6 新增

- `asp-verify.mjs proof-bundle <bundle.json>` rejects `null` or array bundle manifests with `bundle manifest invalid`.
- The manifest object check runs before the proof type check and before proof-file path reads.
- `public-node-proof.test.mjs` proves `null` and `[]` bundle files fail closed.

## 为什么这是 v12.6

v12.5 made unsupported proof types fail before path interpretation. The next smallest external-consumption boundary is the bundle manifest object itself: malformed JSON values should not leak JavaScript property-read errors or be interpreted as proof bundles.

## v12.6 不新增

- 不实现 JSON Schema。
- 不实现 generic proof bundle schema。
- 不实现 artifact byte relocation。
- 不实现 hosted public node。
- 不实现 external public reachability proof。
- 不实现 package publish/signing。
- 不实现 SBOM。
- 不实现 batch verification。
- 不实现 A2A/ARD compatibility。
- 不实现 scheduler-owned routing。

It does not validate every bundle field shape.
