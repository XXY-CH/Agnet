# Agent Space v3.1 Boundary

状态：v3.1 complete
目标：让 Go task execution path 产生并验证 audit evidence。

## v3.1 新增

- Go discovery/runtime accepts:
  - `--audit`
  - `--verify-audit`
- Go writes audit JSONL entries for:
  - `go_fed_event`
  - `go_fed_receipt`
- Audit entries are hash chained:
  - `prev_hash`
  - `record`
  - `hash`
- Go verifier checks:
  - audit hash chain
  - receipt Zone descriptor
  - receipt worker descriptor
  - receipt Zone binding
  - receipt signature
- Node integration test runs the Go verifier after Node verifies the Go receipt.

## 为什么这是 v3.1

v3 made Go execute a minimal federated task。

v3.1 makes that execution auditable:

```text
Go executes task
  -> appends event audit records
  -> appends receipt audit record
  -> verifier replays hash chain
  -> verifier checks receipt signature and bindings
```

This keeps the proof local and boring: JSONL, SHA-256, Ed25519, no database。

## v3.1 不新增

- 不做 audit server。
- 不做 encrypted audit store。
- 不做 remote audit query。
- 不做 multi-worker registry。
- 不做 WebSocket/QUIC transport。
- 不做 real tool execution。

./state/go-fed-discovery-test --verify-audit --audit state/go-fed-discovery-audit.log
