# Agent Space v8.50 Boundary

状态：complete
目标：Human Gateway running task rows can load live transcript snapshots into the local transcript viewer.

## v8.50 新增

- Running task rows render a `live transcript` button.
- `loadLiveTranscript(taskID)` fetches `/api/transcripts/live?task_id=<id>`.
- The existing `transcript-viewer` displays the returned NDJSON snapshot text.
- The Go integration test checks the running page exposes the live transcript action before cancellation.

## 为什么这是 v8.50

v8.49 exposed the running stdout snapshot API.
v8.50 makes that local proof reachable from the Human Gateway page without adding a streaming protocol:

```text
running task row
  -> live transcript button
  -> /api/transcripts/live?task_id=<id>
  -> transcript-viewer
```

## v8.50 不新增

- 不做 automatic polling。
- 不做 SSE/WebSocket continuous tail。
- 不做 MCP live transcript snapshots。
- 不做 signed receipt proof for incomplete running output。
- 不做 transcript search/index。
- 不做 object-store backend。
- 不做 artifact GC。
- 不做 auth model for transcript reads。
- 不做 container namespace sandbox。
- 不做 public deployment。
- 不做 A2A/ARD compatibility。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 test/go-fed-discovery.test.mjs
node --test --test-concurrency=1 test/*.test.mjs
```

The Go integration test must fail if a running task page stops exposing the live transcript snapshot action.
