# Agent Space v8.16 Boundary

状态：complete
目标：Human Gateway queue action actor policy can be configured from a local JSON file instead of being hardcoded to only `human://local`.

## v8.16 新增

- Go gateway accepts `--human-actor-policy`.
- The policy file uses a minimal JSON shape:

```json
{
  "queue_actions": {
    "human://local": ["enqueue", "claim", "drain"],
    "human://guest": ["enqueue"]
  }
}
```

- Queue action grant verification checks actor/action against the configured policy.
- If no policy file is provided, the previous default remains: `human://local` may `enqueue`, `claim`, and `drain`.
- `go_queue_action` audit evidence records `actor_policy_result: "allow"` for actors allowed by the configured policy.

## 为什么这是 v8.16

Ultimate needs task-scoped and context-scoped authorization to become explicit governance, not a hidden constant.

v8.16 keeps the smallest useful version:

```text
signed queue action grant
  -> actor matches grant
  -> local JSON actor policy allows action
  -> action proceeds
```

## v8.16 不新增

- 不做 dynamic policy service。
- 不做 role model。
- 不做 policy UI。
- 不做 policy hot reload。
- 不做 login/session identity。
- 不做 public deployment。
- 不做 A2A/ARD compatibility。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 go-fed-discovery.test.mjs
node --test --test-concurrency=1 *.test.mjs
```

The Go integration test must fail if `--human-actor-policy` is ignored, if a configured guest actor cannot run an allowed queue action, or if denied actions stop recording the policy denial evidence.
