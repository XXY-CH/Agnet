# Agent Space v8.27 Boundary

状态：complete
目标：Local artifacts can be mirrored to a configured filesystem artifact store.

## v8.27 新增

- `--artifact-store <dir>` enables an optional filesystem mirror.
- Artifact bytes are mirrored to `<dir>/by-sha256/<sha256>`.
- Artifact manifest sidecars are mirrored to `<dir>/by-sha256/<sha256>.manifest.json`.
- The existing local named artifact path and `artifacts/by-sha256/<sha256>` copy remain unchanged.
- The Go integration test verifies mirrored summary artifact bytes/sidecar and mirrored transcript artifact bytes/sidecar.

## 为什么这是 v8.27

v8.20-v8.24 made artifacts content-addressed, sidecar-backed, verified, and transcript-aware.
v8.27 adds the smallest storage boundary above local files:

```text
artifact://local/<task>/...
  -> artifacts/by-sha256/<sha256>
  -> --artifact-store/by-sha256/<sha256>
```

This is a real step toward remote artifact storage without pretending a filesystem mirror is S3, MinIO, or a distributed object store.

## v8.27 不新增

- 不做 S3/MinIO/object-store API。
- 不做 remote artifact fetch/download。
- 不做 artifact GC。
- 不做 artifact browser。
- 不做 auth model for mirrored artifacts。
- 不做 container sandbox。
- 不做 public deployment。
- 不做 A2A/ARD compatibility。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 go-fed-discovery.test.mjs
node --test --test-concurrency=1 *.test.mjs
```

The Go integration test must fail if `--artifact-store` does not mirror artifact bytes and sidecars.
