# Agent Space v6.4 Boundary

状态：complete
目标：Human Gateway 只读暴露 durable task state。

## v6.4 新增

- Human Gateway serves `GET /api/tasks`。
- `/api/tasks` returns task state JSON files from the audit-derived task state directory。
- Human Gateway page renders a read-only Tasks table with:
  - task id
  - status
  - worker
  - receipt digest
  - error
- Task rows are sorted by `task_id` for stable inspection。

## 为什么这是 v6.4

v6.0-v6.3 made task state durable for completed, cancelled, failed, and running tasks。

v6.4 makes that state visible to a human without adding a control surface:

```text
task state files
  -> /api/tasks
  -> Human Gateway Tasks table
```

This is the first Human Gateway task view backed by runtime state, not receipt/audit inference。

## v6.4 不新增

- 不做 task action buttons。
- 不做 Human Gateway cancellation flow。
- 不做 task creation UI。
- 不做 scheduler queue。
- 不做 database/index。
- 不做 polling or live updates。
