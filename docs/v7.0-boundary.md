# Agent Space v7.0 Boundary

状态：complete
目标：把已签名任务先持久化进本地 scheduler queue，而不是只能同步执行。

## v7.0 新增

- Go gateway accepts `FED_TASK_ENQUEUE`。
- Enqueue reuses the existing requester descriptor, task signature, target worker, and policy checks。
- Accepted queue items are written under the audit-derived queue directory。
- Queue item records:
  - `task_id`
  - `status: "queued"`
  - `worker`
  - `origin_zone`
  - `task_digest`
- Gateway returns `FED_QUEUE_ACCEPTED` followed by `FED_QUEUE_CLOSE`。

## 为什么这是 v7.0

v6 made task runtime state durable and visible。

v7 starts task lifecycle ownership:

```text
signed task
  -> policy check
  -> durable local queue item
  -> future scheduler drain
```

This is the smallest scheduler boundary that changes the architecture: tasks can now exist before execution。

## v7.0 不新增

- 不做 queue drain worker。
- 不做 leases。
- 不做 retry/backoff。
- 不做 priority。
- 不做 Human Gateway enqueue button。
- 不做 multi-node scheduler。
