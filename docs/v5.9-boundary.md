# Agent Space v5.9 Boundary

状态：complete
目标：把 MCP tools/list metadata 摘要绑定进 sandbox evidence。

## v5.9 新增

- MCP adapter calls `tools/list` once before `tools/call`。
- Sandbox evidence records:
  - `mcp_tools_count`
  - `mcp_tools_digest`
- `sandbox_proof` signs this metadata because it signs sandbox evidence。

## 为什么这是 v5.9

v5.8 records resources/prompts surface。

v5.9 records the tool surface actually exposed before selecting a tool:

```text
tools/list
  -> count + digest
  -> sandbox evidence
  -> signed sandbox_proof
  -> receipt / remote audit proof
```

This makes tool surface drift auditable without creating a registry。

## v5.9 不新增

- 不保存完整 tools catalog。
- 不验证 tool input schema。
- 不做 tool registry。
- 不做 tool selection/ranking。
- 不做 long-running MCP session reuse。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 *.test.mjs
```

The Go integration test must fail if tools/list metadata is absent from sandbox evidence or not covered by the signed sandbox proof。
