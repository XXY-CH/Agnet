# Agent Space v2.4 Boundary

状态：v2.4 complete
目标：让 Go discovery gateway 验证 `FED_TASK_OPEN`。

## v2.4 新增

- Go discovery gateway supports verification-only `FED_TASK_OPEN`。
- Go verifies:
  - trusted origin Zone descriptor
  - requester descriptor self-signature
  - task `from` matches requester `aid`
  - task `to` matches local worker alias
  - task signature by requester key
  - local worker policy for task scope
- Successful verification returns:
  - `FED_TASK_VERIFIED`
  - `FED_TASK_CLOSE`
- Rejected verification returns `FED_TASK_ERROR`。

## 为什么这是 v2.4

v2.3 已经让 Go discovery gateway own signing keys。

v2.4 starts moving task handoff trust checks into Go without pretending Go can execute tasks:

```text
Node requester sends FED_TASK_OPEN
  -> Go verifies origin/requester/task/policy
  -> Go returns FED_TASK_VERIFIED
  -> no execution, receipt, artifact, or audit write
```

这证明 Go 可以判断一个 federated task-open frame 是否可接受，但任务执行仍属于后续边界。

## v2.4 不新增

- 不做 Go task execution。
- 不做 Go worker event stream。
- 不做 Go receipt signing。
- 不做 Go audit writer。
- 不做 key rotation。
- 不做 credential status feed。
