# Agent Space v8.0 Boundary

状态：complete
目标：direct tool task execution pauses on Human Gateway approval instead of auto-granting before tool execution.

## v8.0 新增

- Direct `FED_TASK_OPEN`, `FED_TASK_RESUME`, and `FED_TASK_RETRY` executions write pending approval state when worker policy requires tool approval.
- Human Gateway serves `GET /api/approvals`.
- Human Gateway serves `POST /api/approvals/actions` with local `approve`.
- Approved direct tasks continue execution and emit the existing signed `approval.granted` event.
- The Human Gateway page renders an Approvals table.

## 为什么这是 v8.0

v3.9 made approval evidence signed, but it was still automatically granted.

v8.0 makes the first product-control boundary real for direct task execution:

```text
approval.required
  -> pending approval state
  -> POST /api/approvals/actions approve
  -> signed approval.granted
  -> task.started
```

This is still a local operator proof, not login-backed user identity.

## v8.0 不新增

- 不阻塞 queued drain approval。
- 不做 approval denial。
- 不做 approval expiry。
- 不做 browser-side private-key storage。
- 不做 login/session identity。
- 不做 public transport/deployment hardening。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 test/go-fed-discovery.test.mjs
node --test --test-concurrency=1 test/*.test.mjs
```

The Go integration test must fail if a direct approval-gated task can run before `/api/approvals/actions` approves it, if pending approval state is not visible through `/api/approvals`, or if the final receipt loses the signed approval grant.
