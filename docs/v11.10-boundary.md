# Agent Space v11.10 Boundary

状态：complete
目标：Make `FED_RECEIPT` verifiers reject receipt-shaped frames with the wrong protocol type.

## v11.10 新增

- Node `verifyFederatedReceipt` now requires `frame.type === "FED_RECEIPT"`.
- Go `agnet/verifier.VerifyFederatedReceipt` now requires `frame["type"] == "FED_RECEIPT"`.
- Node and Go tests prove a `FED_TASK_OPEN` frame carrying otherwise valid receipt fields fails closed with `expected FED_RECEIPT frame`.

## 为什么这是 v11.10

The receipt verifiers already checked trust, signatures, task digests, and artifact manifests, but they did not assert the protocol frame type. v11.10 closes that trust-boundary ambiguity at the shared verifier entry points.

## v11.10 不新增

- 不改变 `FED_RECEIPT` frame schema。
- 不实现 receipt store/search。
- 不实现 batch verifier。
- 不改变 interop request semantics。
- 不实现 scheduler-owned routing。
- 不实现 dynamic Swarm decomposition。
- 不实现 A2A/ARD compatibility。
