# Agent Space v4.4 Boundary

状态：v4.4 complete
目标：让 federation transport 先完成 authenticated session handshake，再接受业务帧。

## v4.4 新增

- `HELLO` / `AUTH` / `AUTH_OK` frames。
- Server-generated `session_id` and challenge。
- `AUTH` signs:
  - `session_id`
  - `challenge`
  - `peer_zid`
  - `remote_zid`
- Node federation client verifies the remote Zone descriptor before signing `AUTH`。
- Node and Go gateways reject business frames before session authentication。
- Session binds later `origin_zone.zid` to the authenticated peer。
- Go raw TCP and raw WebSocket test helpers perform the same protocol handshake before `FED_TASK_OPEN` / `FED_QUERY`。

## 为什么这是 v4.4

v4.3 proved credential status evidence。

v4.4 moves transport one layer down:

```text
trusted Zone descriptor
  -> HELLO challenge
  -> Zone-signed AUTH
  -> authenticated session
  -> federation frames
```

This keeps the v4 route on the bottom protocol layer from `agent-space-ultimate-vision.md`: identity, trust, task fabric, and federation, without building public transport.

## v4.4 不新增

- 不做 TLS/QUIC。
- 不做 NAT/public gateway。
- 不做 OAuth。
- 不做 long-lived session resumption。
- 不做 encrypted payloads。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 test/*.test.mjs
```

The integration tests must fail if CLI federation or raw WebSocket federation bypasses the session handshake.
