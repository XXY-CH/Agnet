# Agent Space v9.25 Boundary

状态：complete
目标：Node and Go verify the same static `FED_RECEIPT` conformance fixture.

## v9.25 新增

- `test-vectors/asp-v9.25-fed-receipt.json` captures a signed `FED_RECEIPT` frame with deterministic Zone, worker, zone binding, receipt body, and expected canonical receipt body.
- Node exports `verifyFederatedReceipt(...)` from `asp-core.mjs`.
- Node federation client/audit receipt verification reuses the shared receipt verifier.
- Go package coverage verifies the same fixture through the existing `verifyInteropReceipt` path.
- Node vector coverage verifies the same fixture through `verifyFederatedReceipt`.

## 为什么这是 v9.25

v9.24 made the signed `FED_TASK_OPEN` assumption reusable as the first cross-implementation fixture.
v9.25 adds the matching receipt-side fixture, covering the other half of task interop without expanding into a full conformance suite.
