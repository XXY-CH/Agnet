# Agent Space v12.9 Boundary

状态：complete
目标：Make proof bundle CLI arity count positional arguments exactly.

## v12.9 新增

- `asp-verify.mjs proof-bundle <bundle.json>` now requires exactly two CLI tokens after the script name: `proof-bundle` and one bundle path.
- Empty-string extra positional arguments are rejected by the same usage path as non-empty extras.
- `public-node-proof.test.mjs` proves `proof-bundle <bundle.json> ""` is rejected.

## 为什么这是 v12.9

v12.8 made obvious extra arguments fail closed. The next smallest CLI boundary is the empty-string extra argument, which previously bypassed the truthiness check and was silently ignored.

## v12.9 不新增

- 不改变其他 verifier CLI commands。
- 不实现 batch verification。
- 不实现 JSON Schema。
- 不实现 generic proof bundle schema。
- 不实现 artifact byte relocation。
- 不实现 hosted public node。
- 不实现 external public reachability proof。
- 不实现 A2A/ARD compatibility。
- 不实现 scheduler-owned routing。

It does not add option parsing.
