# v13.8 Boundary: Pinned External Observer Identity

状态：complete

目标：让 external reachability observer 能用 pre-pinned trusted observer identity 签名，为真实 hosted external-host run 做信任根准备。

## What Landed

- `scripts/external-reachability-observer.mjs` accepts `AGNET_REACHABILITY_OBSERVER_SEED_HEX`.
- The seed must be exactly 32 bytes of hex.
- With the same seed, the observer derives the same Zone descriptor and `observer_zid`.
- The signed `external-reachability` evidence and emitted trusted-Zone file use that stable observer identity.
- `public-node-proof.test.mjs` proves the observer output and trusted descriptor match the expected pre-pinned observer identity.

## Non-Goals

- This does not complete hosted/public reachability.
- The real hosted external-host observer run is still pending.
- This does not make container-observer evidence count as hosted external-host evidence.
- This does not add NAT traversal, relay networking, QUIC, DHT, or deployment automation.
- This does not support hostname `external-host` targets; the current verifier still requires a globally routable literal-IP `listen_host`.

## Why This Boundary

The verifier already rejects untrusted observer evidence, but the demo observer previously generated a fresh Zone while writing its own trusted-Zone file. That is acceptable for local proof demos and weak for a hosted proof. A hosted observer needs a stable identity that can be pinned before the run, so trust is not invented after the observation.

v13.8 closes that support gap with the smallest possible change: deterministic observer identity from an explicit seed. The remaining v13.1 exit criterion is still the real hosted external-host observer run against a globally routable literal-IP listener.

## Test Contract

The tests must fail if:

- `AGNET_REACHABILITY_OBSERVER_SEED_HEX` is ignored.
- The observer emits a different `observer_zid` for the same seed.
- The emitted trusted-Zone descriptor does not match the expected pre-pinned observer identity.
- Public docs claim the hosted external-host run has finished.
