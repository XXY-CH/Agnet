# Agent Space v0.2 Boundary

状态：v0.2 candidate  
目标：让 registry entry 从“可信配置”降级为“可验证 descriptor”。

## v0.2 新增

- Agent Descriptor 自签名。
- `resolveAgent()` 必须验证 descriptor signature。
- 修改 descriptor 的 policy、transport、alias、aid 或 public key 后，签名验证失败。
- Worker 收到 `TASK_OPEN` 时验证 requester descriptor。
- Worker 检查 task `from` 必须等于 requester descriptor 的 `aid`。
- 新增篡改测试。

## 为什么这是 v0.2

v0.1 证明 `aid` 和 task/receipt signature 跨语言稳定。

v0.2 证明 registry 不是身份可信根：

```text
registry can route descriptor
registry cannot silently change descriptor
```

Registry 仍然能拒绝、删除或替换 alias 记录，但替换后会变成另一个可验证 Agent，而不是伪装成原 Agent。

后续 hash-chained audit 边界见 `docs/v0.3-boundary.md`。

## v0.2 不新增

- 不做 Zone 签名。
- 不做 DID document。
- 不做撤销列表。
- 不做密钥轮换。
- 不做 federation trust。

这些留给 Zone/Federation 阶段。
