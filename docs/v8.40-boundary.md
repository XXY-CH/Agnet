# Agent Space v8.40 Boundary

状态：complete
目标：Receipt-scoped artifact reads support HEAD proof headers without returning bytes.

## v8.40 新增

- `HEAD /api/artifacts/read?task_id=<id>&uri=<artifact-uri>` runs the same task-scoped receipt and artifact verification path as `GET`.
- The response returns `Content-Type` from the signed receipt artifact manifest.
- The response returns `Content-Length` from the signed receipt artifact manifest size.
- The response returns `X-Agent-Space-Artifact-SHA256`.
- The response returns `X-Agent-Space-Artifact-Manifest-Hash`.
- The response body is empty.
- The Go integration test checks the HEAD response headers and empty body.

## 为什么这是 v8.40

v8.39 exposed digest headers on receipt-scoped artifact reads.
v8.40 makes that proof accessible without downloading the artifact bytes:

```text
receipt task_id + artifact uri
  -> signed receipt manifest
  -> existing artifact verifier
  -> HEAD proof headers
```

This is the smallest useful inspection step before any larger artifact browser or cache work.

## v8.40 不新增

- 不做 artifact browser。
- 不做 transcript browser。
- 不做 remote artifact fetch/download。
- 不做 object-store backend。
- 不做 artifact GC。
- 不做 auth model for artifact reads。
- 不做 HTTP cache protocol。
- 不做 range requests。
- 不做 streamed transcript UI。
- 不做 container namespace sandbox。
- 不做 public deployment。
- 不做 A2A/ARD compatibility。
