# Agent Space v7.15 Boundary

状态：complete
目标：queued checkpoint resume records restored checkpoint state digest.

## v7.15 新增

- `FED_TASK_RESUME` and queued `FED_QUEUE_RESUME` drain look up the parent checkpoint record from audit, not just its id.
- Resumed receipts record `restored_state_digest` from the parent checkpoint.
- Resumed checkpoints record the same `restored_state_digest` alongside `parent_checkpoint`.
- The Go integration test verifies queued resume drain carries the parent checkpoint state digest into the new receipt and checkpoint.

## 为什么这是 v7.15

v7.7 proved queued resume preserves the parent checkpoint id through claim/drain.

v7.15 proves the resume path also consumes the parent checkpoint state digest:

```text
parent checkpoint.state_digest
  -> FED_QUEUE_RESUME
  -> claim/drain
  -> receipt.restored_state_digest
  -> checkpoint.restored_state_digest
```

This is still digest-bound restore evidence, not a full process or model snapshot.

## v7.15 不新增

- 不恢复 model KV/cache。
- 不做 process snapshot restore。
- 不做 long-running MCP session resume。
- 不做 artifact merge。
- 不做 automatic scheduler loop。
- 不做 state restore service。
