# Agent Space v8.10 Boundary

状态：complete
目标：Human Gateway persists the Zone-approved browser requester alias rebinding into a local registry file that existing registry tooling can resolve.

## v8.10 新增

- `POST /api/requester/rebindings` writes `state/go-fed-discovery-requester-registry.json`.
- The registry uses the existing Node registry shape:
  - `zone`
  - `revocations`
  - `agents[].descriptor`
  - `agents[].zone_binding`
- The registry stores the `next_descriptor` after rotation proof verification.
- The registry stores a Zone-signed binding for `agent://browser/requester` to the next `aid`.
- Existing `loadRegistry` / `resolveAgent` can resolve the rebound requester alias.

## 为什么这是 v8.10

v8.9 produced a Zone-signed alias rebinding proof but did not persist alias state.

v8.10 adds the smallest durable alias update:

```text
verified rotation proof
  -> Zone-signed rebinding proof
  -> local requester registry JSON
  -> resolve agent://browser/requester to next aid
```

This makes the browser requester rotation useful after the request completes, while keeping the registry local and explicit.

## v8.10 不新增

- 不做 browser UI for rebinding submission。
- 不做 multi-requester registry。
- 不做 rebinding history table。
- 不做 server-side rotation registry。
- 不做 remote registry sync。
- 不做 automatic rebinding after rotation。
- 不做 multi-tenant admin model。
- 不做 login/session identity。
- 不做 public deployment。
