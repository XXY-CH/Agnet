# Agent Space v9.7 Boundary

状态：complete
目标：unsupported sandbox claims fail before external or MCP tools start.

## v9.7 新增

- Worker `sandbox_claim` is preflighted before `task.started` and before tool execution.
- Supported claims remain the runtime's actual modes:
  - `local-temp-dir` for `external.stdio` and `mcp.stdio` tools.
  - `in-process` for built-in/mock tools.
- Unsupported claims return `unsupported sandbox claim: <claim>`.
- Rejected tasks persist a failed task state with the sandbox claim error.
- The integration test proves a `container-namespace` claim does not start its marker tool.

## 为什么这是 v9.7

Ultimate needs real container namespace sandboxing, but the current runtime is still honest `local-process` isolation.
This slice prevents a dangerous halfway state where a worker advertises stronger isolation and the runtime executes the tool before discovering the claim mismatch.

## v9.7 不新增

- 不实现 container namespace sandbox。
- 不做 Docker/OCI runtime。
- 不做 seccomp/AppArmor/profile enforcement。
- 不做 network namespace。
- 不做 filesystem mount namespace。
- 不做 cgroup limits。
- 不做 remote attestation。
- 不做 scheduler or automatic drain。
- 不做 A2A/ARD compatibility。
