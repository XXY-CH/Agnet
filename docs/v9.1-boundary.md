# Agent Space v9.1 Boundary

状态：complete
目标：artifact-store can produce a GC plan from the verified mirror index and audit receipts.

## v9.1 新增

- `--artifact-store-gc-plan --artifact-store <dir> --audit <file>` prints a JSON GC plan.
- The plan verifies the audit and mirror index before computing orphans.
- The plan treats mirror index entries not referenced by any receipt artifact manifest as `orphans`.
- The Go integration test appends an orphan index entry and expects it in the GC plan.

## 为什么这是 v9.1

v9.0 made `objects.ndjson` part of audit verification.
v9.1 adds the smallest lifecycle step after verification: a mark/sweep plan without deleting bytes.

```text
audit receipt manifests
  -> referenced sha256 set
  -> objects.ndjson entries
  -> orphan list
```

## v9.1 不新增

- 不删除 artifact bytes。
- 不删除 sidecars。
- 不重写 or compact `objects.ndjson`。
- 不做 retention policy。
- 不做 remote object-store API。
- 不做 S3/MinIO backend。
- 不做 artifact browser。
- 不做 auth model for mirrored artifacts。
- 不做 container namespace sandbox。
- 不做 public deployment。
- 不做 A2A/ARD compatibility。
