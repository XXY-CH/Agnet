# Agent Space v11.14 Boundary

状态：complete
目标：Make the Node `FED_SWARM_CLOSE` verifier reject NUL-bearing Swarm identities.

## v11.14 新增

- Node `verifySwarmClose` rejects signed close `swarm_id` values containing NUL.
- Node `verifySwarmClose` rejects close step `step_id` values containing NUL.
- The Node conformance test proves trusted signed close proofs fail closed for both NUL-bearing identity fields.

## 为什么这是 v11.14

The Go audit-backed Swarm verifier already rejects NUL-bearing Swarm ids and step ids because it uses NUL as an internal delimiter. v11.14 brings the Node structural close verifier to the same fail-closed boundary without adding a Node audit verifier.

## v11.14 不新增

- 不实现 Node audit-backed Swarm completeness verification。
- 不验证 step receipts against an audit log。
- 不实现 dynamic Swarm decomposition。
- 不实现 scheduler-owned routing。
- 不实现 parallel or cross-Zone Swarm execution。
- 不实现 public reachability proof。
- 不实现 A2A/ARD compatibility。
