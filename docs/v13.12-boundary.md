# v13.12 Boundary: Credential Validity Window

状态：complete

目标：Add an optional credential validity window to capability credentials so expired credentials remain trusted history but no longer count as active discovery evidence.

## What Landed

- Capability credentials may carry a `valid_until` ISO UTC expiry in claims; expired credentials lower discovery score and report `active: false` in discovery evidence.
- Node credential verification fails closed when `valid_until` is malformed, non-string, unparseable, or in the past.
- Node semantic discovery now uses credential verification for `discovery_evidence.credential.active`, score contribution, and `credential_active` ranking reasons.
- Go FED_QUERY keeps credentials active when no expiry is present and fails closed for malformed or expired `valid_until` claims.
- `test/federation-gateway.test.mjs`, `test/capability-credential.test.mjs`, `test/go-fed-discovery.test.mjs`, and `test/docs-contract.test.mjs` cover the credential validity boundary.

## Non-Goals

- No credential lifecycle service.
- No key custody changes.
- No inter-zone credential transport protocol.
- No distributed credential registry.
- No marketplace or token-weighted reputation.

## Why This Boundary

v13.4, v13.10, and v13.11 made semantic discovery deterministic and evidence-first, but credential trust still behaved as a permanent score boost once a credential existed. v13.12 keeps the same credential shape and adds one optional expiry claim that verifiers can evaluate locally.

A credential without `valid_until` keeps the existing active behavior. A credential with `valid_until` must use the implemented ISO UTC timestamp shape and must be at or after verifier time. When the window has passed, the credential can still be reported as trusted because the signature exists, but it is not active and does not add the discovery score boost.

## Test Contract

The tests must fail if:

- Expired credentials still report `active: true` in discovery evidence.
- Expired credentials still add the credential score boost or `credential_active` reason.
- Malformed `valid_until` claims are accepted as active.
- Public docs describe this as broader credential lifecycle infrastructure instead of the narrow optional expiry claim.
