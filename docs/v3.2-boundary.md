# Agent Space v3.2 Boundary

状态：v3.2 complete
目标：让 Go discovery/runtime 支持 multi-worker registry。

## v3.2 新增

- Discovery fixture can provide `worker_profiles`。
- Each worker profile can provide its own `key_file`。
- Go dynamically generates a descriptor for each worker profile。
- `FED_RESOLVE` routes by exact worker alias。
- `FED_QUERY` returns all exact capability matches。
- `FED_TASK_OPEN` executes against the worker targeted by `task.to`。
- Receipt and audit records bind to the selected worker。

## 为什么这是 v3.2

v3.1 made a single Go worker auditable。

v3.2 proves the gateway/runtime is no longer hardwired to one worker:

```text
worker_profiles[]
  -> descriptor per worker
  -> alias/capability registry
  -> task routes to selected worker
  -> receipt signed by selected worker
```

This is still a registry proof, not a scheduler.

## v3.2 不新增

- 不做 semantic routing。
- 不做 ranking。
- 不做 scheduler。
- 不做 worker lifecycle API。
- 不做 remote registry discovery。
- 不做 real tool execution。

## 验收

```bash
node --test --test-concurrency=1 *.test.mjs
go test ./...
```

The Go integration test must prove:

- `summarize.text` resolves to summarizer。
- `translate.text` resolves to translator。
- task execution receipt is signed by the selected worker。
