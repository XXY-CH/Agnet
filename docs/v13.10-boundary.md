# v13.10 Boundary: Go FED_QUERY Semantic Discovery Parity

状态：complete

目标：Close the Go FED_QUERY semantic discovery parity gap deferred in v13.4 while keeping ranking evidence-first and inspectable.

## What Landed

- `cmd/go-fed-discovery/main.go` `FED_QUERY` now accepts an `intent` string and calls `queryMatch`.
- `queryMatch` returns `discovery_evidence` with `capability`, `credential`, and `reputation` fields, plus `ranking.score` and `ranking.reasons` — mirroring the Node federation-gateway.mjs surface from v13.4.
- `semanticScore` and `tokenize` helpers compute token-overlap intent scoring over alias and capabilities.
- Results are sorted by ranking score descending, with alias ascending as a tiebreaker, and `go-fed-discovery.test.mjs` covers semantic intent query, discovery_evidence shape, ranking score comparison, and ranking reasons.

## Non-Goals

- No vector database.
- No global reputation coin or global reputation service.
- No public marketplace.
- Go stays at token-overlap intent scoring, not embeddings.
- No scheduler integration.
- No Go parity for semantic-only edge cases beyond what the Node side already specifies.

## Why This Boundary

v13.4 explicitly deferred Go FED_QUERY parity while landing evidence-first semantic discovery and reputation ranking in the Node federation gateway. That left the public protocol surface uneven: Node query results exposed inspectable discovery evidence and ranking reasons, while Go query results did not.

v13.10 closes that deferred parity gap with the same evidence-first ranking surface the Node side uses. Go now accepts the same `intent` input class, reports per-candidate `discovery_evidence`, and ranks candidates through inspectable `ranking.score` and `ranking.reasons`, while staying honest that the semantic mechanism is token overlap rather than embeddings.

## Test Contract

The tests must fail if:

- Go FED_QUERY returns matches without `discovery_evidence`.
- Semantic-only candidates outrank exact-match plus trusted-credential candidates.
- `ranking.reasons` is absent from Go matches.
- Public docs describe this as vector search, global reputation, marketplace discovery, scheduler integration, or embedding-based ranking.
