# Agent Space v8.11 Boundary

状态：complete
目标：Human Gateway page can submit the browser-held requester rotation bundle to the existing requester alias rebinding API.

## v8.11 新增

- Browser Requester Key panel includes `Bind Alias`.
- `Bind Alias` reads the stored `agent-space-browser-requester` bundle from browser `localStorage`.
- The browser requires the bundle to contain `previous_descriptor` and `rotation_proof`.
- The browser submits:
  - `previous_descriptor`
  - `next_descriptor`
  - `rotation_proof`
- The submission uses existing `POST /api/requester/rebindings`.
- The submission reuses the existing local Human Gateway bearer token input when configured.
- The response is shown in the existing browser requester status area.

## 为什么这是 v8.11

v8.8 produced a browser-side rotation bundle.
v8.9 verified that bundle and issued a Zone-signed alias rebinding proof.
v8.10 persisted the rebound requester alias in the local registry.

v8.11 connects the already-built pieces from the page:

```text
Rotate Key
  -> stored { previous_descriptor, descriptor, rotation_proof }
  -> Bind Alias
  -> POST /api/requester/rebindings
  -> Zone-approved registry update
```

This keeps rebinding explicit and human-triggered.

## v8.11 不新增

- 不做 automatic rebinding after rotation。
- 不做 multi-requester registry。
- 不做 rebinding history table。
- 不做 server-side rotation registry。
- 不做 encrypted key store。
- 不做 passphrase-protected export。
- 不做 compromised-key recovery。
- 不做 login/session identity。
- 不做 public deployment。
