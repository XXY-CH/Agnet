# Agent Space v9.24 Boundary

状态：complete
目标：Node and Go verify the same static `FED_TASK_OPEN` conformance fixture.

## v9.24 新增

- `test-vectors/asp-v9.24-fed-task-open.json` captures a signed `FED_TASK_OPEN` frame with deterministic Zone, requester, worker, policy, and expected canonical task body.
- Node exports `verifyFederatedTaskOpen(...)` from `asp-core.mjs`.
- Node federation gateway reuses the shared verifier for live `FED_TASK_OPEN` handling.
- Go package coverage verifies the same fixture through the existing `verifyTrustedZone` and `Fixture.verifyTaskOpen` paths.
- Node vector coverage verifies the same fixture through `verifyFederatedTaskOpen`.

## 为什么这是 v9.24

v9.21 and v9.22 proved bidirectional task interop at runtime.
v9.24 makes the core signed-task assumption reusable as a cross-implementation fixture before writing a broader conformance suite.
