# Agent Space v11.12 Boundary

状态：complete
目标：Make the Node `FED_SWARM_CLOSE` verifier reject duplicate step receipts.

## v11.12 新增

- Node `verifySwarmClose` rejects repeated `step_id` values inside one signed close proof.
- The Node conformance test proves a trusted Zone signature is not enough when a close proof double-counts the same Swarm step.

## 为什么这是 v11.12

v11.2 made Swarm close proofs non-empty and structurally typed, but the Node verifier still accepted duplicate step ids. v11.12 closes that structural accountability gap without adding an audit-backed Node verifier.

## v11.12 不新增

- 不实现 Node audit-backed Swarm completeness verification。
- 不验证 step receipts against an audit log。
- 不实现 dynamic Swarm decomposition。
- 不实现 scheduler-owned routing。
- 不实现 parallel or cross-Zone Swarm execution。
- 不实现 public reachability proof。
- 不实现 A2A/ARD compatibility。
