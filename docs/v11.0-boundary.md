# Agent Space v11.0 Boundary

状态：complete
目标：Reject untrusted signed receipt `origin_zone` values in the Node and Go `FED_RECEIPT` verifiers.

## v11.0 新增

- Node `verifyFederatedReceipt` checks that `receipt.origin_zone` exists in the trusted Zone store.
- Go `agnet/verifier.VerifyFederatedReceipt` checks the same trust condition.
- The shared `FED_RECEIPT` vector now carries both trusted origin and executing Zone descriptors.
- The public proof script writes receipt verifier trust files with both origin and executing Zone descriptors.
- Node and Go red tests cover the untrusted-origin rejection path.

## 为什么这是 v11.0

v10 closed the external proof package. The next Ultimate-aligned trust gap is receipt provenance: a receipt already signs `origin_zone`, but the verifier previously accepted an origin Zone that was not trusted. v11.0 closes that gap without changing the frame schema or starting a broader identity/resolver project.

## v11.0 不新增

- 不改变 `FED_RECEIPT` frame schema。
- 不要求 receipt frame 携带 requester descriptor。
- 不实现 requester-to-origin Zone binding。
- 不实现 DID document/resolver。
- 不实现 remote trust-store sync。
- 不实现 public reachability proof。
- 不实现 receipt store/search。
- 不实现 batch verifier。
- 不实现 HTTP verifier service。
- 不实现 A2A/ARD compatibility。
