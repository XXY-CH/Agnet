# Agent Space v14.0 Boundary

状态：active
协议标签：`v14.0-protocol`

## 目标

Open v14 as the local-first milestone that pushes the next three Ultimate layers upward:

- Overlay Network: from local federation toward verifiable cross-zone trust paths and multi-hop routing evidence.
- Agent Swarm Layer: from scheduler-owned DAG execution toward micro-contract negotiation, failure migration, and conflict resolution.
- Multi-signal routing: from capability/reputation/semantic FED_QUERY ranking toward cost, latency, risk, and availability-aware routing.

v13 proved the lower protocol core: Task Fabric receipts/events/artifacts/checkpoints, Trust and Verification evidence, and Discovery ranking inputs. v14 should keep those proofs intact while moving upward one layer at a time.

## Honest assessment at opening

- Overlay Network is still about 18% of the Ultimate shape: current federation is local-first, with no cross-zone trust chains and no real multi-hop routing.
- Swarm Layer is about 35%: DAG scheduling exists, but micro-contract negotiation, failure migration, and conflict resolution are missing.
- Multi-signal routing is about 45%: FED_QUERY has capability/credential/reputation/semantic signals, but cost, latency, risk, and availability are not yet route inputs.

## v14.0 新增

- `docs/v14-roadmap.md` defines the active v14 slices.
- README, implementation status, ASP Core draft, and docs contract point at `v14.0-protocol`.
- The roadmap explicitly scopes v14.1 Swarm micro-contracts, v14.2 Multi-signal FED_QUERY routing, v14.3 Cross-zone trust chains, and v14.4 Task failure migration.

## v14.0 不新增

- No runtime behavior is introduced by the v14.0 opening boundary itself.
- No P2P DHT.
- No token economy.
- No public marketplace.
- No production global overlay.
- No NAT traversal, relay mesh, QUIC transport expansion, or global DHT routing.
- No settlement, billing, escrow, or machine payment layer.
- No claim that cross-zone trust chains, failure migration, conflict resolution, or cost/latency/availability routing are complete.

## Layer boundary

v14 must remain proof-first:

- If a route decision uses a signal, the signal must be inspectable.
- If a Swarm step is selected, the worker commitment must be signed before execution.
- If trust crosses zones, the delegation chain must be carried and verified.
- If a task migrates after failure, the failed attempt and replacement lineage must remain auditable.
