# Agent Space v12.11 Boundary

状态：complete
目标：Make `proof-bundle` reject signed but non-public transport proofs.

## v12.11 新增

- `asp-verify.mjs proof-bundle <bundle.json>` now requires the verified signed receipt to carry an object-shaped `transport_proof` with `public_transport: true`.
- A bundle whose `transport_proof` exactly matches a signed receipt with `public_transport: false` is rejected with `bundle public_transport proof missing`.
- `public-node-proof.test.mjs` proves this by re-signing a receipt with `public_transport: false` and matching the bundle fields.

## 为什么这是 v12.11

`proof: "public-node-proof"` is a narrower claim than a normal receipt proof. v12.10 made verifier CLI entrypoints exact, but a public proof bundle could still accept a signed transport proof that explicitly said the listener was not public as long as the bundle matched the receipt. This closes that overclaim without pretending to prove external-host reachability.

## v12.11 不新增

- 不实现 external public reachability proof。
- 不实现 hosted public node。
- 不实现 DNS, TLS, QUIC, NAT traversal, or remote probe infrastructure。
- 不改变 normal `fed-receipt` verification。
- 不改变 verifier JSON output。
- 不实现 batch verifier。
- 不实现 JSON Schema。
- 不实现 generic proof bundle schema。
- 不实现 package signing or SBOM。
- 不实现 A2A/ARD compatibility。
- 不实现 scheduler-owned routing。

It does not prove reachability from another host.
