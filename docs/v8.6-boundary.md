# Agent Space v8.6 Boundary

状态：complete
目标：Human Gateway page can manage a browser-held requester key and submit signed queue drafts without giving the private key to Go.

## v8.6 新增

- Human Gateway page includes a Browser Requester Key panel.
- The panel uses browser WebCrypto Ed25519 key generation.
- The browser stores the requester private key in `localStorage` under `agent-space-browser-requester`.
- The browser derives the requester `aid` from the exported SPKI public key.
- The browser self-signs the requester descriptor with `descriptor_signature`.
- The browser signs a draft task and submits `{ requester, task }` to the existing `POST /api/queue/drafts` endpoint.

## 为什么这是 v8.6

v8.3 proved the gateway can carry an externally signed requester task without holding the requester private key.

v8.6 adds the missing first browser UX surface above that protocol path:

```text
browser WebCrypto key
  -> signed requester descriptor
  -> signed task draft
  -> Human Gateway /api/queue/drafts
  -> existing enqueue verification path
```

This moves the Human Gateway from a test/API surface toward a real requester control plane while keeping private-key custody in the browser.

## v8.6 不新增

- 不做 login/session identity。
- 不做 role model。
- 不做 encrypted key store。
- 不做 cross-browser key sync。
- 不做 key rotation UI。
- 不做 polished task builder。
- 不做 public deployment。
