# Agent Space v10.29 Boundary

状态：complete
目标：Prove public-listen artifact reads reject receipt-valid artifact URIs when local artifact bytes drift after receipt creation.

## v10.29 新增

- `scripts/public-node-proof.mjs` first performs a successful authenticated `FED_ARTIFACT_READ` and verifies the fetched artifact bytes with `asp-verify.mjs fed-receipt-artifacts`.
- The proof then mutates the same artifact file with equal-length different bytes and sends another authenticated `FED_ARTIFACT_READ` for the receipt-valid URI.
- The gateway returns a `FED_TASK_ERROR` from the existing receipt-bound artifact byte verification.
- The proof output includes:
  - `artifact_tamper_reject: true`
  - `artifact_tamper_error` matching `artifact bytes digest mismatch`
- The proof restores the valid artifact bytes before exit so verifier-ready receipt/trust closure files remain independently checkable.
- `test/public-node-proof.test.mjs` asserts the tamper rejection evidence.
