# Agent Space v5.4 Boundary

状态：complete
目标：让 sandbox claim 进入 receipt 和 signed proof，防止把弱隔离说成强隔离。

## v5.4 新增

- Worker profile may declare `sandbox_claim`。
- Execution rejects a declared claim that does not match actual sandbox mode。
- Receipt records `sandbox_claim` when declared。
- `sandbox_proof` signs the same `sandbox_claim`。
- Audit verifier checks receipt/proof claim alignment。

## 为什么这是 v5.4

Ultimate trust depends on not overclaiming sandbox strength。

v5.4 adds a claim boundary:

```text
worker declares sandbox claim
  -> actual sandbox evidence
  -> receipt.sandbox_claim
  -> sandbox_proof.sandbox_claim
  -> verifier checks alignment
```

This is a stronger honesty boundary before real container sandboxing。

## v5.4 不新增

- 不做 Docker/gVisor/Firecracker。
- 不做 OS-level network enforcement。
- 不做 hardware attestation。
- 不做 sandbox broker。
- 不把 `local-temp-dir` 描述成 container。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 *.test.mjs
```

The Go integration test must fail if the declared sandbox claim is missing from receipt/proof or if proof claim does not match receipt claim。
