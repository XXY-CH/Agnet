# Agent Space v8.15 Boundary

状态：complete
目标：Human Gateway queue action grants are consumed through a durable local nonce index instead of replay checks depending on audit scans.

## v8.15 新增

- Successful Human Gateway queue action grant verification creates a local grant-use record under the audit-derived queue grant directory.
- The grant-use filename is the existing grant digest.
- The grant-use record stores grant digest, action, task id, actor, and consumed timestamp.
- Replay rejection now checks the durable grant-use file with exclusive create semantics.
- The existing `go_queue_action` audit evidence still records queue action result and grant digest.

## 为什么这是 v8.15

v7.10 rejected grant replay, but the check depended on scanning successful queue action audit entries.

v8.15 makes replay state explicit and durable:

```text
signed queue action grant
  -> verify grant
  -> create grant digest file once
  -> reject the same digest on replay
```

This moves toward Ultimate's task-scoped, auditable governance without building a public gateway or distributed nonce service.

## v8.15 不新增

- 不做 distributed nonce service。
- 不做 durable queue action concurrency model beyond exclusive local file create。
- 不做 configurable actor authorization policy。
- 不做 login/session identity。
- 不做 token storage or rotation。
- 不做 public deployment。
- 不做 A2A/ARD compatibility。
