# Agent Space v0.4 Boundary

状态：v0.4 candidate  
目标：让 `agent://` alias 进入 Zone 治理边界，而不削弱 Agent ID 独立性。

## v0.4 新增

- Zone identity：`zid:ed25519:<hash>`。
- Zone Descriptor 自签名。
- Registry 改为 `{ zone, agents }`。
- 每个 registry agent entry 包含：
  - Agent Descriptor
  - Zone-signed alias binding
- `resolveAgent()` 验证：
  - Agent `aid` 由 Agent public key 计算
  - Agent Descriptor 自签名
  - Zone `zid` 由 Zone public key 计算
  - Zone Descriptor 自签名
  - Zone binding 的 `alias -> aid` 未被篡改
  - Zone binding signature 有效

## 为什么这是 v0.4

v0.3 让 audit log 可发现篡改。

v0.4 明确三层身份边界：

```text
aid:       Agent 身份，由 Agent key 控制
zid:       Zone 身份，由 Zone key 控制
agent://   Zone 内路由 alias，由 Zone 签名绑定到 aid
```

这解决了一个关键问题：registry 可以负责路由，但不能悄悄把 `agent://local/summarizer` 指向另一个 `aid` 而不留下签名变化。

## v0.4 不新增

- 不做多 Zone federation。
- 不做 Zone 信任列表。
- 不做撤销。
- 不做 key rotation。
- 不做 DNS/DID resolver。
