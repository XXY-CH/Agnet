# v13.6 Boundary: Sandbox Proof Verifier

状态：complete

目标：把 existing local sandbox proof 推进到独立 verifier-owned sandbox class validation, without claiming hardware attestation or real container isolation.

## What Landed

- `asp-verify.mjs sandbox-proof <frame.json> <trusted-zones.json> [required-sandbox-class]`.
- It verifies a signed `local.sandbox.v1` proof embedded in a verified `FED_RECEIPT` frame.
- It checks task id, authority Zone, worker, policy digest, sandbox claim, and sandbox evidence binding.
- It verifies the sandbox proof signature with the receipt signing Zone key.
- It requires sandbox evidence to carry mode, isolation level, network surface, command digest, binary digest, and transcript digest.
- It reports verifier-owned `sandbox_class: "local-process"` for the current local sandbox proof.
- It fails closed when a stronger required sandbox class such as `remote-attestation` is requested without matching signed evidence.

## Non-Goals

- This is not hardware remote attestation, not container namespace execution, and not a VM/TEE claim.
- It does not make `container-namespace` supported.
- It does not run tools inside a new sandbox runtime.
- It does not infer strong isolation from local-process evidence.
- It does not change Go task execution semantics.

## Why This Boundary

Ultimate needs stronger isolation and remote-verifiable execution claims, but claiming those without runtime evidence would be worse than leaving the gap open.

v13.6 therefore moves the verifier boundary first:

```text
FED_RECEIPT
  -> signed local.sandbox.v1 proof
  -> task/authority/worker/policy/claim/sandbox binding
  -> digest-bearing sandbox evidence
  -> verifier-owned sandbox_class
```

The next sandbox step must either provide real stronger runtime evidence or continue failing closed.

## Test Contract

The tests must fail if:

- `sandbox-proof` accepts a receipt without a signed sandbox proof.
- `sandbox-proof` accepts sandbox proof evidence that is not bound to the receipt.
- `sandbox-proof` accepts local-process proof as `remote-attestation`.
- `sandbox-proof` accepts evidence missing command, binary, or transcript digests.
