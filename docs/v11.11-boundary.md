# Agent Space v11.11 Boundary

状态：complete
目标：Make `FED_TASK_OPEN` verifiers reject task-shaped frames with the wrong protocol type.

## v11.11 新增

- Node `verifyFederatedTaskOpen` now requires `frame.type === "FED_TASK_OPEN"`.
- Go `Fixture.verifyTaskOpen` now requires `frame["type"] == "FED_TASK_OPEN"`.
- Internal Go queue/resume/retry and Swarm replay callers now pass explicit `FED_TASK_OPEN` type when re-validating embedded or stored signed tasks.
- Node and Go tests prove a `FED_RECEIPT` frame carrying otherwise valid task-open fields fails closed with `expected FED_TASK_OPEN frame`.

## 为什么这是 v11.11

v11.10 made receipt verification protocol-type strict. `FED_TASK_OPEN` verification had the same ambiguity. v11.11 closes the task-entry counterpart without changing task shape, routing, scheduling, or storage semantics.

## v11.11 不新增

- 不改变 `FED_TASK_OPEN` frame schema。
- 不实现 task store/search。
- 不实现 scheduler-owned routing。
- 不实现 dynamic Swarm decomposition。
- 不改变 requester Zone binding semantics。
- 不实现 A2A/ARD compatibility。
