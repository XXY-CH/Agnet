# Agent Space v8.45 Boundary

状态：complete
目标：Artifact manifest retrieval can be task-scoped and bound to the selected signed receipt.

## v8.45 新增

- `GET /api/artifacts/manifest?task_id=<id>&uri=<artifact-uri>` returns `manifest`.
- The response also returns `receipt_digest` and `audit_hash`.
- The manifest comes from the signed receipt selected by task-scoped audit proof lookup.
- Existing URI-only `GET /api/artifacts/manifest?uri=...` remains as the local sidecar read.
- Human Gateway receipt table manifest links include `task_id`.
- The Go integration test checks both the old URI-only response and the new task-scoped proof response.

## 为什么这是 v8.45

v8.44 made artifact verification self-contained.
v8.45 gives clients the same task-scoped proof shape when they only need the manifest:

```text
receipt task_id + artifact uri
  -> signed receipt manifest
  -> receipt_digest
  -> audit_hash
```

This keeps manifest inspection on the receipt/audit path without deleting the older sidecar read surface.
