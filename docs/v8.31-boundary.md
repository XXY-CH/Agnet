# Agent Space v8.31 Boundary

状态：complete
目标：Direct Human Gateway approvals reject body actors that mismatch the configured bearer approval session actor.

## v8.31 新增

- `/api/approvals/actions` still accepts explicit body actors for local compatibility.
- If the bearer token maps to an `approval_sessions` actor, a different body actor is rejected.
- If the body omits `actor`, the bearer session actor is still used.
- The matched or derived actor still passes the existing `approval_actions` allowlist.
- The Go integration test rejects a body actor of `human://local` when the bearer session maps to `human://operator`.

## 为什么这是 v8.31

v8.30 let a local bearer session supply the approval actor.
v8.31 prevents the body from overriding that session identity:

```text
Authorization: Bearer <token>
  -> approval_sessions[token] = human://operator
  -> body.actor = human://local
  -> reject: approval actor session mismatch
```

This is the smallest enforcement step toward login-backed approval identity without adding a real account/session product surface.

## v8.31 不新增

- 不做 login UI。
- 不做 password/session store。
- 不做 roles/admin model。
- 不做 token rotation/storage。
- 不做 CSRF/session cookies。
- 不做 dynamic policy service。
- 不做 public deployment。
- 不做 A2A/ARD compatibility。
