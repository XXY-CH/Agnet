# Agent Space v12.38 Boundary

状态：complete
目标：Sign the local package proof manifest with an ASP agent identity.

## v12.38 新增

- `scripts/package-proof.mjs` creates or reuses `state/keys/package-proof-signer.pkcs8`.
- The package proof manifest includes the signer Agent descriptor in `signer`.
- The package proof manifest includes `signature`, signed by the package-proof signer over the proof body without `proof_digest` or `signature`.
- `asp-verify.mjs package-proof <manifest.json>` verifies the signer descriptor, `proof_digest`, and package proof signature before accepting the proof.
- The package proof verifier returns `signer_aid` in its JSON output.
- `package-contract.test.mjs` covers valid signed proof output plus missing and invalid package proof signatures.

## 为什么这是 v12.38

Earlier package proof slices verified tarball bytes and npm digest metadata, but the manifest itself was only self-digested. v12.38 reuses the existing ASP Agent descriptor and Ed25519 object-signature path so a package proof is tied to a concrete signer identity.

## v12.38 不新增

- 不实现 npm registry signing。
- 不实现 external signer trust pin or release transparency。
- 不实现 SBOM。
- 不实现 package publish。
- 不实现 hosted public node。
- 不实现 real outside-host reachability。
- 不实现 scheduler-owned routing。
- 不实现 A2A/ARD compatibility。

This is an ASP-signed local package proof, not a full package release trust system.
