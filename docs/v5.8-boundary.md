# Agent Space v5.8 Boundary

状态：complete
目标：把 MCP resources/prompts list metadata 摘要绑定进 sandbox evidence。

## v5.8 新增

- MCP adapter calls `resources/list` once after initialize。
- MCP adapter calls `prompts/list` once after initialize。
- Sandbox evidence records:
  - `resources_count`
  - `resources_digest`
  - `prompts_count`
  - `prompts_digest`
- `sandbox_proof` signs this metadata because it signs sandbox evidence。

## 为什么这是 v5.8

MCP initialize metadata identifies the server。

v5.8 records what resource/prompt surface the server exposed at task time:

```text
resources/list + prompts/list
  -> count + digest
  -> sandbox evidence
  -> signed sandbox_proof
  -> receipt / remote audit proof
```

This gives audit-visible capability surface evidence without storing full resource or prompt catalogs。

## v5.8 不新增

- 不保存完整 resources/prompts 列表。
- 不读取 resource contents。
- 不执行 prompts/get。
- 不做 long-running MCP session reuse。
- 不做 MCP registry。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 *.test.mjs
```

The Go integration test must fail if resources/prompts metadata is absent from sandbox evidence or not covered by the signed sandbox proof。
