# Agent Space v8.29 Boundary

状态：complete
目标：External/MCP local-process sandbox runs with home/cache/temp environment variables bound to the sandbox directory.

## v8.29 新增

- External/MCP subprocesses receive `HOME=<sandbox cwd>`.
- External/MCP subprocesses receive `TMPDIR=<sandbox cwd>`.
- External/MCP subprocesses receive `XDG_CACHE_HOME=<sandbox cwd>/cache`.
- The sandbox cache directory is created before tool execution.
- Sandbox evidence records the exact environment list passed to the subprocess.
- The Go integration test verifies the MCP tool observes the sandbox-bound environment.

## 为什么这是 v8.29

The runtime already labels external/MCP execution honestly as `local-process`.
v8.29 hardens the local-process sandbox without overstating it:

```text
local temp sandbox cwd
  -> HOME/TMPDIR/XDG_CACHE_HOME bound inside cwd
  -> receipt sandbox evidence records env
  -> tool transcript proves the process observed env
```

This reduces accidental writes to the user's real home/cache/temp paths while leaving true namespace isolation to a later container boundary.

## v8.29 不新增

- 不做 container namespace sandbox。
- 不做 syscall filtering。
- 不做 filesystem mount isolation。
- 不做 network namespace isolation。
- 不做 long-running MCP session reuse。
- 不做 public deployment。
- 不做 A2A/ARD compatibility。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 test/go-fed-discovery.test.mjs
node --test --test-concurrency=1 test/*.test.mjs
```

The Go integration test must fail if the subprocess environment is not bound to the sandbox directory.
