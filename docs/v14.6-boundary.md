# v14.6 Boundary — Knowledge Gateway Proto

状态：complete
协议标签：`v14.6-protocol`

## 目标

v14.6 adds a narrow local-first Knowledge Gateway proof surface from Ultimate §8: Agent systems should not all scrape the web directly. A trusted requester Zone can sign a knowledge query, and a trusted gateway Zone can return source-cited, freshness-tagged, license-noted local results bound back to that query digest.

```text
Web / Structured feeds
  ↓
Knowledge Gateway
  parsing / dedup / source citation / freshness tag / license note
  ↓
Agent Task Fabric (signed response)
```

## v14.6 新增

- Node `knowledgeQuery(requesterZone, intent, sources, policyDigest)` creates a signed `FED_KNOWLEDGE_QUERY` frame.
- The query body carries `intent`, `sources`, `policy_digest`, generated `query_id`, `query_digest`, and `query_signature`.
- `query_digest` is the SHA-256 digest of canonical `{ intent, sources, policy_digest, query_id }`.
- Node `verifyKnowledgeQuery(frame, trustedZones)` verifies the frame type, Zone descriptor, local Zone trust, source list, 64-hex `policy_digest`, recomputed `query_digest`, and Zone signature.
- Node `knowledgeResponse(gatewayZone, queryId, results, queryDigest)` creates a signed `FED_KNOWLEDGE_RESPONSE` frame.
- The response body carries `query_id`, `query_digest`, `results`, `result_digest`, and `response_signature`.
- Each result is local gateway evidence with `source`, `title`, `summary`, `freshness_at`, and `license`.
- `result_digest` is the SHA-256 digest of canonical `{ query_id, query_digest, results }`.
- Node `verifyKnowledgeResponse(frame, trustedZones, queryFrame)` verifies the gateway Zone trust, response signature, result digest, and query binding by requiring `response.query_digest` to match the verified `FED_KNOWLEDGE_QUERY.query.query_digest`.

## v14.6 不新增

- Not a web crawler.
- Not HTTP fetching or browser automation.
- Not a semantic cache.
- Not a vector store.
- Not a RAG pipeline.
- Not source reputation scoring beyond carrying explicit source citation fields.
- Not conflict resolution across sources.
- Not a license compliance engine; `license` is a signed note carried with local gateway evidence.

## 为什么这是 v14.6

v14.5 made intent decomposition signed before Swarm execution. v14.6 opens the Ultimate §8 Knowledge Gateway seam without building the full Agent Knowledge Layer: the protocol can now bind a requester intent, source set, freshness/license-bearing results, and gateway signature before those results flow into the Agent Task Fabric.

## 验收

- `knowledgeQuery` produces a signed `FED_KNOWLEDGE_QUERY` with `intent`, `sources`, `policy_digest`, `query_id`, `query_digest`, and `query_signature`.
- `verifyKnowledgeQuery` accepts the valid query and rejects tampered `query_signature` or untrusted requester Zones.
- `knowledgeResponse` produces a signed `FED_KNOWLEDGE_RESPONSE` with source-cited results, `result_digest`, and `response_signature`.
- `verifyKnowledgeResponse` accepts the valid response and rejects tampered `response_signature` or mismatched `query_digest` bindings.
- Docs-contract tests assert `FED_KNOWLEDGE_QUERY`, `verifyKnowledgeResponse`, and `Knowledge Gateway` wording while preserving the non-claims above.
