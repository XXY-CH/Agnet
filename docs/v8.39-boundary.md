# Agent Space v8.39 Boundary

状态：complete
目标：Receipt-scoped artifact reads expose digest headers from the signed receipt manifest.

## v8.39 新增

- `GET /api/artifacts/read?task_id=<id>&uri=<artifact-uri>` returns `X-Agent-Space-Artifact-SHA256`.
- The same endpoint returns `X-Agent-Space-Artifact-Manifest-Hash`.
- Both headers come from the signed receipt artifact manifest selected after task-scoped audit proof lookup.
- The endpoint still verifies the receipt's artifact manifests, local bytes, named sidecars, digest sidecars, and optional filesystem mirror before returning bytes.
- The Go integration test checks the read response headers against the artifact manifest.

## 为什么这是 v8.39

v8.38 made receipt-scoped artifact bytes readable.
v8.39 adds the smallest client-verification hook:

```text
receipt task_id + artifact uri
  -> signed receipt manifest
  -> verified bytes
  -> digest headers
```

This lets a client bind returned bytes back to receipt evidence without building a broader artifact API.

## v8.39 不新增

- 不做 artifact browser。
- 不做 transcript browser。
- 不做 remote artifact fetch/download。
- 不做 object-store backend。
- 不做 artifact GC。
- 不做 auth model for artifact reads。
- 不做 HTTP cache protocol。
- 不做 streamed transcript UI。
- 不做 container namespace sandbox。
- 不做 public deployment。
- 不做 A2A/ARD compatibility。
