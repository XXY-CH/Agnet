# Agent Space v9.2 Boundary

状态：complete
目标：Node `appendAudit` serializes concurrent audit writes.

## v9.2 新增

- Node `appendAudit` uses a module-level Promise queue.
- Concurrent callers append audit entries one at a time.
- The audit chain keeps a single valid `prev_hash -> hash` sequence.
- The Node audit-chain test calls `appendAudit` concurrently and verifies the resulting chain.

## 为什么这是 v9.2

The review report identified Node audit writes as a correctness race:

```text
read audit.head
  -> append audit.log
  -> write audit.head
```

v9.2 keeps the existing file format and fixes the race at the shared writer.

## v9.2 不新增

- 不做 lockfile。
- 不做 database。
- 不做 cross-process locking。
- 不做 audit log compaction。
- 不做 Go approval locking。
- 不做 scanner buffer changes。
- 不做 state-file atomic writes。
- 不做 A2A/ARD compatibility。

## 验收

```bash
node --test --test-concurrency=1 audit-chain.test.mjs
node --test --test-concurrency=1 *.test.mjs
go test ./...
```

The Node audit-chain test must fail if concurrent `appendAudit` calls can produce a forked audit hash chain.
