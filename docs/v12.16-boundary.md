# Agent Space v12.16 Boundary

状态：complete
目标：Make public proof bundle verification label its current reachability scope.

## v12.16 新增

- `asp-verify.mjs proof-bundle <bundle.json>` now returns `reachability_scope: "local-interface"`.
- `scripts/public-node-proof.mjs` forwards that verifier-owned value into the public proof summary.
- `public-node-proof.test.mjs` checks both the public proof summary and direct `proof-bundle` verifier output.

## 为什么这是 v12.16

v12.15 removed loopback and unspecified listener overclaims, but the implemented proof is still a local-interface proof. This names that scope directly in verifier output before future external probe work exists.

## v12.16 不新增

- 不实现 external public reachability proof。
- 不实现 hosted public node。
- 不增加 DNS, TLS, QUIC, NAT traversal, or remote probe infrastructure。
- 不改变 normal `fed-receipt` verification。
- 不改变 verifier JSON output beyond `proof-bundle.reachability_scope`。
- 不实现 transport negotiation。
- 不实现 batch verifier。
- 不实现 JSON Schema。
- 不实现 generic proof bundle schema。
- 不实现 package signing or SBOM。
- 不实现 A2A/ARD compatibility。
- 不实现 scheduler-owned routing。

It does not prove reachability from another host.
