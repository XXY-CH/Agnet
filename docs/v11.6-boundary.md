# Agent Space v11.6 Boundary

状态：complete
目标：Keep Node receipt-only and receipt-plus-artifact verifier CLIs aligned on optional task evidence checks.

## v11.6 新增

- `asp-verify.mjs fed-receipt-artifacts` accepts an optional task JSON file after the trusted-Zones file.
- The artifact-closure verifier rejects supplied task evidence whose canonical digest does not match `receipt.task_digest`.
- The existing local artifact byte and manifest checks still run after receipt/task evidence verification.

## 为什么这是 v11.6

v11.5 added optional signed task evidence checking to the receipt verifier CLI, but the receipt-plus-artifact CLI still skipped that same optional evidence. v11.6 keeps the two external verifier entry points consistent without adding a store, batch mode, or a new protocol object.

## v11.6 不新增

- 不实现 task store/search。
- 不实现 batch verifier。
- 不改变 receipt-only verification when no task evidence is supplied。
- 不改变 artifact byte or manifest verification semantics。
- 不实现 scheduler-owned routing。
- 不实现 dynamic Swarm decomposition。
- 不实现 A2A/ARD compatibility。
