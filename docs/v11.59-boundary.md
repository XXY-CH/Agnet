# Agent Space v11.59 Boundary

状态：complete
目标：Make Go filesystem artifact mirror index readers reject present URIs that are not strings.

## v11.59 新增

- Go `readArtifactStoreIndex` rejects present `uri` fields that are not strings with `artifact mirror index invalid`.
- Focused Go test proves an index row with `uri: {"path":"artifact://local/out.md"}` no longer reads as a valid orphan/index row.
- The fix stays on the mirror-index proof input surface: URI is validated when present, but this slice does not make it required and does not add URI namespace/path validation or full mirror index schema validation.

## 为什么这是 v11.59

Artifact manifest verification already treats `uri` as string metadata before local artifact reads. Mirror index rows can also preserve URI proof metadata into artifact-store verification and GC outputs, so present index metadata should follow the same shape rule at the reader boundary.

## v11.59 不新增

- 不改变 artifact manifest metadata schema。
- 不实现 generic JSON Schema validation。
- 不改变 receipt artifact manifest comparison semantics。
- 不实现 remote artifact fetch。
- 不实现 object-store artifact backend。
- 不实现 artifact retention policy。
- 不实现 scheduler-owned routing。
- 不实现 dynamic Swarm decomposition。
- 不实现 A2A/ARD compatibility。
