# Agent Space v8.38 Boundary

状态：complete
目标：Human Gateway can read receipt-scoped artifact bytes only after receipt artifact verification succeeds.

## v8.38 新增

- `GET /api/artifacts/read?task_id=<id>&uri=<artifact-uri>` returns artifact bytes for an artifact referenced by that task receipt.
- The endpoint reuses the existing task-scoped audit proof lookup.
- The endpoint verifies the receipt's artifact manifests, local bytes, named sidecars, digest sidecars, and optional filesystem mirror before returning bytes.
- The response `Content-Type` comes from the signed receipt artifact manifest.
- Human Gateway receipt artifact rows render a `read` link next to each artifact, manifest, and verify link.
- The Go integration test checks both the API response bytes and the page read link.

## 为什么这是 v8.38

v8.37 proved a Human Gateway artifact can be verified from the signed receipt.
v8.38 makes the next smallest useful artifact access path:

```text
receipt task_id + artifact uri
  -> signed receipt proof
  -> existing artifact verifier
  -> local artifact bytes
```

This keeps artifact reads tied to receipt evidence without building an artifact browser.

## v8.38 不新增

- 不做 artifact browser。
- 不做 transcript browser。
- 不做 remote artifact fetch/download。
- 不做 object-store backend。
- 不做 artifact GC。
- 不做 auth model for artifact reads。
- 不做 streamed transcript UI。
- 不做 container namespace sandbox。
- 不做 public deployment。
- 不做 A2A/ARD compatibility。
