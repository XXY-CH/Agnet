# Agent Space v12.42 Boundary

状态：complete
目标：Reject non-string package identity metadata before string interpolation.

## v12.42 新增

- `asp-verify.mjs package-proof <manifest.json>` rejects empty or non-string `name`, `version`, and `filename` values with `package proof identity invalid`.
- Package identity binding now happens only after those identity fields are scalar strings.
- `test/package-contract.test.mjs` covers an array-valued package `name` that previously passed through JavaScript string coercion.

## 为什么这是 v12.42

The package proof identity check binds `filename` to `<name>-<version>.tgz`. Without scalar guards, JavaScript template interpolation can coerce arrays or objects into strings and accept malformed proof metadata.

## v12.42 不新增

- 不实现 JSON Schema。
- 不实现 tarball member proof。
- 不实现 npm registry signing。
- 不实现 release transparency。
- 不实现 SBOM。
- 不实现 package publish。
- 不实现 hosted public node。
- 不实现 real outside-host reachability。
- 不实现 scheduler-owned routing。
- 不实现 A2A/ARD compatibility。

This is package identity shape validation for the local package proof verifier, not a generic package release trust system.
