# Agent Space v8.22 Boundary

状态：complete
目标：Audit verification checks local artifact manifest sidecars against signed receipt manifests.

## v8.22 新增

- `--verify-audit` reads each named local artifact `.manifest.json` sidecar.
- The verifier rejects sidecars that differ from the receipt's signed artifact manifest.
- The Go integration test tampers with the sidecar after a successful audit verify and expects `--verify-audit` to fail.

## 为什么这是 v8.22

v8.19 made artifact manifests durable as sidecars, and v8.21 made audit verification check artifact bytes.

v8.22 closes the sidecar verification loop:

```text
receipt artifact manifest
  -> local .manifest.json sidecar
  -> sidecar equality check
  -> audit verification result
```

## v8.22 不新增

- 不校验 digest-addressed sidecar。
- 不做 remote artifact fetch。
- 不校验 remote object store。
- 不做 artifact GC。
- 不做 artifact browser。
- 不做 full transcript artifact storage。
- 不做 container sandbox。
- 不做 public deployment。
- 不做 A2A/ARD compatibility。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 test/go-fed-discovery.test.mjs
node --test --test-concurrency=1 test/*.test.mjs
```

The Go integration test must fail if `--verify-audit` accepts a receipt whose local named artifact sidecar no longer matches the signed manifest.
