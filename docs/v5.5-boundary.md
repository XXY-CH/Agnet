# Agent Space v5.5 Boundary

状态：complete
目标：让 external/MCP tool command identity 进入 sandbox evidence 和 signed proof。

## v5.5 新增

- External/MCP sandbox evidence records `tool_command_digest`。
- `sandbox_proof` signs the same digest because it signs sandbox evidence。
- Remote audit proof returns the command digest inside receipt sandbox evidence。

## 为什么这是 v5.5

Ultimate trust needs to know not only that a tool ran, but which local command identity was executed。

v5.5 adds the smallest provenance hook:

```text
tool_command
  -> digest
  -> sandbox evidence
  -> signed sandbox_proof
  -> receipt / remote audit proof
```

This avoids storing full local paths or command arguments in the protocol proof while still binding execution to a stable command identity。

## v5.5 不新增

- 不做 tool registry。
- 不做 binary hash。
- 不做 package signature verification。
- 不暴露完整 local command line。
- 不做 supply-chain policy。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 test/*.test.mjs
```

The Go integration test must fail if external/MCP sandbox evidence lacks a command digest or the signed sandbox proof does not carry the same digest。
