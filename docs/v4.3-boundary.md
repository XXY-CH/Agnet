# Agent Space v4.3 Boundary

状态：v4.3 complete
目标：给 capability credential 增加 Zone-signed status evidence。

## v4.3 新增

- Go `FED_QUERY` returns `credential_statuses` with credentials。
- Credential status object includes:
  - `issuer`
  - `credential_id`
  - `subject`
  - `status`
  - `status_signature`
- `credential_id` is derived from the credential body digest。
- Node query verifier checks credential status signature when status evidence is present。
- Query verifier rejects non-`active` statuses。
- Node integration test verifies CLI query status, raw WebSocket status, credential id, and status signature。

## 为什么这是 v4.3

v4.2 proved canonical policy scope evidence。

v4.3 starts credential status evidence:

```text
capability credential
  -> credential_id
  -> Zone-signed status
  -> query verifier checks active status
```

This adds revocation-ready evidence without building a public status service。

## v4.3 不新增

- 不做 OCSP-like public service。
- 不做 distributed revocation network。
- 不做 status cache。
- 不做 credential renewal。
- 不做 task-time credential negotiation。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 test/*.test.mjs
```

The Go integration test must fail if query credential status is missing, unsigned, or not active。
