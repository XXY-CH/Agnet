# Agent Space v8.30 Boundary

状态：complete
目标：Direct Human Gateway approvals can derive a local actor from a configured bearer approval session.

## v8.30 新增

- `--human-actor-policy` accepts an `approval_sessions` map.
- `approval_sessions` maps bearer tokens to local `human://...` actors.
- `/api/approvals/actions` can omit `actor` when the bearer token maps to a local approval session.
- The session-derived actor still passes the existing `approval_actions` allowlist.
- Existing explicit body actors remain supported for local compatibility.
- The Go integration test approves a direct tool task without a body actor and verifies the signed approval grant uses `human://operator`.

Example:

```json
{
  "approval_actions": {
    "human://operator": ["approve"]
  },
  "approval_sessions": {
    "local-token": "human://operator"
  }
}
```

## 为什么这是 v8.30

v8.25 preserved named approval actors.
v8.26 put those actors behind a local allowlist.
v8.30 adds the smallest login-backed step:

```text
Authorization: Bearer <token>
  -> approval_sessions[token]
  -> human://operator
  -> approval_actions allowlist
  -> signed approval grant
```

This keeps the Human Gateway moving toward real human identity without adding a login UI or server-side account system yet.

## v8.30 不新增

- 不做 login UI。
- 不做 password/session store。
- 不做 roles/admin model。
- 不做 token rotation/storage。
- 不做 dynamic policy service。
- 不做 public deployment。
- 不做 A2A/ARD compatibility。
