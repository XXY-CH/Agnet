# Agent Space v3.8 Boundary

状态：v3.8 complete
目标：给外部/MCP tool execution 增加 approval gate。

## v3.8 新增

- Worker policy can require `approval_required: ["tool"]`。
- Go requires tool approval before running:
  - `external.stdio`
  - `mcp.stdio`
- Go emits:
  - `approval.required`
  - `approval.granted`
- Receipt records:
  - `approvals: ["tool"]`
  - updated `event_count`
- Node integration test verifies approval events and receipt approvals for MCP tool execution。

## 为什么这是 v3.8

v3.7 proved MCP `tools/call`。

v3.8 adds the first governance gate for external tool execution:

```text
task accepted
  -> approval.required
  -> approval.granted
  -> task started
  -> tool call
  -> artifact
  -> receipt records approval
```

This is still simulated approval, not a real Human Gateway approval workflow。

## v3.8 不新增

- 不做 login。
- 不做 approval UI。
- 不做 pending task queue。
- 不做 rejection path。
- 不做 approval signatures。
- 不做 container sandbox。
- 不做 file/network tool capabilities。

## 验收

```bash
node --test --test-concurrency=1 test/*.test.mjs
go test ./...
```

The Go integration test must fail if MCP tool execution skips the approval events。
