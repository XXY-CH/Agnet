# Agent Space v2.3 Boundary

状态：v2.3 complete
目标：让 Go discovery gateway 从 Go key store 读取签名密钥。

## v2.3 新增

- Go discovery gateway accepts:
  - `--authority-key`
  - `--worker-key`
- Key files store 32-byte Ed25519 seed as hex。
- Discovery fixture no longer needs signing seeds for gateway startup。
- Go gateway verifies the authority key matches the fixture Zone descriptor。
- Go gateway uses key store material to generate and sign:
  - worker descriptor
  - Zone binding
  - capability credential
- Node client continues to verify Go responses end to end。

## 为什么这是 v2.3

v2.2 已经让 Go 从 worker seed + profile 生成 worker descriptor。

v2.3 把 signing seed 从 discovery fixture 移到 Go-owned key store：

```text
Go reads key files
  -> verifies authority key matches local Zone descriptor
  -> derives worker identity
  -> signs worker descriptor, Zone binding, and capability credential
  -> Node verifies
```

测试会用删除了 `authority_seed_hex`、`worker_seed_hex`、`worker` descriptor 和旧 `zone_binding` 的 fixture 启动 Go gateway，证明 gateway 不再从 fixture 读取 signing seeds 或搬运预签名 worker artifacts。

## v2.3 不新增

- 不做 key rotation。
- 不做 encrypted key store。
- 不做 descriptor registry。
- 不做 Go task execution。
- 不做 Go `FED_TASK_OPEN` verification。
- 不做 credential status feed。
