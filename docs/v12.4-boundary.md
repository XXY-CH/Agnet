# Agent Space v12.4 Boundary

状态：complete
目标：Make proof bundle proof-file paths fail closed before reading paths outside the bundle directory.

## v12.4 新增

- `asp-verify.mjs proof-bundle <bundle.json>` rejects empty, absolute, backslash-bearing, `.` segment, and `..` segment proof-file paths before file reads.
- The error names the unsafe bundle field, such as `bundle receipt_frame path invalid`.
- `public-node-proof.test.mjs` proves `../README.md` and `/tmp/receipt.json` in `receipt_frame` are rejected.

## 为什么这是 v12.4

v12.3 made bundle proof-file references self-relative, but a supplied bundle could still point the verifier at files outside the bundle directory. v12.4 keeps the existing proof-bundle verifier shape and closes only that path traversal boundary.

## v12.4 不新增

- 不实现 artifact byte relocation。
- 不实现 hosted public node。
- 不实现 external public reachability proof。
- 不实现 package publish/signing。
- 不实现 SBOM。
- 不实现 batch verification。
- 不实现 A2A/ARD compatibility。
- 不实现 scheduler-owned routing。

It does not make arbitrary bundle file layouts valid.
