# Agent Space v12.35 Boundary

状态：complete
目标：Run the external reachability observer from a Docker container without claiming real outside-host reachability.

## v12.35 新增

- `scripts/docker-external-reachability-observer.sh <bundle.json> <observed-bundle.json> <observer-trusted-zones.json>` runs the existing observer script inside Docker.
- The wrapper uses `${AGNET_NODE_BASE_IMAGE:-node:22-bookworm-slim}` so constrained Docker environments can override the Node base image.
- The wrapper mounts the current repository at `/app` and delegates directly to `node scripts/external-reachability-observer.mjs "$@"`.
- The wrapper adds `host.docker.internal` through Docker's host gateway so a containerized observer can probe a host-side proof target.
- `docker-demo.test.mjs` covers the wrapper contract without making Docker a required full-suite dependency.

## 为什么这是 v12.35

v12.34 made external reachability evidence runnable, but the observer still ran in the same local process environment by default. v12.35 adds the smallest independent runtime boundary around that observer. This is useful rehearsal for a real outside-host probe while preserving the verifier-owned evidence gate from v12.33.

## v12.35 不新增

- 不实现 hosted public node。
- 不证明 observer 运行在另一台物理主机。
- 不把 Docker bridge 证明描述成公网可达证明。
- 不增加 DNS, tunnel, NAT traversal, TLS, QUIC, or remote probe infrastructure。
- 不让 bundle 自己声明 `reachability_scope`。
- 不把 observer trusted-Zone descriptor 嵌进 bundle。
- 不实现 package signing。
- 不实现 SBOM。
- 不实现 scheduler-owned routing。
- 不实现 A2A/ARD compatibility。

This is a containerized observer runner, not outside-host deployment proof by itself.
