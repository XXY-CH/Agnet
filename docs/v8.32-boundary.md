# Agent Space v8.32 Boundary

状态：complete
目标：Human Gateway exposes read-only local approval session state.

## v8.32 新增

- `GET /api/session` reports whether the request maps to a configured local approval session.
- `GET /api/session` reports the session-derived `approval_actor`.
- `GET /api/session` reports the actor's configured `approval_actions`.
- `GET /api/session` reports whether a Human Gateway write token is required.
- Missing or unknown bearer tokens return `authenticated: false`.
- The endpoint never lists all configured sessions.
- The Go integration test checks both bearer-backed and anonymous session state responses.

## 为什么这是 v8.32

v8.30 and v8.31 made local bearer approval sessions meaningful for direct approvals.
v8.32 adds the smallest read surface above that:

```text
Authorization: Bearer <token>
  -> approval_sessions[token]
  -> /api/session
  -> approval_actor + approval_actions
```

This is enough for a later Human Gateway UI to show "who am I approving as" without adding a login product yet.

## v8.32 不新增

- 不做 login UI。
- 不做 password/session store。
- 不做 roles/admin model。
- 不做 token rotation/storage。
- 不做 CSRF/session cookies。
- 不做 dynamic policy service。
- 不做 public deployment。
- 不做 A2A/ARD compatibility。
