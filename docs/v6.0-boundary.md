# Agent Space v6.0 Boundary

状态：complete
目标：把 Go task 从一次性 receipt 推进到最小 durable state。

## v6.0 新增

- Completed/cancelled Go tasks write one JSON state file per task。
- The state file records:
  - `task_id`
  - `status`
  - `worker`
  - `receipt_digest`
- The task state directory is derived from the audit log path, for example:

```text
state/go-fed-discovery-audit.log
  -> state/go-fed-discovery-audit-tasks/
```

## 为什么这是 v6.0

v5 closed the execution evidence chain。

v6 starts the durable runtime layer:

```text
task execution
  -> signed receipt
  -> receipt digest
  -> per-task durable state file
```

This gives later cancellation, restore, task list, and Human Gateway actions a concrete state surface。

## v6.0 不新增

- 不做 scheduler queue。
- 不做 live external/MCP process interruption。
- 不做 checkpoint restore。
- 不做 task search index。
- 不做 database。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 test/*.test.mjs
```

The Go integration test must fail if completed or cancelled tasks do not persist receipt-linked task state files。
