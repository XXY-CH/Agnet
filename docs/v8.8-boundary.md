# Agent Space v8.8 Boundary

状态：complete
目标：Human Gateway page can rotate the browser-held requester key and produce a browser-side Agent rotation proof without giving either private key to Go.

## v8.8 新增

- Browser Requester Key panel includes `Rotate Key`.
- Rotation requires an existing browser requester key bundle in `localStorage`.
- The browser generates a new Ed25519 requester key.
- The browser stores the new requester key bundle under `agent-space-browser-requester`.
- The stored bundle keeps `previous_descriptor`.
- The stored bundle includes `rotation_proof` with:
  - `previous_aid`
  - `next_aid`
  - `previous_signature`
  - `next_signature`

## 为什么这是 v8.8

v0.6 already defined the Agent key rotation proof shape: old key signs the transition, new key signs the same transition.

v8.8 brings that ceremony into the browser requester key surface:

```text
old browser requester key
  -> new browser requester key
  -> { previous_aid, next_aid }
  -> old-key signature + new-key signature
  -> stored browser bundle
```

This proves requester key continuity locally without making Go a browser-key custodian.

## v8.8 不新增

- 不做 Zone alias rebinding。
- 不做 server-side rotation registry。
- 不做 automatic key rotation schedule。
- 不做 multi-key account manager。
- 不做 encrypted key store。
- 不做 passphrase-protected export。
- 不做 compromised-key recovery。
- 不做 login/session identity。
- 不做 public deployment。
