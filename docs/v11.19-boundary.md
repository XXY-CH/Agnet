# Agent Space v11.19 Boundary

状态：complete
目标：Make the Node `FED_SWARM_CLOSE` verifier reject malformed step receipt entries with a protocol error.

## v11.19 新增

- Node `verifySwarmClose` rejects non-object entries inside signed `step_receipts` before reading step fields.
- The Node conformance test proves a signed `FED_SWARM_CLOSE` body with `step_receipts: [null]` fails with `swarm close step receipt missing`, not a JavaScript `TypeError`.
- The portable Swarm close verifier keeps its boundary structural: signing Zone object, close proof object, close signature, signed Swarm id, frame/body Swarm id match, and step receipt object shape.

## 为什么这是 v11.19

`FED_SWARM_CLOSE` is a verifier-facing proof frame. A signed close proof with malformed step entries should be rejected as bad protocol evidence, not leak runtime implementation errors. v11.19 keeps the fix to one verifier guard and one red test.

## v11.19 不新增

- 不实现 generic Swarm close schema validation。
- 不实现 Node audit-backed Swarm completeness verification。
- 不验证 step receipts against an audit log。
- 不实现 dynamic Swarm decomposition。
- 不实现 scheduler-owned routing。
- 不实现 parallel or cross-Zone Swarm execution。
- 不实现 public reachability proof。
- 不实现 A2A/ARD compatibility。
