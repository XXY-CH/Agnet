# Agent Space v10.2 Boundary

状态：complete
目标：Node `FED_RECEIPT` verification checks signed artifact manifest metadata when receipts carry it.

## v10.2 新增

- `verifyFederatedReceipt` validates optional `artifact_manifests`.
- The verifier checks manifest count against `artifact_refs`.
- The verifier checks manifest `uri`, required fields, `size`, and `manifest_hash`.
- A signed receipt with a bad artifact manifest hash is rejected even when the worker receipt signature is valid.

## 为什么这是 v10.2

v10.1 made Node receipts carry artifact manifests. v10.2 makes the existing Node receipt verifier enforce that metadata instead of treating it as inert signed JSON.

## v10.2 不新增

- 不读取 artifact bytes。
- 不做 Node artifact sidecar verifier。
- 不做 Node artifact verify/read API。
- 不做 content-addressed mirror。
- 不做 receipt store/search。
- 不做 Go package extraction。

## 验收

```bash
node --test --test-concurrency=1 test-vectors.test.mjs
node --test --test-concurrency=1 federation-gateway.test.mjs mvp-demo.test.mjs
go test ./...
node --test --test-concurrency=1 *.test.mjs
git diff --check
```

The vector test must fail before the verifier change because a validly signed receipt with a bad `artifact_manifests[0].manifest_hash` is accepted.
