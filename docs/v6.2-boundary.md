# Agent Space v6.2 Boundary

状态：complete
目标：签名 `FED_TASK_CANCEL` 到达时，中断正在运行的 external tool process。

## v6.2 新增

- In-memory runtime registry maps `task_id` to a cancel function。
- `FED_TASK_CANCEL` marks the task cancelled and calls the registered cancel function when present。
- External tool execution derives its timeout from the task context, so cancellation interrupts it before timeout。
- Cancelled task state remains durable through the v6.0 state file path。

## 为什么这是 v6.2

v5.2 proved cancellation as signed protocol evidence。

v6.2 makes cancellation operational for the first runtime path:

```text
running external tool
  -> signed FED_TASK_CANCEL
  -> runtime cancel function
  -> process context cancellation
  -> cancelled receipt + task state
```

This is the smallest honest live cancellation slice before a scheduler exists。

## v6.2 不新增

- 不做 persisted running registry。
- 不做 scheduler queue。
- 不做 distributed/multi-node cancellation。
- 不做 checkpoint restore。
- 不做 Human Gateway action button。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 test/*.test.mjs
```

The Go integration test must fail if a signed cancel receipt is produced but the running external process still times out instead of being cancelled。
