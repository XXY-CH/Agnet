# Agent Space v7.4 Boundary

状态：complete
目标：failed queue item 可以被显式 retry 重新排队，并记录 retry/backoff 状态。

## v7.4 新增

- Go gateway accepts `FED_QUEUE_RETRY` with `retry_of`, `requester`, and a new signed `task`。
- Retry only accepts a failed queue item as parent。
- Retry creates a new queued item that records:
  - `retry_of`
  - `retry_attempt`
  - `retry_after_at`
- `FED_QUEUE_CLAIM` rejects queued items whose `retry_after_at` is still in the future。

## 为什么这是 v7.4

v5.3 proved direct retry lineage。
v7.3 proved queue ownership can recover after lease expiry。

v7.4 connects retry lineage to the durable queue without adding an automatic scheduler loop:

```text
failed queue item
  -> FED_QUEUE_RETRY(new signed task, retry_after_at)
  -> queued retry item
  -> claim rejected until retry_after_at
```

## v7.4 不新增

- 不做 automatic retry。
- 不做 automatic backoff scan。
- 不做 retry worker。
- 不做 priority。
- 不做 multi-node scheduler。

## 验收

