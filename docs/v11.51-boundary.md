# Agent Space v11.51 Boundary

状态：complete
目标：Make Go filesystem artifact mirror index matching require exact manifest field values and types instead of string-coerced equality.

## v11.51 新增

- Go `artifactStoreIndexContains` compares mirror `objects.ndjson` entries to receipt artifact manifests with exact Go/JSON field values.
- A mirror index entry with `size: "7"` no longer matches a manifest with numeric `size: 7`.
- Focused Go test covers the previous `fmt.Sprint` coercion hole.
- The fix is scoped to artifact-store index proof matching; artifact manifest metadata, local bytes, mirror bytes, and sidecar verification stay unchanged.

## 为什么这是 v11.51

Filesystem artifact mirror proof depends on the mirror `objects.ndjson` index matching the signed receipt artifact manifest. The old matcher stringified both sides, so a malformed mirror index entry could preserve proof shape while changing field types. Artifact mirror index proof should fail closed when index fields are not the same values as the receipt manifest.

## v11.51 不新增

- 不改变 artifact manifest metadata schema。
- 不实现 generic JSON Schema validation。
- 不改变 receipt artifact manifest comparison semantics。
- 不实现 remote artifact fetch。
- 不实现 object-store artifact backend。
- 不实现 artifact retention policy。
- 不实现 scheduler-owned routing。
- 不实现 dynamic Swarm decomposition。
- 不实现 A2A/ARD compatibility。
