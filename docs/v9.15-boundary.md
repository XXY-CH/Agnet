# Agent Space v9.15 Boundary

状态：complete
目标：Go sandbox probe reports configured container runtime candidates without claiming container isolation.

## v9.15 新增

- `--sandbox-probe container-namespace` reads optional `AGNET_CONTAINER_RUNTIME`.
- Without `AGNET_CONTAINER_RUNTIME`, the probe reports `runtime_configured: false` and `runtime_available: false`.
- With a discoverable runtime command, the probe reports `runtime_configured: true`, `runtime_available: true`, `runtime_command`, and `runtime_path`.
- `--sandbox-require container-namespace` still exits non-zero because isolated container namespace execution is not implemented.
- Failed unsupported container-claim task state persists the same structured runtime candidate evidence in `sandbox_probe`.

## 为什么这是 v9.15

Ultimate needs real sandbox execution, but the truthful next step is runtime preflight evidence.
v9.15 lets operators and tests distinguish "no runtime configured" from "runtime candidate present but execution not implemented" while keeping `supported: false`.

## v9.15 不新增

- 不实现 container namespace sandbox。
- 不执行 Docker/OCI container tasks。
- 不做 Linux namespace setup。
- 不做 seccomp/AppArmor/profile enforcement。
- 不做 network namespace。
- 不做 filesystem mount namespace。
- 不做 cgroup limits。
- 不做 remote attestation。
- 不做 scheduler or automatic drain。
- 不做 A2A/ARD compatibility。
