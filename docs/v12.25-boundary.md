# Agent Space v12.25 Boundary

状态：complete
目标：Make package proof tarball verification resolve safe tarball paths relative to the package proof manifest file.

## v12.25 新增

- `asp-verify.mjs package-proof <manifest.json>` resolves safe `tarball` paths relative to the manifest file before reading tarball bytes.
- `scripts/package-proof.mjs` writes `tarball` and `manifest` as package-directory-relative file names.
- `package-contract.test.mjs` covers a nested package proof manifest with a sibling tarball.

## 为什么这是 v12.25

v12.24 made package proof tarball paths fail closed before byte reads. v12.25 keeps that safety gate: the generated package proof directory can be copied and verified by pointing the verifier at the manifest file and reading the tarball beside it.
