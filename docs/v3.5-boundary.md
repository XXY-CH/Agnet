# Agent Space v3.5 Boundary

状态：v3.5 complete
目标：让 Go worker execution 通过最小 tool adapter。

## v3.5 新增

- Worker profile can set `tool`。
- Go execution calls a built-in pure-text tool adapter。
- Supported proof tools:
  - `summarize.mock`
  - `translate.mock`
  - `text.echo`
- Tool output becomes the task artifact。
- Receipt records the selected tool。
- Node integration test verifies:
  - selected translator worker uses `translate.mock`
  - artifact content comes from the tool output
  - receipt includes `tool`
  - Human Gateway serves the tool-produced artifact

## 为什么这是 v3.5

v3.4 added a human-facing observer。

v3.5 changes execution from a fixed artifact string into a tool adapter call:

```text
worker profile
  -> selected worker
  -> selected tool
  -> constrained input envelope
  -> artifact output
  -> receipt includes tool
```

The sandbox boundary here is intentionally small: built-in pure-text tools receive task metadata and return text. They do not receive file, process, or network handles。

## v3.5 不新增

- 不做 MCP wire protocol。
- 不做 external process tools。
- 不做 OS/container sandbox。
- 不做 file read tools。
- 不做 network tools。
- 不做 approval-gated tools。

## 验收

```bash
node --test --test-concurrency=1 test/*.test.mjs
go test ./...
```

The Go integration test must prove the artifact is tool-produced, not the old deterministic fallback。
