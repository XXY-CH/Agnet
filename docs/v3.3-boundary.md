# Agent Space v3.3 Boundary

状态：v3.3 complete
目标：给 Go federation frames 增加 WebSocket transport binding。

## v3.3 新增

- Go gateway accepts `--ws-port`。
- WebSocket binding carries the same JSON federation frames as local TCP。
- TCP and WebSocket share one frame handler。
- Node integration test uses a raw WebSocket client to prove:
  - HTTP Upgrade handshake
  - masked client text frame
  - unmasked server text frame
  - `FED_QUERY` over WebSocket
  - multi-worker capability routing still works

## 为什么这是 v3.3

v3.2 proved the Go runtime can route across multiple workers。

v3.3 starts moving transport from toy local TCP toward browser/proxy-friendly bindings:

```text
same FED_* JSON frame
  -> newline TCP
  -> WebSocket text frame
```

The protocol frame contract stays unchanged。

## v3.3 不新增

- 不做 TLS。
- 不做 auth handshake。
- 不做 WebSocket subprotocol negotiation。
- 不做 binary frames。
- 不做 fragmentation。
- 不做 reconnect/session resume。
- 不做 public deployment。

## 验收

```bash
node --test --test-concurrency=1 test/*.test.mjs
go test ./...
```

The Go integration test must prove `FED_QUERY` works through `--ws-port`。
