# Agent Space v8.7 Boundary

状态：complete
目标：Human Gateway page can export and import the browser-held requester key bundle without giving the private key to Go.

## v8.7 新增

- Browser Requester Key panel includes `Export Key`.
- Browser Requester Key panel includes `Import Key`.
- Export writes the current requester `{ descriptor, privateJwk }` bundle into a browser text field.
- Import reads that same bundle shape from the browser text field and stores it under `agent-space-browser-requester`.
- Imported keys reuse the existing browser-side signed draft path to `POST /api/queue/drafts`.

## 为什么这是 v8.7

v8.6 proved the first browser-held requester key and signed draft flow.

v8.7 adds the smallest useful key portability surface:

```text
browser localStorage requester key
  -> exported JSON bundle
  -> imported JSON bundle
  -> same browser signing path
```

This keeps Go out of private-key custody while making the browser requester identity movable across browser sessions by explicit human action.

## v8.7 不新增

- 不做 encrypted key store。
- 不做 passphrase-protected export。
- 不做 key rotation ceremony。
- 不做 multi-key account manager。
- 不做 cross-browser sync。
- 不做 login/session identity。
- 不做 server-side key custody。
- 不做 public deployment。
