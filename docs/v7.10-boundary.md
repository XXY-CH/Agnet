# Agent Space v7.10 Boundary

状态：complete
目标：queue action grants carry explicit scope and expiry.

## v7.10 新增

- Action grant requires `scope.actions`.
- Requested action must be included in `scope.actions`.
- Action grant requires `expires_at`.
- Expired grants are rejected before applying the queue action.
- Failed grant checks are still recorded as `go_queue_action` audit records.

## 为什么这是 v7.10

v7.9 proved queue actions can require signed grants.

v7.10 makes those grants bounded:

```text
signed grant
  -> scope.actions includes requested action
  -> expires_at is in the future
  -> apply queue action
  -> audit result
```

## v7.10 不新增

- 不做 user/session identity。
- 不做 grant revocation list。
- 不做 replay nonce。
- 不做 browser-side signing UX。
- 不开放非 localhost。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 test/go-fed-discovery.test.mjs
node --test --test-concurrency=1 test/*.test.mjs
```

The Go integration test must fail if an expired grant can perform a queue action, if an out-of-scope action can reuse a grant, or if failed grant checks are not captured as audit evidence.
