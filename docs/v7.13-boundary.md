# Agent Space v7.13 Boundary

状态：complete
目标：Human Gateway queue actions pass a local actor policy before execution.

## v7.13 新增

- Human Gateway queue action actor policy runs after actor binding and before applying the action.
- The local proof policy allows `human://local` for `enqueue`, `claim`, and `drain`.
- Other actors are rejected before applying the queue action.
- Actor policy failures are still recorded as `go_queue_action` audit records.

## 为什么这是 v7.13

v7.12 proved signed queue action grants can bind to a local actor string.

v7.13 adds the first local policy gate for that actor:

```text
signed grant
  -> actor matches request body
  -> local actor policy allows action
  -> signature / scope / expiry / replay checks pass
  -> apply queue action
  -> audit result
```

## v7.13 不新增

- 不做 login/session identity。
- 不做 role model。
- 不做 actor policy config file。
- 不做 persistent user accounts。
- 不做 browser-side signing UX。
- 不开放非 localhost。
