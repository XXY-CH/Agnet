# Agent Space v0.5 Boundary

状态：v0.5 candidate  
目标：让 runtime 身份跨进程重启保持稳定。

## v0.5 新增

- Runtime Agent 私钥持久化到 `state/keys/agent-local-summarizer.pkcs8`。
- Runtime requester 私钥持久化到 `state/keys/agent-local-requester.pkcs8`。
- Local Zone 私钥持久化到 `state/keys/zone-local.pkcs8`。
- 私钥文件使用 PKCS#8 DER。
- 新建私钥文件时设置权限 `0600`。
- 新增 `loadOrCreateAgent()`。
- 新增 `loadOrCreateZone()`。
- 新增 runtime 重启后 worker `aid` 稳定性测试。

## 为什么这是 v0.5

v0.4 定义了：

```text
aid       Agent key controls identity
zid       Zone key controls zone identity
agent://  Zone signs routing alias
```

但如果 runtime 每次启动都生成新 key，`aid` 和 `zid` 只在单进程生命周期内有效。

v0.5 让身份进入真实运行时：

```text
same key file
  -> same public key
  -> same aid/zid
  -> same descriptor identity across restart
```

后续 key rotation proof 边界见 `docs/v0.6-boundary.md`。

## v0.5 不新增

- 不做加密 keystore。
- 不做硬件密钥。
- 不做 key rotation。
- 不做 revocation。
- 不做 multi-instance `iid`。
- 不做 cloud secret manager。

这些留给安全运行时阶段。
