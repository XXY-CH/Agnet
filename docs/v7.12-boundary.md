# Agent Space v7.12 Boundary

状态：complete
目标：queue action grants bind to a local actor string.

## v7.12 新增

- Human Gateway queue action grants require `actor`.
- The request body must carry the same `actor`.
- Missing request `actor` is rejected before applying the queue action.
- Actor mismatch is rejected before applying the queue action.
- Actor mismatch failures are still recorded as `go_queue_action` audit records.

## 为什么这是 v7.12

v7.11 made successful queue action grants one-time within local audit history.

v7.12 binds the signed grant to the local actor asking for the action:

```text
signed grant
  -> actor matches request body
  -> signature / scope / expiry / replay checks pass
  -> apply queue action
  -> audit result
```

## v7.12 不新增

- 不做 login/session identity。
- 不做 actor authorization policy。
- 不做 persistent user accounts。
- 不做 browser-side signing UX。
- 不开放非 localhost。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 test/go-fed-discovery.test.mjs
node --test --test-concurrency=1 test/*.test.mjs
```

The Go integration test must fail if a queue action can omit the request actor or use a grant signed for a different local actor.
