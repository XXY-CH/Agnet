# v14.4 Boundary — Task failure migration

状态：complete
协议标签：`v14.4-protocol`

## 目标

v14.4 adds a narrow local-first Swarm failure migration proof surface: when a Swarm step worker fails, the gateway retries the step once on the next available different worker with the same capability and records the migration in the signed Swarm close proof.

## v14.4 新增

- Node `FED_SWARM_OPEN` retries a failed step once on a different same-capability worker.
- Go `FED_SWARM_OPEN` and `FED_SWARM_SCHEDULE` retry a failed step once on a different same-capability worker.
- `FED_SWARM_CLOSE.close.migration_log` is always part of the signed close body. It is `[]` when no migration occurred.
- Each migration entry carries `step_id`, `original_worker_aid`, `reason`, `migrated_to_worker_aid`, and `migration_at`.
- `migrated_to_worker_aid` names the replacement worker that produced the final successful step receipt.
- `migration_at` is an ISO UTC timestamp, and `close_signature` covers the whole close body including `migration_log`.
- `verifySwarmClose` validates migration log shape and rejects entries whose `step_id` does not reference a real close step receipt.

## v14.4 不新增

- 不做 invisible retry loops。
- 不做 more than one automatic retry per step。
- 不做 model KV/cache restore。
- 不做 distributed worker pool。
- 不做 automatic decomposition or human-free high-risk escalation。
- 不给 individual migration log entries 单独签名；`close_signature` signs the close body.

## 为什么这是 v14.4

v14.1 made Swarm worker commitments explicit before execution. v14.4 adds the next accountability surface for failed execution: migration is visible in `FED_SWARM_CLOSE`, the original failed worker remains named, and the final receipt remains bound to the replacement worker.

## 验收

- Node and Go tests prove a failing Swarm step migrates to a same-capability replacement worker.
- Close proof verification accepts valid `migration_log` evidence signed by the close signature.
- Close proof verification rejects a `migration_log` entry whose `step_id` is not present in `step_receipts`.
- Public docs and docs-contract tests name `migration_log` and `migrated_to_worker_aid` without claiming distributed worker pools or hidden retries.
