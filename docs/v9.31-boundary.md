# Agent Space v9.31 Boundary

状态：complete
目标：Swarm step identity uniqueness is enforced even when a completed step receipt has no artifact manifests.

## v9.31 新增

- `--verify-audit` records each completed Swarm `swarm_id` + `step_id` before checking artifact manifests.
- Duplicate Swarm step receipts are rejected even when `artifact_refs` and `artifact_manifests` are both empty.
- If a downstream receipt depends on an artifactless completed step, verification now reports `swarm dependency artifact missing` instead of treating the step as missing.
- Go package coverage proves an exact duplicate artifactless Swarm receipt is rejected.

## 为什么这是 v9.31

v9.30 rejected duplicate Swarm step receipts only when the receipt exposed at least one artifact manifest.
v9.31 makes the identity rule independent from artifact production: a Swarm step id names one completed step, whether or not that step emitted an artifact.

## v9.31 不新增

- 不做 artifactless dependency semantics。
- 不做 duplicate step merge 语义。
- 不做 Swarm retry step semantics。
- 不做 cross-audit dependency lookup。
- 不做 remote audit sync。
- 不做 scheduler。
- 不做 dynamic decomposition。
- 不做 parallel DAG execution。
- 不做 conflict resolution。
- 不做 Swarm UI。
- 不做 Node Swarm server。
- 不做 container namespace sandbox。
- 不做 A2A/ARD compatibility。
