# v13.14 Boundary: Multi-Signal Agent Score in Reputation

状态：complete

目标：Enrich FED_QUERY reputation evidence with local freshness, signed revocation count, and a labelled composite `agent_score` while keeping ranking deterministic and inspectable.

## What Landed

- Node and Go `FED_QUERY` now return `discovery_evidence.reputation.completed_receipts`, `last_completed_at`, `revocation_count`, and `agent_score` for each matched worker.
- `agent_score` exposes labelled components: `receipt_score`, `credential_score`, `freshness_score`, and `revocation_penalty`, plus capped non-negative `total`.
- `receipt_score` is derived from completed receipt count, `credential_score` from active credential state, `freshness_score` from recent completed receipt time, and `revocation_penalty` from signed local authority Zone revocation evidence.
- `ranking.score` is now computed from `agent_score.total` plus exact capability and semantic intent scores, so reputation contribution is inspectable in one labelled object.
- Missing receipt timestamps remain `last_completed_at: null` on Node and empty on Go before JSON conversion; they produce `freshness_score: 0` instead of inventing freshness.
- `federation-gateway.test.mjs`, `go-fed-discovery.test.mjs`, and `docs-contract.test.mjs` cover the multi-signal agent score reputation boundary.

## Non-Goals

- No remote reputation feed.
- No distributed registry.
- No public marketplace or token-weighted reputation.
- No scheduler integration.
- No predictive scoring or opaque model.

## Why This Boundary

v13.11 made receipt-count reputation audit-backed, and v13.13 made credential activity sensitive to signed authority Zone revocation. v13.14 folds those local signals into one labelled `agent_score` so callers can see why a worker ranked higher or lower without reverse-engineering `ranking.score`.

The score stays narrow and deterministic: completed receipts, active credential state, freshness from the latest completed receipt timestamp when present, and signed local revocation evidence. Revocations reduce the local reputation contribution through `revocation_penalty`; they do not create a remote blacklist or third-party scoring dependency.

## Test Contract

The tests must fail if:

- `discovery_evidence.reputation.agent_score` is missing from Node or Go query results.
- `freshness_score` or `revocation_penalty` is omitted from public docs or query evidence.
- `ranking.score` bypasses `agent_score.total` and reintroduces separate hidden credential or receipt boosts.
- Public docs describe the slice as a remote reputation feed, predictive scoring, opaque model, distributed registry, or public marketplace instead of deterministic local discovery evidence.
