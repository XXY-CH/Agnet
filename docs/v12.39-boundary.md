# Agent Space v12.39 Boundary

状态：complete
目标：Let package proof verification pin the signer to a caller-supplied trusted signer file.

## v12.39 新增

- `asp-verify.mjs package-proof <manifest.json> [trusted-signers.json]` accepts an optional trusted signer file.
- The trusted signer file may be `{ "signers": [...] }` or a raw descriptor array.
- The verifier validates trusted signer descriptors through the existing Agent descriptor path.
- When a trusted signer file is supplied, the package proof signer `aid` must appear in that trusted signer set.
- Successful trusted verification returns `signer_trusted: true`.
- `test/package-contract.test.mjs` covers trusted signer acceptance and untrusted signer rejection.

## 为什么这是 v12.39

v12.38 bound package proof metadata to an ASP signer identity, but the signer descriptor still lived inside the manifest being verified. v12.39 adds the smallest external trust root: a caller can pin acceptable signer descriptors without changing the package proof format or inventing a release transparency system.

## v12.39 不新增

- 不实现 npm registry signing。
- 不实现 release transparency。
- 不实现 SBOM。
- 不实现 package publish。
- 不实现 hosted public node。
- 不实现 real outside-host reachability。
- 不实现 scheduler-owned routing。
- 不实现 A2A/ARD compatibility。

This is a trusted-signer pin for the local package proof verifier, not a full package release trust system.
