# Agent Space v12.12 Boundary

状态：complete
目标：Make `proof-bundle` reject incomplete signed transport proofs.

## v12.12 新增

- `asp-verify.mjs proof-bundle <bundle.json>` now requires the verified signed receipt `transport_proof` to include non-empty `transport`, non-empty `listen_host`, decimal-string `port`, and `public_transport: true`.
- A bundle whose signed receipt transport proof only carries `public_transport: true` is rejected with `bundle transport_proof invalid`.
- `public-node-proof.test.mjs` proves this by re-signing a receipt with an incomplete transport proof and matching the bundle fields.

## 为什么这是 v12.12

v12.11 required `public_transport: true`, but a public proof bundle still needed the rest of the local listener evidence to be present. This keeps the public proof from collapsing into a bare boolean while still staying short of external-host reachability.

## v12.12 不新增

- 不实现 external public reachability proof。
- 不实现 hosted public node。
- 不改变 normal `fed-receipt` verification。
- 不改变 verifier JSON output。
- 不改变 Go receipt transport proof field shape。
- 不实现 DNS, TLS, QUIC, NAT traversal, or remote probe infrastructure。
- 不实现 batch verifier。
- 不实现 JSON Schema。
- 不实现 generic proof bundle schema。
- 不实现 package signing or SBOM。
- 不实现 A2A/ARD compatibility。
- 不实现 scheduler-owned routing。

It does not prove reachability from another host.
