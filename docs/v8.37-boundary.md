# Agent Space v8.37 Boundary

状态：complete
目标：Human Gateway verifies a receipt-scoped artifact against the signed receipt manifest.

## v8.37 新增

- `GET /api/artifacts/verify?task_id=<id>&uri=<artifact-uri>` returns `{ ok, task_id, uri, manifest }` for an artifact referenced by that task receipt.
- The endpoint reuses the existing task-scoped audit proof lookup.
- The endpoint verifies the receipt's artifact manifests, local bytes, named sidecars, digest sidecars, and optional filesystem mirror before returning success.
- Human Gateway receipt artifact rows render a `verify` link next to each artifact and manifest link.
- The Go integration test checks both the API response and the page verify link.

## 为什么这是 v8.37

v8.35 exposed artifact manifests.
v8.36 exposed task-scoped receipt proofs.
v8.37 joins those two surfaces into the smallest user-facing verification action:

```text
receipt task_id + artifact uri
  -> signed receipt proof
  -> receipt artifact manifest
  -> existing artifact verifier
  -> verification JSON
```

This keeps the Human Gateway on the Ultimate proof path without building a broad artifact product surface.

## v8.37 不新增

- 不做 artifact browser。
- 不做 transcript browser。
- 不做 remote artifact fetch/download。
- 不做 object-store backend。
- 不做 artifact GC。
- 不做 auth model for artifact reads。
- 不做 streamed transcript UI。
- 不做 container namespace sandbox。
- 不做 public deployment。
- 不做 A2A/ARD compatibility。
