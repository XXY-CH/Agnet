# Agent Space v12.10 Boundary

状态：complete
目标：Make every `asp-verify.mjs` command reject extra positional CLI arguments.

## v12.10 新增

- `asp-verify.mjs artifact <manifest.json>` now requires exactly one input path.
- `asp-verify.mjs fed-receipt <frame.json> <trusted-zones.json> [task.json]` and `fed-receipt-artifacts` now accept only the no-task and one-task arities.
- `asp-verify.mjs swarm-close <frame.json> <trusted-zones.json>` now rejects extra positional arguments.
- Existing `proof-bundle` exact arity remains unchanged.
- `mvp-demo.test.mjs` and `test-vectors.test.mjs` prove extra positional arguments reach the usage path instead of being ignored.

## 为什么这是 v12.10

v12.8 and v12.9 closed the same issue for `proof-bundle`. The sibling verifier commands still used truthiness-based dispatch, so an external verifier caller could pass extra tokens that were silently ignored or masked by a later validation error. This closes the shared CLI trust-boundary shape without adding a parser.

## v12.10 不新增

- 不增加 option parsing。
- 不改变 verifier JSON output。
- 不改变 command names or valid arities。
- 不实现 batch verification。
- 不实现 JSON Schema。
- 不实现 generic proof bundle schema。
- 不实现 artifact byte relocation。
- 不实现 hosted public node。
- 不实现 external public reachability proof。
- 不实现 package signing or SBOM。
- 不实现 A2A/ARD compatibility。
- 不实现 scheduler-owned routing。

It does not add option parsing.
