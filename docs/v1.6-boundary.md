# Agent Space v1.6 Boundary

状态：v1.6 candidate  
目标：为 v2.0 Go runtime 做 trust verifier preflight。

## v1.6 新增

- `test-vectors/asp-v1.5-capability-credential.json`
- Go verification for:
  - Zone `zid`
  - Zone descriptor signature
  - worker `aid`
  - worker descriptor signature
  - capability credential issuer/subject binding
  - capability credential signature

## 为什么这是 v1.6

v1.5 在 Node runtime 中实现 signed capability credential。

v1.6 证明 credential 不是 Node-only 结构：

```text
same Zone descriptor
same worker descriptor
same capability credential
  -> Go verifier accepts
```

这是 v2.0 Go Federation Gateway 的前置条件。

## v1.6 不新增

- 不做 Go gateway。
- 不做 Go registry service。
- 不做 Go policy engine。
- 不做 authority chain。
- 不做 credential revocation。

这是 verifier preflight，不是 runtime migration。

## v2.0 候选目标

v2.0 应该是 Go 版 Federation Gateway 的最小可运行实现：

```text
Go gateway verifies trusted Zone
Go gateway accepts FED_RESOLVE / FED_QUERY
Go gateway returns signed descriptors / credentials
Node requester can verify and use them
```

不要求 v2.0 一次性替代全部 Node runtime。
