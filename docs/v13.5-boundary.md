# Agent Space v13.5 Boundary

状态：complete
目标：Land the first scheduler-owned ready-DAG Swarm execution primitive without claiming upper-layer orchestration.

## v13.5 新增

- Go federation gateway accepts `FED_SWARM_SCHEDULE`.
- The frame carries the same signed Swarm DAG shape as `FED_SWARM_OPEN`.
- Out-of-order input steps execute in dependency-ready order and close proof records `scheduler.step_order`.
- Scheduled steps reuse the existing signed task execution path, per-step receipts, artifact dependency evidence, and audit-backed `FED_SWARM_CLOSE`.
- Close proof signs scheduler evidence with `scheduler.mode: "ready-dag"`.

## 硬边界

- `FED_SWARM_SCHEDULE` accepts a DAG; it does not invent the DAG.
- Close proof remains complete, ordered, deduplicated, digest-checked, and tied to the same audit.
- Out-of-order input steps execute in dependency-ready order and close proof records `scheduler.step_order`.
- This is not automatic decomposition, not parallel execution, not upper-layer master-agent orchestration, and not economic settlement.

## v13.5 不新增

- No automatic task decomposition, no parallel worker pool, no upper-layer master-agent orchestration, and no economic settlement in this slice.
- 不完成 real hosted external-host observer evidence。
- 不实现 strong sandbox/remote attestation。
