# Agent Space v11.3 Boundary

状态：complete
目标：Make task identifiers fail closed before they reach execution, queue state, artifacts, or audit evidence.

## v11.3 新增

- Node `verifyFederatedTaskOpen` rejects `task_id` values outside the implemented token format.
- Go `Fixture.verifyTaskOpen` rejects the same unsafe task ids before task execution, queue enqueue, resume, retry, or Swarm step execution can proceed.
- Go direct queue claim/reclaim/drain actions reject unsafe task ids before reading or writing queue state.
- The implemented `task_id` format is `^[A-Za-z0-9._:-]{1,128}$`.
- Node and Go tests prove `../...` task ids fail closed at federation task-entry boundaries.

## 为什么这是 v11.3

The attachment's Phase A still had one live trust-boundary bug after v11.1/v11.2: task ids were signed but not format-checked. Because task ids flow into artifact URIs, task state files, queue files, transcript files, and audit lookups, the smallest honest fix is a shared token boundary at task-open verification and queue task-id actions.

## v11.3 不新增

- 不实现 global task-id registry。
- 不实现 UUID migration。
- 不改变 existing valid task ids。
- 不实现 scheduler-owned task namespace。
- 不实现 public reachability proof。
- 不实现 A2A/ARD compatibility。
