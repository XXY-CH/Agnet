# Agent Space v8.2 Boundary

状态：complete
目标：pending Human Gateway approvals can deny or expire before tool execution.

## v8.2 新增

- Human Gateway `POST /api/approvals/actions` supports local `deny`.
- Pending approval state records `expires_at`.
- Denied approvals stop direct execution before `task.started`.
- Expired approvals stop direct execution before `task.started` and persist `expired` state.
- Approved tasks still emit the existing signed `approval.granted` evidence.

## 为什么这是 v8.2

v8.0 and v8.1 made tool execution wait for approval.

v8.2 makes pending approval a real control state instead of a one-way approve button:

```text
pending
  -> approve -> signed approval.granted -> tool execution
  -> deny    -> FED_TASK_ERROR before tool execution
  -> expire  -> FED_TASK_ERROR before tool execution
```

This is still local operator proof, not account-backed user identity.

## v8.2 不新增

- 不做 denial reason schema。
- 不做 approval role model。
- 不做 login/session identity。
- 不做 background expiry scanner。
- 不做 browser-side private-key storage。
- 不做 public transport/deployment hardening。

## 验收

```bash
go test ./...

node --test --test-concurrency=1 test/*.test.mjs
```

The Go integration test must fail if denied or expired approvals can reach `task.started`, if denied/expired state is not visible through `/api/approvals`, or if approved tasks lose signed approval grant evidence.
