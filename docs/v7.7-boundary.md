# Agent Space v7.7 Boundary

状态：complete
目标：checkpoint resume can enter the durable queue before execution.

## v7.7 新增

- Go gateway accepts `FED_QUEUE_RESUME`.
- Queue resume requires `checkpoint_id`.
- Queue resume rejects unknown checkpoints through the existing audit-backed checkpoint lookup.
- Queue resume creates a queued item with `resume_checkpoint`.
- `FED_QUEUE_DRAIN` executes that queued item with the stored parent checkpoint.
- The drained receipt records `resumed_from`, and the new checkpoint records `parent_checkpoint`.

## 为什么这是 v7.7

v5.1 proved immediate resume execution.
v7.6 proved Human Gateway and federation can create durable queued work.

v7.7 joins those paths:

```text
existing checkpoint evidence
  -> FED_QUEUE_RESUME
  -> durable queued item + resume_checkpoint
  -> explicit claim/drain
  -> receipt.resumed_from
```

## v7.7 不新增

- 不恢复 model KV/cache。
- 不做 state restore service。
- 不做 automatic scheduler loop。
- 不做 long-running MCP session resume。
- 不做 artifact merge。
