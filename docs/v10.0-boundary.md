# Agent Space v10.0 Boundary

状态：complete
目标：Add an Ed25519 `did:key` bridge for Agent descriptors while keeping `aid:` as the canonical Agent Space identity.

## v10.0 新增

- Node `createAgent` descriptors now include `did_key`.
- Node exports:
  - `didKeyFromPublicKey`
  - `didKeyFromPublicKeySPKI`
  - `didKeyFromDescriptor`
  - `publicKeySPKIFromDidKey`
- Node descriptor resolution validates an optional `did_key` against `public_key_spki`.
- Go-generated requester, worker, and local descriptors include `did_key`.
- Go descriptor verification validates an optional `did_key` against `public_key_spki`.
- Go package tests and Node vector tests prove the stable Ed25519 bridge:

```text
public_key_spki = MCowBQYDK2VwAyEAA6EHv_POEL4dcN0Y50vAmWfk1jCbpQ1fHdyGZBJVMbg
did_key         = did:key:z6MkehRgf7yJbgaGfYsdoAsKdBPE3dj2CYhowQdcjqSJgvVd
```

## 为什么这是 v10.0

v9 closed the accountability proof line around artifacts, audit, queue hardening, and Swarm receipt verification.
v10 starts by making Agnet identity easier to bridge into existing identity ecosystems without changing the internal security root.

`aid:` remains the canonical Agent Space identity. `did_key` is a deterministic bridge field for Ed25519 descriptors.

## v10.0 不新增

- 不做 DID-native identity。
- 不做 DID document。
- 不做 DID resolver。
- 不做 DID service endpoint routing。
- 不做 `did:web` / `did:peer` / other DID methods。
- 不做 ANP/A2A/AGNTCY compatibility。
- 不做 public registry。
- 不做 scheduler。
- 不做 dynamic Swarm decomposition。
- 不做 container namespace sandbox。
