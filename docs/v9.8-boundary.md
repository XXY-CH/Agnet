# Agent Space v9.8 Boundary

状态：complete
目标：one Go receipt record can be verified directly from a JSON file.

## v9.8 新增

- `--verify-receipt <file>` verifies one receipt record JSON file and exits.
- The verifier reuses the existing receipt-record checks from audit verification.
- It checks Zone descriptor, worker descriptor, Zone binding, receipt signature, approval grants, checkpoints, artifact manifests, policy scope, and sandbox proof.
- On success it prints `{"go_receipt_verify":"ok","task_id":"..."}`.
- The integration test writes the `FED_RECEIPT` record to a file and verifies it with the CLI.

## 为什么这是 v9.8

Ultimate's Trust & Verification layer needs receipts to be independently checkable.
Audit verification already checked receipts, but only as part of a full audit log.
This slice exposes the same verifier for one receipt record without adding a receipt database or new proof format.

## v9.8 不新增

- 不做 receipt store。
- 不做 receipt search。
- 不做 receipt export format beyond the existing record JSON shape。
- 不做 revocation feed or renewal checking。
- 不做 remote verifier service。
- 不做 batch receipt verification。
- 不做 A2A/ARD compatibility。

## 验收

```bash
node --test --test-concurrency=1 test/go-fed-discovery.test.mjs
go test ./...
node --test --test-concurrency=1 test/*.test.mjs
```

The integration test must fail if `--verify-receipt` is missing or does not reject through the shared receipt verifier.
