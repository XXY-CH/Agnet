# Agent Space v8.13 Boundary

状态：complete
目标：The local requester registry preserves multiple requester aliases instead of replacing the registry on each rebinding.

## v8.13 新增

- Successful `POST /api/requester/rebindings` upserts the rebound requester alias into `state/go-fed-discovery-requester-registry.json`.
- A rebinding for a new alias appends a new registry agent entry.
- A rebinding for an existing alias replaces that alias entry.
- The registry keeps the existing Node registry shape:
  - `zone`
  - `revocations`
  - `agents[].descriptor`
  - `agents[].zone_binding`
- Existing `loadRegistry` / `resolveAgent` can resolve more than one rebound requester alias from the same file.

## 为什么这是 v8.13

v8.10 persisted one rebound requester alias.
v8.12 made rebinding history inspectable.

v8.13 removes the single-alias overwrite:

```text
POST rebinding for agent://browser/requester
  -> registry contains requester
POST rebinding for agent://browser/assistant
  -> registry contains requester and assistant
```

This is the smallest useful multi-requester registry behavior without adding a browser account manager.

## v8.13 不新增

- 不做 browser multi-key manager。
- 不做 Human Gateway requester selector UI。
- 不做 alias delete/disable。
- 不做 conflict policy beyond same-alias replace。
- 不做 server-side rotation registry。
- 不做 remote registry sync。
- 不做 login/session identity。
- 不做 public deployment。
