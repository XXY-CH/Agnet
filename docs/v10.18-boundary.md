# Agent Space v10.18 Boundary

状态：complete
目标：Reject delimiter-breaking Swarm ids.

## v10.18 新增

- `FED_SWARM_OPEN` rejects `swarm_id`, `step_id`, and dependency ids containing NUL bytes.
- `--verify-audit` rejects Swarm receipt identities and close proof step identities containing NUL bytes.
- This protects the verifier's internal `swarm_id + "\x00" + step_id` key boundary from cross-identity confusion.
- `cmd/go-fed-discovery/main_test.go` proves NUL-bearing Swarm receipt ids are rejected.

## 为什么这是 v10.18

v10.17 required a close proof to close real same-audit work. v10.18 hardens the identity delimiter used to connect those same-audit Swarm receipts and close proofs.

## v10.18 不新增

- 不做 dynamic Swarm decomposition。
- 不做 scheduler-owned DAG execution。
- 不做 parallel Swarm execution。
- 不做 conflict/merge receipts。
- 不做 cross-Zone Swarm。
- 不做 Swarm UI。
- 不做 receipt store/search。
- 不做 A2A/ANP/AGNTCY compatibility。
- 不做 full Swarm id normalization beyond NUL delimiter rejection。
