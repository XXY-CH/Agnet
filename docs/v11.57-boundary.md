# Agent Space v11.57 Boundary

状态：complete
目标：Make Go filesystem artifact mirror index readers reject present sizes that are not non-negative integers.

## v11.57 新增

- Go `readArtifactStoreIndex` rejects present `size` fields that are strings, negative numbers, or fractional numbers with `artifact mirror index invalid`.
- Focused Go test proves index rows with `size: "7"`, `size: -1`, or `size: 1.5` no longer read as valid orphan/index rows.
- The fix stays on the mirror-index proof input surface: size is validated when present, but this slice does not make it required and does not add full mirror index schema validation.

## 为什么这是 v11.57

Mirror index rows feed both artifact-store verification and GC proof outputs. v11.51 stopped string-coerced index matching, but `readArtifactStoreIndex` could still preserve impossible size metadata in rows that were not matched against a receipt manifest. Present size metadata now follows the same non-negative integer rule already used by receipt artifact manifest verification.

## v11.57 不新增

- 不改变 artifact manifest metadata schema。
- 不实现 generic JSON Schema validation。
- 不改变 receipt artifact manifest comparison semantics。
- 不实现 remote artifact fetch。
- 不实现 object-store artifact backend。
- 不实现 artifact retention policy。
- 不实现 scheduler-owned routing。
- 不实现 dynamic Swarm decomposition。
- 不实现 A2A/ARD compatibility。
