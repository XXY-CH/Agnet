# v14.2 Boundary — Multi-signal FED_QUERY routing

状态：complete
协议标签：`v14.2-protocol`

## 目标

v14.2 adds labelled, deterministic routing evidence to `FED_QUERY` so route selection uses cost, latency, and availability alongside the existing capability, credential, reputation, freshness, revocation, and semantic signals.

## v14.2 新增

- `discovery_evidence.routing` exposes `cost_score`, `latency_score`, `availability_score`, and `signals_used` for each query match.
- Worker descriptor `policy.cost_tokens_per_task` maps lower cost to a higher `cost_score`; missing cost evidence falls back to neutral `5` and does not increment `signals_used`.
- Worker descriptor `policy.latency_ms_p95` maps lower p95 latency to a higher `latency_score`; missing latency evidence falls back to neutral `5` and does not increment `signals_used`.
- Local audit entries provide `availability_score` from completed versus started task evidence over the last 50 audit entries; missing availability evidence falls back to neutral `5` and does not increment `signals_used`.
- `agent_score.total` includes `cost_score`, `latency_score`, and `availability_score`, still capped at 100, and Node and Go both expose these score fields.

## v14.2 不新增

- 不做 opaque ML router。
- 不做 global reputation oracle。
- 不做 public marketplace。
- 不做 token-weighted ranking。
- 不做 predictive availability service without local audit evidence.

## 为什么这是 v14.2

v14.0 opened the local-first overlay/swarm/routing line. v14.1 added signed Swarm worker commitments. v14.2 turns FED_QUERY routing into an inspectable multi-signal ranking surface without adding a marketplace, global reputation oracle, or opaque router.

## 验收

- Node `FED_QUERY` and `queryMatch` expose `discovery_evidence.routing.cost_score`, `latency_score`, `availability_score`, and `signals_used`.
- Go semantic `FED_QUERY` exposes the same `routing` score object.
- Tests prove a lower-cost worker can outrank an equivalent worker with neutral cost evidence.
- Missing signal evidence stays neutral rather than inventing routing advantage.
