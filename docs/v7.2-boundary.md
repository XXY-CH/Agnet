# Agent Space v7.2 Boundary

状态：complete
目标：queued task 必须先被显式 claim，携带 `lease_id` 后才能 drain。

## v7.2 新增

- Go gateway accepts `FED_QUEUE_CLAIM` with `task_id` and `owner`。
- Claim changes queue state from `queued` to `claimed`。
- Claimed queue item records:
  - `lease_owner`
  - `lease_id`
- `FED_QUEUE_DRAIN` now requires matching `lease_id`。
- Drain changes queue state from `claimed` to `running` to `completed`。
- Completed queue item preserves the lease fields and records `receipt_digest`。

## 为什么这是 v7.2

v7.1 proved explicit queue drain。

v7.2 adds the scheduler ownership point:

```text
queued
  -> claim(owner)
  -> claimed + lease_id
  -> drain(lease_id)
  -> running
  -> completed
```

This prevents unowned drain and gives future crash recovery a durable ownership record。

## v7.2 不新增

- 不做 lease expiry。
- 不做 reclaim。
- 不做 automatic drain loop。
- 不做 retry/backoff。
- 不做 priority。
- 不做 multi-node scheduler。

## 验收

