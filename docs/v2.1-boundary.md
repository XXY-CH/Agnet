# Agent Space v2.1 Boundary

状态：v2.1 candidate  
目标：让 Go discovery gateway 动态签发 discovery artifacts。

## v2.1 新增

- `authority_seed_hex` test fixture field。
- Go discovery gateway derives authority private key from seed。
- Go discovery gateway dynamically signs:
  - Zone binding
  - capability credential
- Node client verifies Go-signed results.

## 为什么这是 v2.1

v2 Go gateway 只服务预签名 fixture。

v2.1 开始让 Go 掌握签发能力：

```text
Go reads authority seed
  -> signs zone binding
  -> signs capability credential
  -> Node verifies
```

这仍然不是完整 Go runtime，但它把可信发现面的关键签发动作移到了 Go。

## v2.1 不新增

- 不做 Go key store。
- 不做 Go task execution。
- 不做 Go audit writer。
- 不做 dynamic worker descriptor generation。
- 不做 credential status feed。

## 验收

```bash
node --test --test-concurrency=1 *.test.mjs
go test ./...
```

必须通过。
