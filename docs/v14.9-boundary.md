# v14.9 Cross-Netns Reachability Evidence

状态：complete

协议标签：`v14.9-protocol`

## What changed

v14.9 adds a third verifier-owned reachability tier, `cross-netns`, between `container-observer` and `external-host`.

`cross-netns` means a trusted observer ran from a separate network namespace / VM and TCP-connected to the gateway over a literal private IP. The verifier accepts it only for `vantage: "cross-netns"` when the signed receipt `listen_host` is a private inter-namespace IP: literal IP, not loopback or unspecified, and not globally routable.

The concrete motivating experiment used Apple `container` CLI 1.0.0 on macOS 27.0. The gateway VM had private IP `192.168.64.6/24`, and a second Apple container VM TCP-connected successfully: `{"reached":true,"from":"separate-container-vm","to":"192.168.64.6:9999"}`.

## Reachability tiers

- `container-observer`: trusted `vantage: "container"` evidence from the container observer boundary.
- `cross-netns`: trusted `vantage: "cross-netns"` evidence from a separate network namespace / VM over a private inter-namespace IP such as `192.168.64.6`.
- `external-host`: trusted `vantage: "external-host"` evidence only over a globally routable literal IP.

## Honest boundary

`cross-netns` is not public reachability. It does not prove hosted reachability, global routing, NAT traversal, or production external access. It does not collapse into `external-host` and does not complete external-host.

The v13.1 hosted external-host observer run is still pending. The recorded hosted attempt that failed with `ENETUNREACH` remains the current public-host blocker, and v14.9 does not weaken that pending language.

## Verifier behavior

`asp-verify.mjs proof-bundle` now reports `reachability_scope: "cross-netns"` only when trusted observer evidence binds `vantage: "cross-netns"`, the receipt transport proof, receipt digest, observed endpoint, freshness, and a private inter-namespace literal IP listen host.

The cross-netns gate fails closed for loopback listen hosts such as `127.0.0.1` and for globally routable listen hosts such as `93.184.216.34` with `external reachability cross-netns listen host not a private inter-namespace IP`.

`scripts/container-cross-netns-observer.sh` is the Apple-container wrapper for running the existing observer from a separate Apple `container` VM with `cross-netns` vantage. It uses `${AGNET_NODE_BASE_IMAGE:-node:24-alpine}`.
