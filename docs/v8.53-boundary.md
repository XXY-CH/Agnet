# Agent Space v8.53 Boundary

状态：complete
目标：filesystem artifact mirrors maintain a minimal content-addressed object index.

## v8.53 新增

- `--artifact-store` mirror writes append-only `objects.ndjson`.
- Each mirrored artifact entry records `uri`, `sha256`, `size`, `media_type`, and `manifest_hash`.
- The index is written beside the existing `by-sha256/<digest>` mirror files.
- The Go integration test checks the summary artifact and transcript artifact appear in the mirror index.

## 为什么这是 v8.53

v8.27-v8.28 made filesystem artifact mirrors writable and verifiable.
v8.53 adds the smallest inventory surface needed before any larger artifact lifecycle work:

```text
artifact manifest
  -> filesystem mirror by sha256
  -> objects.ndjson index line
```

## v8.53 不新增

- 不做 S3/MinIO/object-store API。
- 不做 remote artifact upload/download。
- 不做 artifact GC。
- 不做 index compaction/dedup。
- 不做 receipt/artifact schema change。
- 不做 artifact browser。
- 不做 auth model for mirrored artifacts。
- 不做 container namespace sandbox。
- 不做 public deployment。
- 不做 A2A/ARD compatibility。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 test/go-fed-discovery.test.mjs
node --test --test-concurrency=1 test/*.test.mjs
```

The Go integration test must fail if the filesystem artifact mirror no longer records mirrored artifact manifests in `objects.ndjson`.
