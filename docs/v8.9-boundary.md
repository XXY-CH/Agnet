# Agent Space v8.9 Boundary

状态：complete
目标：Human Gateway can issue a Zone-signed requester alias rebinding proof after verifying a browser requester rotation proof.

## v8.9 新增

- Human Gateway serves `POST /api/requester/rebindings`.
- The endpoint uses the existing local write-token gate when configured.
- The request provides:
  - `previous_descriptor`
  - `next_descriptor`
  - `rotation_proof`
- Go verifies both requester descriptors.
- Go verifies the browser-side Agent rotation proof.
- Go signs an `alias_rebinding_proof` with the local Zone authority key.
- The proof includes the embedded `agent_rotation_proof`.

## 为什么这是 v8.9

v8.8 proved browser-side requester key continuity:

```text
old requester key + new requester key
  -> rotation_proof
```

v8.9 adds the missing Zone authority approval step:

```text
rotation_proof
  -> Go verifies descriptors and both requester signatures
  -> Zone signs { zone, alias, previous_aid, next_aid }
  -> alias_rebinding_proof
```

This is the smallest bridge from browser key rotation to Zone-approved alias continuity.

## v8.9 不新增

- 不持久更新 registry alias binding。
- 不做 server-side rotation registry。
- 不做 browser UI for rebinding submission。
- 不做 automatic rebinding after rotation。
- 不做 multi-key account manager。
- 不做 compromised-key recovery。
- 不做 login/session identity。
- 不做 public deployment。
