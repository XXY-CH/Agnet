# Agent Space v8.12 Boundary

状态：complete
目标：Human Gateway keeps a local requester alias rebinding history that humans can inspect.

## v8.12 新增

- Successful `POST /api/requester/rebindings` appends a local history record to `state/go-fed-discovery-requester-rebindings.json`.
- Each record stores:
  - `zone`
  - `alias`
  - `previous_aid`
  - `next_aid`
  - `proof_digest`
  - `alias_rebinding_proof`
- `GET /api/requester/rebindings` returns the local history.
- The Human Gateway page renders a `Requester Rebindings` table.
- The table shows alias, previous requester `aid`, next requester `aid`, and the proof digest.

## 为什么这是 v8.12

v8.11 made rebinding explicit from the browser.

v8.12 leaves a local trail for that explicit action:

```text
Bind Alias
  -> verified Zone-signed alias rebinding proof
  -> registry update
  -> local rebinding history
  -> Human Gateway table
```

This makes requester key continuity inspectable without introducing multi-requester selection rules.

## v8.12 不新增

- 不做 multi-requester registry。
- 不做 server-side rotation registry。
- 不做 remote registry sync。
- 不做 automatic rebinding after rotation。
- 不做 audit hash-chain entry for rebinding history。
- 不做 login/session identity。
- 不做 public deployment。
