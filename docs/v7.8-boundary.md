# Agent Space v7.8 Boundary

状态：complete
目标：Human/local queue actions become first-class audit evidence.

## v7.8 新增

- Human Gateway `POST /api/queue/actions` records `go_queue_action` audit entries.
- Successful action records include:
  - `action`
  - `task_id`
  - `source`
  - `status`
  - `result_digest`
- Failed action records include:
  - `action`
  - `task_id`
  - `source`
  - `status`
  - `error`
- Audit verification covers these entries through the existing hash chain.

## 为什么这是 v7.8

v7.5-v7.7 made queue actions real, but Human/local actions were only visible indirectly through queue files and execution receipts.

v7.8 makes the control-plane action itself auditable:

```text
POST /api/queue/actions
  -> apply queue action
  -> go_queue_action audit entry
  -> normal audit hash-chain verification
```

## v7.8 不新增

- 不签名 queue action。
- 不做 auth/login。
- 不做 approval workflow。
- 不把 federation frames 全量改成 action audit。
- 不做 append-only queue action index。

## 验收
