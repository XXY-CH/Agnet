# Agent Space v9.4 Boundary

状态：complete
目标：Go approval state modifications are serialized inside one process.

## v9.4 新增

- `applyApprovalAction` serializes approval read/check/write with a shared mutex.
- The `waitForApproval` expired-state write path uses the same mutex.
- Go package test races concurrent approve calls against one pending approval.
- Only one concurrent approve can succeed.

## 为什么这是 v9.4

The review report identified a TOCTOU race in Human Gateway approvals:

```text
read pending approval
  -> check status
  -> write approved/denied/expired
```

v9.4 fixes the shared in-process mutation path without changing approval files or replacing polling.

## v9.4 不新增

- 不做 cross-process locking。
- 不做 lockfile。
- 不做 channel/sync.Cond approval notification。
- 不做 approval polling rewrite。
- 不做 state-file atomic writes。
- 不做 role/login model。
- 不做 A2A/ARD compatibility。

## 验收

```bash
go test ./cmd/go-fed-discovery
go test ./...
node --test --test-concurrency=1 *.test.mjs
```

The Go package test must fail if concurrent approve calls can both approve the same pending approval.
