# Agent Space v10.7 Boundary

状态：complete
目标：Publish a narrow English ASP Core draft for the implemented proof layer.

## v10.7 新增

- `docs/asp-core-draft.md` describes the current ASP Core proof surface.
- The draft covers identity, Zone trust, `FED_TASK_OPEN`, `FED_RECEIPT`, artifact manifests, local artifact byte checks, and audit hash-chain evidence.
- The draft states that `aid:` remains canonical and `did:key` is only an Ed25519 bridge field.
- The draft keeps A2A compatibility, scheduling, public discovery, remote artifact fetch, receipt stores, batch verification, reputation, payments, and product UI out of scope.
- `docs-contract.test.mjs` guards the required draft boundary phrases.

## 为什么这是 v10.7

v10.0 through v10.6 made the proof layer easier to verify from code and CLI. v10.7 makes that same narrow proof layer understandable to an external implementer without expanding runtime behavior.

## v10.7 不新增

- 不做 npm package。
- 不做 HTTP verifier service。
- 不做 public node。
- 不做 Docker demo。
- 不做 A2A/ANP/AGNTCY compatibility。
- 不做 DID document resolver。
- 不做 scheduler。
- 不做 semantic routing。
- 不做 receipt store/search。
- 不做 batch verification。
