# Agent Space v9.19 Boundary

状态：complete
目标：Go federation TLS listener can require client certificates signed by an operator-provided CA.

## v9.19 新增

- `--tls-client-ca <path>` enables mTLS client certificate verification on the main Go federation listener.
- `--tls-client-ca` requires `--tls-cert` and `--tls-key`.
- The listener reports `transport: "fed+mtls"` when client certificate verification is enabled.
- Go package coverage proves a client without a certificate is rejected and a CA-signed client certificate is accepted.

## 为什么这是 v9.19

Ultimate federation needs transport peers to prove possession of a trusted certificate before protocol frames are accepted.
v9.19 adds the transport-level mTLS gate while leaving protocol identity in the existing authenticated session handshake.

## v9.19 不新增

- 不把 client certificate 映射为 Zone identity。
- 不替代 `HELLO` / `AUTH` authenticated session handshake。
- 不做 public bind address。
- 不做 WebSocket TLS。
- 不做 Human Gateway HTTPS。
- 不做 QUIC。
- 不做 DHT / relay / edge gateway。
- 不做 A2A/ARD compatibility。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 go-fed-discovery.test.mjs
node --test --test-concurrency=1 *.test.mjs
git diff --check
```

The Go package test must fail if `--tls-client-ca` allows clients without certificates or rejects CA-signed client certificates.
