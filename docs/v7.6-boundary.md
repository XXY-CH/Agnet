# Agent Space v7.6 Boundary

状态：complete
目标：Human Gateway can create durable queued work from an already signed task.

## v7.6 新增

- `POST /api/queue/actions` supports `enqueue`.
- Enqueue action accepts:
  - `origin_zone`
  - `requester`
  - signed `task`
- Enqueue action reuses the same verification and queue write path as `FED_TASK_ENQUEUE`.
- `/api/queue` exposes the newly queued item.

## 为什么这是 v7.6

v7.5 exposed explicit local claim/drain actions.

v7.6 adds the missing creation edge without making the Human Gateway a key custodian:

```text
signed task + requester descriptor
  -> POST /api/queue/actions { action: "enqueue" }
  -> verifyTaskOpen
  -> durable queued item
  -> later explicit claim/drain
```

## v7.6 不新增

- 不做 browser-side signing。
- 不做 requester private-key storage。
- 不做 task drafting form。
- 不做 automatic scheduler loop。
- 不做 auth/login。
