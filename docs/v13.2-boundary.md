# Agent Space v13.2 Boundary

状态：complete
目标：Land ASP-native release trust/SBOM evidence over the produced package artifact without claiming ecosystem supply-chain integrations.

## v13.2 新增

- `scripts/release-trust.mjs` consumes the existing package proof path, verifies `state/package-proof/package-proof.json`, and writes `state/package-proof/release-trust.json`.
- `asp-verify.mjs release-trust <release-trust.json> [trusted-release-signers.json]` verifies release trust manifests and optional trusted release signer pins.
- Release trust evidence binds package `name`, `version`, `filename`, `tarball`, tarball byte `sha256`, tarball `size`, package proof digest, release signer identity, and packaged file list.
- The release trust signer requires `release.trust.sign`, and trusted release signer descriptors must carry the same capability.
- Negative gates reject malformed manifests, unsafe package proof and tarball paths, stale package proof digests, mismatched bound fields, invalid `released_at`, missing signatures, bad signatures, missing signer capability, untrusted release signers, malformed trusted signer lists, and extra CLI arguments.

## 为什么这是 v13.2

v13.1 landed verifier-owned reachability scope mechanics but still leaves the real hosted external-host observer run pending. v13.2 advances the latest complete protocol tag to `v13.2-protocol` because release trust/SBOM closes completely with local producer, verifier, negative gates, and documentation guards. The v13.1 hosted external-host evidence gap remains visible as pending work; this supersession records the latest complete slice without rewriting v13.1 history.

## 硬边界

- Format is `asp-release-trust/v1`: not CycloneDX, not SPDX, not SLSA provenance, not npm registry signing, not package publish, not release transparency, not a generic supply-chain platform.
- Evidence binds name, version, tarball bytes, package proof digest, signer identity, and file list from the verified package proof.
- Stale release trust means the referenced package proof drifted from `package_proof_digest`; releases do not expire by elapsed time.
- Release trust fails closed when unsigned, signed by a descriptor without `release.trust.sign`, signed by the wrong release signer under an optional trust pin, or bound to mismatched package proof fields.
- Trusted release signer pinning covers the release signer only; it does not pin the embedded package proof signer.
- Protocol tag advances to `v13.2-protocol` as the latest complete slice while the v13.1 hosted external-host observer run remains pending and visible.
