# Agent Space v8.25 Boundary

状态：complete
目标：Human Gateway direct approvals preserve a named local human actor in signed approval evidence.

## v8.25 新增

- `/api/approvals/actions` accepts explicit `human://...` approval actors instead of only `human://local`.
- Signed approval grants preserve the actor in `by`.
- `approval.granted` events and receipts carry the same signed actor evidence.
- The Go integration test approves a direct tool task as `human://operator` and verifies that actor through the grant and receipt.

## 为什么这是 v8.25

Ultimate governance needs approval evidence to identify which human authority acted.
Before v8.25, direct Human Gateway approvals were forced through one local constant.

v8.25 keeps the smallest local version:

```text
Human Gateway approval action
  -> human://operator
  -> Zone-signed approval grant
  -> receipt approval_grants
```

## v8.25 不新增

- 不做 login/session store。
- 不做 roles/admin model。
- 不做 approval actor policy。
- 不做 token storage / rotation。
- 不做 public deployment。
- 不做 A2A/ARD compatibility。

## 验收

```bash
go test ./...
```
