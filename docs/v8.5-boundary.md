# Agent Space v8.5 Boundary

状态：complete
目标：Human Gateway exposes its local deployment security posture as a read-only API.

## v8.5 新增

- Human Gateway serves `GET /api/security`.
- The security posture response records:
  - `listen_host: "127.0.0.1"`
  - `write_token_required`
  - `public_transport: false`
- The response is derived from the existing localhost bind and `--human-token` flag.

## 为什么这是 v8.5

v8.4 added a local bearer-token gate for Human Gateway writes.

v8.5 makes that deployment boundary machine-readable before adding any real public transport:

```text
localhost Human Gateway
  -> /api/security
  -> explicit local-only and token-required posture
```

This lets future UI, deployment scripts, and tests inspect the gateway boundary without pretending the gateway is production-authenticated.

## v8.5 不新增

- 不做 login/session identity。
- 不做 role model。
- 不做 CSRF/session cookies。
- 不做 TLS/QUIC/public transport。
- 不做 token storage or rotation。
- 不开放非 localhost bind。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 test/go-fed-discovery.test.mjs
node --test --test-concurrency=1 test/*.test.mjs
```

The Go integration test must fail if `/api/security` is missing, if it omits the localhost bind posture, or if it stops reporting whether Human Gateway writes require the configured bearer token.
