# Agent Space v0.8 Boundary

状态：v0.8 candidate  
目标：定义 Zone-local revocation，让 Zone 可以拒绝继续解析某个 `aid` 或 `agent://`。

## v0.8 新增

- `zoneRevocation(zone, subject, reason)`。
- `verifyZoneRevocation(revocation, zoneDescriptor)`。
- `verifyNotRevoked(entry, descriptor, alias)`。
- Registry 支持 `revocations` 数组。
- `resolveAgent()` 在验证 descriptor 和 Zone binding 后检查 revocation。
- 新增 revoked aid 测试。
- 新增 tampered revocation 测试。

Revocation body：

```json
{
  "zone": "zid:ed25519:...",
  "subject": "aid:ed25519:...",
  "reason": "compromised"
}
```

`subject` 可以是：

```text
aid:ed25519:...
agent://local/summarizer
```

## 为什么这是 v0.8

v0.7 允许 Zone 批准 alias rebinding。

v0.8 允许 Zone 拒绝继续解析某个身份或别名：

```text
valid descriptor
valid Zone binding
valid revocation
  -> resolver rejects
```

这让 Zone 开始具备最小生命周期治理能力。

## v0.8 不新增

- 不做全球 revocation。
- 不做 revocation propagation。
- 不做 expiry。
- 不做 revocation sequence number。
- 不做 dispute recovery。
- 不做 compromised-key automatic recovery。

这些需要 Federation 和 Zone policy。
