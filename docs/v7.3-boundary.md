# Agent Space v7.3 Boundary

状态：complete
目标：claimed queue item 的 lease 过期后不能 drain，只能被显式 reclaim 换 owner/lease。

## v7.3 新增

- `FED_QUEUE_CLAIM` records `lease_expires_at`。
- `FED_QUEUE_DRAIN` rejects expired leases。
- Go gateway accepts `FED_QUEUE_RECLAIM` with `task_id` and `owner`。
- Reclaim only accepts expired `claimed` queue items。
- Reclaim changes:
  - `lease_owner`
  - `lease_id`
  - `lease_expires_at`

## 为什么这是 v7.3

v7.2 proved explicit ownership before drain。

v7.3 adds the crash-recovery handoff point without adding a scheduler loop:

```text
queued
  -> claim(owner, lease_expires_at)
  -> claimed + lease_id
  -> drain rejected after lease expiry
  -> reclaim(new_owner)
  -> claimed + new lease_id + new lease_expires_at
  -> drain(new_lease_id)
  -> completed
```

## v7.3 不新增

- 不做 automatic lease scan。
- 不做 automatic drain loop。
- 不做 retry/backoff。
- 不做 priority。
- 不做 multi-node scheduler。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 go-fed-discovery.test.mjs
```

The Go integration test must fail if an expired lease can drain, if reclaim cannot swap ownership for an expired claim, or if the reclaimed lease cannot drain successfully。
