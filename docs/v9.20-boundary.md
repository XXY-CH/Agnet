# Agent Space v9.20 Boundary

状态：complete
目标：Go federation mTLS client certificates are bound to the claimed `HELLO.origin_zone`.

## v9.20 新增

- When a federation TCP connection presents a TLS client certificate, the certificate URI SANs must include the claimed `origin_zone.zid`.
- A client certificate signed by the configured CA can no longer claim a different trusted Zone during `HELLO`.
- Go package coverage proves a mismatched certificate Zone is rejected before `AUTH`.

## 为什么这是 v9.20

v9.19 proved mTLS client certificate verification.
v9.20 closes the next trust gap: CA acceptance alone is not Zone identity unless the transport certificate is bound to the protocol Zone claim.

## v9.20 不新增

- 不设计完整 PKI policy。
- 不做 certificate revocation / OCSP。
- 不替代 `HELLO` / `AUTH` signatures。
- 不做 public bind address。
- 不做 WebSocket TLS。
- 不做 Human Gateway HTTPS。
- 不做 QUIC。
- 不做 DHT / relay / edge gateway。
- 不做 A2A/ARD compatibility。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 test/go-fed-discovery.test.mjs
node --test --test-concurrency=1 test/*.test.mjs

```

The Go package test must fail if a CA-signed client certificate for one Zone can send `HELLO` as another trusted Zone.
