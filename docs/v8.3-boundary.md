# Agent Space v8.3 Boundary

状态：complete
目标：Human Gateway draft endpoint can accept a browser/external requester descriptor with an already signed task.

## v8.3 新增

- `POST /api/queue/drafts` accepts `requester` and signed `task`.
- Externally signed draft tasks keep the original requester `aid` and task signature.
- External draft enqueue reuses the existing actor-bound queue action grant path.
- The resulting queued task is visible through `/api/queue`.
- `go_queue_action` audit records the local enqueue action with actor policy evidence.

## 为什么这是 v8.3

v7.16 proved the Human Gateway can locally draft/sign queued work.

v8.3 removes that custody assumption for the draft endpoint:

```text
browser/external requester key
  -> signed task + requester descriptor
  -> Human Gateway /api/queue/drafts
  -> existing queue action grant + enqueue path
```

This is the smallest browser-key boundary: the gateway can carry a browser-held signature without owning the requester private key.

## v8.3 不新增

- 不做 browser private-key storage。
- 不做 WebCrypto UI。
- 不做 login/session identity。
- 不做 requester key management UI。
- 不做 public transport/deployment hardening。
- 不做 automatic scheduler loop。

## 验收

The Go integration test must fail if `/api/queue/drafts` rewrites the external requester, replaces the signed task signature, skips the existing enqueue path, or omits the `go_queue_action` audit evidence.
