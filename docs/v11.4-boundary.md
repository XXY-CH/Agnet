# Agent Space v11.4 Boundary

状态：complete
目标：Bind signed receipts to the signed task object they claim to complete.

## v11.4 新增

- Node local runtime receipts include `task_digest`, the SHA-256 digest of the canonical signed task object.
- Node federation receipts include `task_digest` for incoming `FED_TASK_OPEN` frames.
- Go federation execution receipts include `task_digest` for the signed task object received at execution time.
- Go cancellation receipts include `task_digest` for the signed cancellation request object.
- Node `verifyFederatedReceipt` rejects receipts whose `task_digest` is missing or is not a 64-hex digest.
- Go CLI and reusable `agnet/verifier.VerifyFederatedReceipt` paths reject the same missing or malformed `task_digest`.
- The shared `FED_RECEIPT` conformance vector now carries a `task_digest`, so Node and Go package tests exercise the field.

## 为什么这是 v11.4

The Phase A / H3-H4 feedback called out that a receipt identified a task by `task_id` and parties, but did not carry a compact causal anchor for the exact signed task body. `task_digest` closes that immediate proof gap without storing raw task bodies in receipts and without introducing a task-store lookup layer.

## v11.4 不新增

- 不存 raw task in every receipt。
- 不实现 receipt store/search。
- 不实现 external task-store lookup against `task_digest`。
- 不验证 `task_digest` against a separately supplied task frame。
- 不改变 artifact manifest verification semantics。
- 不实现 scheduler-owned routing。
- 不实现 dynamic Swarm decomposition。
- 不实现 A2A/ARD compatibility。
