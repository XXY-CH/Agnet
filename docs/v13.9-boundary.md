# v13.9 Boundary: Hosted Observer Runner

状态：complete

目标：让真实 hosted external-host reachability attempt 有可复现的 runner、显式监听地址和 keepalive window，同时诚实记录当前 IPv6 hosted runner blocker。

## What Landed

- `scripts/public-node-proof.mjs` accepts `AGNET_PUBLIC_LISTEN_HOST` so the public-listen proof can bind an explicit global literal IP instead of auto-selecting a local interface address.
- `scripts/public-node-proof.mjs` accepts `AGNET_PUBLIC_PROOF_KEEPALIVE_MS` so the proof listener can stay alive while a separate hosted observer attempts the TCP connection.
- `.github/workflows/hosted-reachability-observer.yml` defines the Hosted Reachability Observer workflow.
- The workflow decodes verifier-ready proof bundle inputs into `state/hosted/`, runs `scripts/external-reachability-observer.mjs` with `external-host`, and then runs `asp-verify.mjs proof-bundle` against the observed bundle and observer trusted-Zone file.
- `test/hosted-reachability-workflow.test.mjs` locks the workflow shape so it keeps running the external-host observer and verifier.
- `test/public-node-proof.test.mjs` covers explicit global IPv6 listen host support when the machine has one.

## Recorded Hosted Attempt

- Workflow run `28916288568` reached the GitHub hosted observer step.
- The runner failed to connect to the current IPv6 listener with `ENETUNREACH`.
- This means the runner path exists, but the real hosted external-host observer run is still pending.

## Non-Goals

- This does not claim real hosted/public reachability.
- This does not make `ENETUNREACH` a successful external-host proof.
- This does not make container-observer evidence count as hosted external-host evidence.
- This does not support hostname `external-host` targets; the verifier still requires a globally routable literal-IP `listen_host`.
- This does not add NAT traversal, tunnel hostname support, relay networking, QUIC, DHT, or production deployment automation.

## Why This Boundary

v13.1 already made `external-host` verifier-owned: it requires trusted signed observer evidence, the same transport proof, the same receipt digest, and a globally routable literal-IP listener. v13.8 made the observer identity pre-pinnable. The missing operational support was a reproducible hosted observer path and a way to keep the proof listener alive while that path runs.

v13.9 closes that support gap, then records the first real hosted attempt without overclaiming it. The failure is useful evidence: GitHub-hosted runner networking cannot reach the current IPv6 listener from this environment. The remaining close path needs an IPv4-routable listener, a hosted observer with IPv6 route, or a real hosted node with a globally routable literal IP.

## Test Contract

The tests must fail if:

- Public docs claim the hosted external-host run has finished.
- The Hosted Reachability Observer workflow stops invoking `external-reachability-observer.mjs` with `external-host`.
- The workflow stops running `asp-verify.mjs proof-bundle`.
- `AGNET_PUBLIC_LISTEN_HOST` is no longer documented as the explicit listener override.
- `AGNET_PUBLIC_PROOF_KEEPALIVE_MS` is no longer documented as the hosted observation window.
- The recorded `28916288568` / `ENETUNREACH` blocker disappears from public status docs.
