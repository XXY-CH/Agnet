# Agent Space v2.2 Boundary

状态：v2.2 complete
目标：让 Go discovery gateway 动态生成 worker descriptor。

## v2.2 新增

- `worker_seed_hex` test fixture field。
- `worker_profile` test fixture field：
  - alias
  - transports
  - capabilities
  - policy
- Go discovery gateway derives the worker private key from seed。
- Go discovery gateway dynamically generates:
  - worker `public_key_spki`
  - worker `aid`
  - worker `descriptor_signature`
- Node client continues to verify:
  - Go-generated worker descriptor
  - Go-signed Zone binding
  - Go-signed capability credential

## 为什么这是 v2.2

v2.1 已经把 Zone binding 和 capability credential 的签发动作迁到 Go。

v2.2 继续把 discovery response 里的 worker identity 从预签名 fixture 移到 Go：

```text
Go reads worker seed + profile
  -> derives worker public key
  -> computes aid
  -> signs worker descriptor
  -> signs Zone binding and capability credential
  -> Node verifies
```

测试会用删除了 `worker` descriptor 和旧 `zone_binding` 的 fixture 启动 Go gateway，证明 Go 不再搬运预签名 worker descriptor。

## v2.2 不新增

- 不做 Go key store。
- 不做 Go task execution。
- 不做 Go audit writer。
- 不做 Go `FED_TASK_OPEN` verification。
- 不做 credential status feed。
