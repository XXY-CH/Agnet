# Agent Space v3.9 Boundary

状态：v3.9 complete
目标：把外部/MCP tool approval 从 simulated event 升级为 signed local approval evidence，并把 tool execution 的本地隔离证据写入 receipt。

## v3.9 新增

- Go creates a signed approval grant before running approval-gated tools。
- Approval grants are signed by the Zone authority key with `approval_signature`。
- `approval.granted` event carries the signed grant。
- Receipt records:
  - `approvals: ["tool"]`
  - `approval_grants`
  - `sandbox`
- Go audit verifier checks approval grant count and signature。
- Human Gateway receipt table shows signed approval count and sandbox mode。
- External/MCP tools run with:
  - local temporary working directory
  - restricted `PATH=/usr/bin:/bin`
  - empty inherited environment
  - sandbox evidence recorded in receipt

## 为什么这是 v3.9

v3.8 proved that MCP/external tool execution cannot happen without approval events。

v3.9 makes that approval auditable:

```text
approval.required
  -> signed approval grant by Zone authority
  -> approval.granted carries grant
  -> tool runs in local temp-dir sandbox
  -> receipt records approval_grants + sandbox
  -> audit verifier checks grant signature
```

This is still a local proof, not a deployed approval product。

## v3.9 不新增

- 不做 login。
- 不做 pending task queue。
- 不做 rejection path。
- 不做 click-to-approve flow。
- 不做 container/gVisor/Firecracker sandbox。
- 不做 OS-level network blocking。
- 不做 long-running MCP sessions。

## 验收

```bash
go test ./...
node --test --test-concurrency=1 *.test.mjs
```

The Go integration test must fail if MCP tool execution lacks signed approval evidence or sandbox evidence。
